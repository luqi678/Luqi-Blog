## 二、题目描述

公园园区提供小火车单向通行，从园区站点编号最小到最大通行（例如 1 -> 2 -> 3 -> 4 -> 1），供员工在各个办公园区穿梭。通过对公司 $N$ 个员工调研统计到每个员工的坐车区间，包含前后站点。请设计一个程序计算出小火车在哪个园区站点时人数最多。

**注意：**
- 坐车区间包含起始站和终点站。
- 如果有多站人数相同且均为最大，返回编号最小的园区站点。

## 二、输入描述

- 第一行：为一个整数 $N$，表示调研的员工人数。
- 接下来的 $N$ 行：每行为每个员工的开始上车站点和下车站点，用空格分隔。

## 三、输出描述

- 人数最多时的园区站点编号。如果人数最小时有多个站点，返回编号最小的那个。

## 四、用例

### 用例1

**输入：**
```text
3
1 3
2 4
1 4
```

**输出：**
```text
2
```

**说明：**
- 1号员工：在 1 站上车，3 站下车，经过站点：[1, 2, 3]
- 2号员工：在 2 站上车，4 站下车，经过站点：[2, 3, 4]
- 3号员工：在 1 站上车，4 站下车，经过站点：[1, 2, 3, 4]
- 各站点人数统计：
    - 1号站：2人（员工1, 3）
    - 2号站：3人（员工1, 2, 3）
    - 3号站：3人（员工1, 2, 3）
    - 4号站：2人（员工2, 3）
- 人数最多为3人，对应站点2和3，取最小编号，输出 2。

### 用例2

**输入：**
```text
2
5 10
1 5
```

**输出：**
```text
5
```

**说明：**
- 5号站点同时属于两个区间，人数为2，是最大值。

## 五、备注

- $N$ 的范围通常在 $10^5$ 以内。
- 站点的范围可能较大，建议考虑离散化或使用 Map 记录差分，但如果站点编号在合理范围内（如 $10^6$），直接开数组即可。

## 六、题目解析

### 1. 题目分析
本题属于典型的**区间覆盖统计**问题。给定若干个闭区间 $[S, E]$，要求找出被覆盖次数最多的点。由于题目要求“包含前后站点”，即在 $S$ 站和 $E$ 站，乘客都在车上。

### 2. 解题思路
处理区间覆盖问题最高效的方法是**差分数组 (Difference Array)**：

1.  **记录变化量**：对于每个员工的坐车区间 $[S, E]$，我们在 $S$ 站点人数 $+1$，在 $E+1$ 站点人数 $-1$。
2.  **求前缀和**：由于差分数组的前缀和即为当前位置的累计值，我们从最小的站点编号开始遍历到最大的站点编号，计算前缀和。
3.  **找最大值**：在求前缀和的过程中，实时记录当前人数的最大值 `maxCount` 以及产生该最大值的最小索引 `ansStation`。

由于站点编号可能不是连续的或者分布较广，我们可以使用 `TreeMap` 来存储差分值。`TreeMap` 会自动按站点编号升序排序，方便我们进行前缀和计算。

### 3. 代码实现

#### Java实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        
        // 使用 TreeMap 存储差分值，自动按站点编号排序
        // key: 站点编号, value: 该站点的变化量
        TreeMap<Integer, Integer> diff = new TreeMap<>();
        
        for (int i = 0; i < n; i++) {
            int start = sc.nextInt();
            int end = sc.nextInt();
            
            // 区间 [start, end] 均包含，所以 end 之后才下车
            // start 站人数 +1
            diff.put(start, diff.getOrDefault(start, 0) + 1);
            // end + 1 站人数 -1 (表示在 end 站还在车上，end+1 站才不在)
            diff.put(end + 1, diff.getOrDefault(end + 1, 0) - 1);
        }
        
        int maxPeople = 0;
        int currentPeople = 0;
        int bestStation = -1;
        
        // 遍历有序的差分记录点
        for (Map.Entry<Integer, Integer> entry : diff.entrySet()) {
            int station = entry.getKey();
            int change = entry.getValue();
            
            // 在遇到下一个关键站点之前，人数是保持不变的
            // 所以我们只需要在每个关键点更新 currentPeople
            currentPeople += change;
            
            // 注意：题目要求包含前后站点。
            // 差分数组在 station 点的值代表的是 [station, next_station - 1] 这个范围的人数
            // 因此我们需要检查当前的 currentPeople
            if (currentPeople > maxPeople) {
                maxPeople = currentPeople;
                bestStation = station;
            }
            // 因为 TreeMap 是升序遍历，如果 currentPeople == maxPeople，
            // 此时的 station 一定比之前的 bestStation 大，
            // 而题目要求返回最小的编号，所以不需要更新 bestStation。
        }
        
        System.out.println(bestStation);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N \log M)$，其中 $N$ 是员工人数，$M$ 是不同站点的数量。主要开销在 `TreeMap` 的插入和遍历。
- **空间复杂度**：$O(M)$，用于存储每个站点的差分值。
