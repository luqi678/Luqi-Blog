## 一、 题目描述

部门在进行需求开发时需要进行人力安排。当前部门需要完成 `N` 个需求，需求用 `requirements[i]` 表示，`requirements[i]` 表示第 `i` 个需求的工作量大小，单位：人月。这部分需求需要在 `M` 个月内完成开发，进行人力安排后每个月的人力是固定的。

目前要求**每个月最多有 2 个需求开发**，并且每个月需要完成的需求不能超过部门人力。

请帮部门评估在满足需求开发进度的情况下，每个月需要的**最小人力**是多少？

## 二、输入描述

输入第一行为 `M`，表示需要开发时间要求。
输入第二行为 `requirements`，表示每个需求工作量大小，以空格分隔。

**约束说明：**
- $N$ 为 `requirements` 的长度。
- $1 \le N/2 \le M \le N \le 10000$
- $1 \le requirements[i] \le 10^9$

## 三、输出描述

对于每一组测试数据，输出部门需要人力需求，行末无多余的空格。

## 四、备注

- 本题的核心在于寻找一个满足条件的最小值，属于典型的**二分查找（Binary Search）**应用场景。
- 题目限定每个月最多做 2 个需求，且 $M \ge N/2$，这意味着我们主要的策略是如何尽可能多地凑成“一对”任务，使得它们和不超过限制。
- 这也是一个**贪心 + 双指针**的问题：为了让用的月数最少，在给定人力限制下，应该尝试把最大的任务和最小的任务组合。如果这两个加起来都超标，那最大的任务只能单独占用一个月。

## 五、用例

| 输入 | 输出 | 说明 |
| :--- | :--- | :--- |
| `3`<br>`3 5 3 4` | `6` | 需求列表为 `[3, 5, 3, 4]`，排序后 `[3, 3, 4, 5]`。<br>**假设人力为 5**：<br> `5` 单独一月，`4` 单独一月，`3` 单独一月，`3` 单独一月 -> 共 4 个月 > M(3)，不可行。<br>**假设人力为 6**：<br> `5` 只能和空配对(因为5+3>6)，单独一月；`4` 可以和 `3` 配对(4+3=7>6不行，此处需注意逻辑，排序后配对逻辑是 `5` 单独, `4` 单独, 剩下两个 `3` 可以合吗? <br>正确贪心逻辑：<br>人力6：<br>最大 `5`，最小 `3` -> `5+3=8 > 6`，`5` 必须单独一月。剩 `[3, 3, 4]`。<br>最大 `4`，最小 `3` -> `4+3=7 > 6`，`4` 必须单独一月。剩 `[3, 3]`。<br>最大 `3`，最小 `3` -> `3+3=6 <= 6`，合一月。<br>总共需要 3 个月，满足条件。 |
| `3`<br>`10 20 30 40` | `50` | 排序后 `[10, 20, 30, 40]`。<br>人力 50 时：`40+10=50` (1月), `30+20=50` (1月)。共2个月 < 3，满足。 |

## 六、题目解析

### 核心思路

1.  **单调性分析**：
    * 如果给定的部门人力无穷大，肯定能在 $M$ 个月内完成（甚至 $N/2$ 个月）。
    * 如果人力太少（比如小于最大那个需求），则连那个需求都完成不了，肯定完不成任务。
    * 随着人力 `k` 的增加，能完成任务的可能性单调递增。因此可以使用**二分查找**来确定最小的 `k`。

2.  **二分查找范围**：
    * 下界 (`low`)：所有需求中的最大值（必须至少能完成最大的那个任务）。
    * 上界 (`high`)：所有需求的总和（最坏情况），或者更紧凑一点，最大的两个需求之和（因为一个月最多做两个）。

3.  **Check 函数逻辑 (贪心 + 双指针)**：
    给定一个人力限制 `limit`，判断是否能在 `M` 个月内完成：
    * 首先将需求数组**排序**。
    * 使用双指针，`left` 指向最小元素，`right` 指向最大元素。
    * 循环条件 `left <= right`：
        * 如果 `requirements[left] + requirements[right] <= limit`：说明最大和最小可以拼在同一个月做完。消耗 1 个月，`left++`, `right--`。
        * 否则（和大于 limit）：说明最大的那个任务太重了，连最小的任务都带不动，它必须单独占用一个月。消耗 1 个月，`right--`。
        * 记录使用的月数 `count`。
    * 最后判断 `count <= M`。

### 算法实现 (Java)

```java
import java.util.Arrays;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取 M
        if (!sc.hasNext()) return;
        int m = sc.nextInt();
        
        // 读取 requirements
        // 注意：题目输入格式可能是一行字符串，也可能是先读M再读N再读数组
        // 根据题目描述 "输入第一行为 M ，第二行为 requirements"，这里处理方式如下：
        // 由于没有直接给出 N，我们需要读取这一行所有的整数
        
        // 为了兼容机考常见输入模式，建议先读取整行再解析
        // 但如果用 Scanner 的 nextInt() 只要是空白符分隔都能读到
        // 这里假设输入流中接下来的所有整数都是 requirements
        
        // 实际上机考中往往不知道 N 的具体大小，或者题目描述略有出入
        // 比较稳妥的方式是读取一行解析
        if (!sc.hasNextLine()) return; // 消耗掉 M 后的换行符
        sc.nextLine(); 
        
        if (!sc.hasNextLine()) return;
        String line = sc.nextLine();
        int[] requirements = Arrays.stream(line.split(" "))
                                   .mapToInt(Integer::parseInt)
                                   .toArray();
        
        System.out.println(solve(m, requirements));
    }

    public static long solve(int m, int[] requirements) {
        int n = requirements.length;
        Arrays.sort(requirements);

        // 二分查找的范围
        // 下界：最大单个任务的工作量
        long low = requirements[n - 1];
        // 上界：所有任务总和 (或者最大的两个之和，这里用总和保底)
        long high = 0;
        for (int req : requirements) {
            high += req;
        }

        long ans = high;

        while (low <= high) {
            long mid = low + (high - low) / 2;
            if (check(requirements, m, mid)) {
                ans = mid;
                high = mid - 1; // 尝试更小的人力
            } else {
                low = mid + 1; // 人力不够，增加
            }
        }
        return ans;
    }

    /**
     * 检查在 limit 的人力限制下，是否能在 m 个月内完成
     */
    public static boolean check(int[] requirements, int m, long limit) {
        int left = 0;
        int right = requirements.length - 1;
        int monthsUsed = 0;

        while (left <= right) {
            // 如果最大的任务加上最小的任务 <= 限制，则两人组队
            if (requirements[left] + requirements[right] <= limit) {
                left++;
                right--;
            } else {
                // 否则最大的任务只能单独干
                right--;
            }
            monthsUsed++;
        }

        return monthsUsed <= m;
    }
}
```
