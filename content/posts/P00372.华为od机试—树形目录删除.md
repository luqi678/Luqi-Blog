## 一、 题目描述

某文件系统中有 $N$ 个目录，每个目录都一个独一无二的 ID。每个目录只有一个父目录，但每个父目录下可以有零个或者多个子目录，目录结构呈树状结构。

假设，根目录的 ID 为 0，且根目录没有父目录，其他所有目录的 ID 用唯一的正整数表示，并统一编号。

现给定目录 ID 和其父目录 ID 的对应父子关系表 `[子目录ID，父目录ID]`，以及一个待删除的目录 ID，请计算并返回一个 ID 序列，表示因为删除指定目录后剩下的所有目录，返回的 ID 序列以递增序输出。

**注意：**
1. 被删除的目录或文件编号一定在输入的 ID 序列中；
2. 当一个目录删除时，它所有的子目录都会被删除。

## 二、输入描述

1. 输入的第一行为父子关系表的长度 $m$；
2. 接下来的 $m$ 行为 $m$ 个父子关系对（子目录ID 父目录ID）；
3. 最后一行为待删除的 ID。
4. 序列中的元素以空格分割。

## 三、输出描述

输出一个序列，表示因为删除指定目录后，剩余的目录 ID，以空格分隔。

## 四、用例

### 用例1

**输入**
```text
5
8 6
10 8
6 0
20 8
2 6
8
```

**输出**
```text
2 6
```

**说明**
目录结构如下：
- 0 (根目录)
    - 6
        - 8
            - 10
            - 20
        - 2

删除目录 8，及其所有子目录（10 和 20）后，剩下的目录有 2 和 6。

### 用例2

**输入**
```text
3
1 0
2 1
3 2
1
```

**输出**
```text
 
```

**说明**
删除目录 1 后，其子目录 2 和 3 也被删除，没有剩余目录（根目录 0 不在输出范围内，输出为空）。

## 五、备注

- $N$ 的范围较大。
- 根目录 ID 为 0，不记录在父子关系表的第一列。
- 输出的 ID 序列需升序排列。

## 六、题目解析

### 1. 题目分析

本题的核心是处理一个树形结构。给定若干对 `(childID, parentID)`，我们需要构建出这棵树，然后删除指定的节点及其所有子树节点，最后输出剩下的节点。

关键点：
- 只有出现在“子目录ID”列的 ID 才是需要考虑输出的目录（根目录 0 除外）。
- 删除一个节点意味着要删除以该节点为根的整个子树。
- 最终结果需要按 ID 从小到大排序。

### 2. 解题思路

1.  **数据存储**：
    - 使用一个 `Map<Integer, List<Integer>>` 来存储父子关系，其中 Key 是父目录 ID，Value 是该父目录下的子目录列表。
    - 使用一个 `Set<Integer>` 或 `List<Integer>` 存储所有存在的目录 ID（即所有出现在输入第一列的子目录 ID）。
2.  **构建树**：遍历输入的 $m$ 对关系，填充 Map。
3.  **查找待删除节点及其后代**：
    - 使用深度优先搜索（DFS）或广度优先搜索（BFS）从待删除的 `targetID` 开始遍历。
    - 将遍历到的所有 ID 存入一个 `Set`（称为 `toDelete`）。
4.  **过滤与排序**：
    - 遍历所有原始目录 ID，如果该 ID 不在 `toDelete` 集合中，则将其加入结果列表。
    - 对结果列表进行升序排序。
5.  **输出**：按空格分隔打印。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();

        // 存储父子关系：parentID -> list of childrenID
        Map<Integer, List<Integer>> tree = new HashMap<>();
        // 存储所有出现的子目录ID
        Set<Integer> allNodes = new HashSet<>();

        for (int i = 0; i < m; i++) {
            int child = sc.nextInt();
            int parent = sc.nextInt();
            allNodes.add(child);
            tree.computeIfAbsent(parent, k -> new ArrayList<>()).add(child);
        }

        int target = sc.nextInt();

        // 查找所有需要删除的节点
        Set<Integer> toDelete = new HashSet<>();
        findChildren(target, tree, toDelete);

        // 准备结果列表
        List<Integer> result = new ArrayList<>();
        for (int node : allNodes) {
            if (!toDelete.contains(node)) {
                result.add(node);
            }
        }

        // 排序并输出
        Collections.sort(result);
        for (int i = 0; i < result.size(); i++) {
            System.out.print(result.get(i) + (i == result.size() - 1 ? "" : " "));
        }
        System.out.println();
    }

    /**
     * 递归查找并记录待删除节点及其子孙节点
     */
    private static void findChildren(int current, Map<Integer, List<Integer>> tree, Set<Integer> toDelete) {
        toDelete.add(current);
        if (tree.containsKey(current)) {
            for (int child : tree.get(current)) {
                findChildren(child, tree, toDelete);
            }
        }
    }
}
```

### 4. 复杂度分析

-   **时间复杂度**：$O(M + N \log N)$。其中 $M$ 是关系对数量，构建树和 DFS 遍历时间复杂度为 $O(M)$。最后排序的时间复杂度取决于剩余节点的数量 $N$，即 $O(N \log N)$。
-   **空间复杂度**：$O(M)$。主要消耗在存储树结构的 Map 和存储所有节点的 Set。
