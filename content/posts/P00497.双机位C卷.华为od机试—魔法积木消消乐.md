## 一、 题目描述

小明在玩一个积木消消乐的游戏。桌面上放着 $n$ 堆积木，每堆积木的数量不尽相同。小明拥有一种特殊的“减半魔法”，魔法的使用规则如下：

1. 每次施放魔法时，小明可以指定一个当前桌面上存在的积木数量 $V$。
2. 施放魔法后，桌面上所有数量为 $V$ 的积木堆，其数量都会瞬间变为原来的一半（向下取整，即 $V' = \lfloor V/2 \rfloor$）。
3. 当某堆积木的数量变为 0 时，该堆积木被视为“收完”。

小明希望使用最少的魔法次数将所有的积木堆全部收完。请你帮他计算出这个最少次数。

## 二、输入描述

第一行输入一个正整数 $n$，表示积木的堆数（$1 < n < 10^5$）。  
第二行输入 $n$ 个正整数，表示每堆积木的初始个数，数值之间用空格分隔（每个数值范围 $[1, 10^9]$）。

## 三、输出描述

输出一个整数，表示收完所有积木堆所需的最少魔法次数。

## 四、用例

### 用例1

**输入：**
```
4
4 4 4 4
```

**输出：**
```
3
```

**说明：**
- 初始状态：[4, 4, 4, 4]。
- 第一次魔法：指定 $V=4$。所有堆变为 $4 // 2 = 2$。状态变为 [2, 2, 2, 2]。
- 第二次魔法：指定 $V=2$。所有堆变为 $2 // 2 = 1$。状态变为 [1, 1, 1, 1]。
- 第三次魔法：指定 $V=1$。所有堆变为 $1 // 2 = 0$。状态变为 [0, 0, 0, 0]。
共需 3 次。

### 用例2

**输入：**
```
2
3 4
```

**输出：**
```
4
```

**说明：**
- 初始状态：[3, 4]。
- 整个过程中出现的不同非零数量集合为 {4, 3, 2, 1}。
- 无论先后顺序，每一个出现在桌面上的非零数值 $V$ 都必须被指定一次魔法才能变为其一半。
- 4 的变化过程：4 -> 2 -> 1 -> 0。
- 3 的变化过程：3 -> 1 -> 0。
- 合并所有过程中的非零值：{4, 2, 1} ∪ {3, 1} = {4, 3, 2, 1}。
- 因此最少需要 4 次魔法。

## 五、备注

- 魔法只能作用于当前存在的积木数量。
- 只要数值相同，一次魔法可以同时处理多堆。
- 目标是计算产生的所有不同非零数值的个数。

## 六、题目解析

### 1. 题目分析
本题的核心在于理解“魔法”的作用范围：它是针对“数值”而非“某一堆”。如果桌面上有多堆积木的数量都是 $V$，那么一次魔法可以让它们全部减半。

### 2. 解题思路
为了让魔法次数最少，我们需要充分利用“一次魔法处理所有相同数值”的特性。
1. 对于任何一堆积木，其数量 $X$ 要变为 0，必须经历一系列的减半过程：$X \to \lfloor X/2 \rfloor \to \dots \to 0$。
2. 在这个过程中，产生的每一个非零的数值，都必须作为一次魔法的“指定值 $V$”。
3. 如果不同堆在减半过程中产生了相同的数值，这些数值只需要施放一次魔法即可。
4. **结论：** 题目要求的最少次数，实际上就是所有积木堆在不断减半至 0 的过程中，所产生的**所有不同非零数值**的个数。

### 3. 代码实现

#### Java8实现

```java
import java.util.HashSet;
import java.util.Scanner;
import java.util.Set;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取积木堆数
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        
        // 使用 HashSet 记录所有出现过的不同非零积木数量
        Set<Integer> uniqueValues = new HashSet<>();
        
        for (int i = 0; i < n; i++) {
            if (sc.hasNextInt()) {
                int val = sc.nextInt();
                // 将当前堆在变为0的过程中产生的所有值加入集合
                while (val > 0) {
                    // 如果集合中已存在该值，说明其他堆已经或将会处理这个数值，
                    // 且其后续产生的 val/2 也会被处理，可以提前跳出优化
                    if (uniqueValues.contains(val)) {
                        break;
                    }
                    uniqueValues.add(val);
                    val /= 2;
                }
            }
        }
        
        // 集合的大小即为最少魔法次数
        System.out.println(uniqueValues.size());
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(n \cdot \log(\text{max\_val}))$。其中 $n$ 是积木堆数，$\text{max\_val}$ 是积木的最大初始数量（$10^9$）。由于 $\log_2(10^9) \approx 30$，对于 $10^5$ 的数据量，计算次数约为 $3 \times 10^6$，在 1 秒限制内非常充裕。
- **空间复杂度**：$O(n \cdot \log(\text{max\_val}))$，主要取决于 `HashSet` 存储的不同数值个数。最坏情况下（所有路径不重叠），空间占用也属于可接受范围。
