## 二、 题目描述

给定一个正整数 $n$，如果能够分解为 $m (m > 1)$ 个连续正整数之和，请输出所有分解中，$m$ 最小的分解。

如果给定整数无法分解为连续正整数，则输出字符串 "N"。

## 二、 输入描述

输入数据为一整数，范围为 $(1, 2^{30}]$。

## 三、 输出描述

输出格式为：`n=a+(a+1)+...+(a+m-1)`。
例如，输入为 `21`，输出 `21=10+11`。

## 四、 用例

### 用例1

输入：
```
21
```
输出：
```
21=10+11
```
说明：
21 可以分解的连续正整数组合有：
- $21 = 1+2+3+4+5+6$ ($m=6$)
- $21 = 6+7+8$ ($m=3$)
- $21 = 10+11$ ($m=2$)
其中 $m=2$ 是最短的序列。

### 用例2

输入：
```
8
```
输出：
```
N
```
说明：
8 无法分解为连续正整数之和（它是 2 的幂）。

## 五、 备注

- $n$ 的范围较大（达到 $2^{30}$），需要使用 `long` 类型处理。
- 连续正整数序列意味着这是一个公差为 1 的等差数列。

## 六、 题目解析

### 1. 题目分析

我们需要将 $n$ 表示为 $m$ 个连续正整数之和：
$$n = a + (a+1) + (a+2) + \dots + (a+m-1)$$
其中 $a \ge 1$ 且 $m > 1$。
根据等差数列求和公式：
$$n = \frac{m(2a + m - 1)}{2}$$
变形得：
$$2n = m(2a + m - 1)$$

我们的目标是找到满足条件的**最小**的 $m$ ($m > 1$)。

### 2. 解题思路

1. **寻找 $m$ 的范围**：
   - 因为 $a \ge 1$，所以 $2a + m - 1 \ge 2(1) + m - 1 = m + 1$。
   - 即 $m(m+1) \le 2n$。
   - $m^2 < 2n \implies m < \sqrt{2n}$。
   - 给定 $n \le 2^{30}$，则 $2n \le 2^{31}$，$\sqrt{2n} \approx 46340$。这个范围很小，可以直接遍历。

2. **判定条件**：
   对于每一个可能的 $m$（从 2 开始从小到大遍历）：
   - $m$ 必须是 $2n$ 的因数，即 `(2 * n) % m == 0`。
   - 令 $target = \frac{2n}{m}$。根据公式 $target = 2a + m - 1$。
   - 解出 $a$：$2a = target - m + 1 \implies a = \frac{target - m + 1}{2}$。
   - $a$ 必须是正整数，因此：
     - `(target - m + 1)` 必须是偶数。
     - `(target - m + 1) > 0`（即 $target \ge m$）。

3. **输出结果**：
   - 找到第一个符合条件的 $m$ 即为最小的 $m$。
   - 构造字符串并输出。
   - 如果遍历完所有可能的 $m$（直到 $m(m+1) > 2n$）都没有找到，则输出 "N"。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.StringJoiner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLong()) return;
        long n = sc.nextLong();
        
        // 目标：找到最小的 m (m > 1) 使得 n 为 m 个连续正整数之和
        // 2 * n = m * (2 * a + m - 1)
        // 令 target = 2 * a + m - 1 = (2 * n) / m
        // 则 a = (target - m + 1) / 2
        
        long resM = -1;
        long resA = -1;
        
        long limit = 2 * n;
        // 遍历 m，从 2 开始。根据 m * (m + 1) <= 2 * n 确定上限
        for (long m = 2; m * (m + 1) <= limit; m++) {
            if (limit % m == 0) {
                long target = limit / m;
                // 判断 (target - m + 1) 是否为偶数且 a > 0
                if ((target - m + 1) % 2 == 0) {
                    long a = (target - m + 1) / 2;
                    if (a > 0) {
                        resM = m;
                        resA = a;
                        break; // 找到最小的 m，直接跳出
                    }
                }
            }
        }
        
        if (resM != -1) {
            StringBuilder sb = new StringBuilder();
            sb.append(n).append("=");
            StringJoiner sj = new StringJoiner("+");
            for (int i = 0; i < resM; i++) {
                sj.add(String.valueOf(resA + i));
            }
            sb.append(sj.toString());
            System.out.println(sb.toString());
        } else {
            System.out.println("N");
        }
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(\sqrt{n})$。我们遍历 $m$ 直到 $\sqrt{2n}$，对于每个 $m$ 的操作是 $O(1)$ 的（除了最后的字符串拼接，但序列长度 $m$ 也被 $\sqrt{n}$ 限制）。
- **空间复杂度**：$O(\sqrt{n})$。主要用于存储结果字符串。
