## 一、 题目描述

某软件系统会在运行过程中持续产生日志，系统每天运行 $N$ 单位时间，运行期间每单位时间产生的日志条数保存在数组 `records` 中。`records[i]` 表示第 $i$ 单位时间内产生的日志条数。

由于系统磁盘空间限制，每天可记录保存的日志总数上限为 `total` 条。

如果一天产生的日志总条数大于 `total`，则需要对当天内每单位时间产生的日志条数进行限流后保存，请计算每单位时间最大可保存日志条数 `limit`，以确保当天保存的总日志条数不超过 `total`。

规则如下：
1. 对于单位时间内产生日志条数不超过 `limit` 的日志，全部记录保存；
2. 对于单位时间内产生日志条数超过 `limit` 的日志，则只记录保存 `limit` 条日志。

如果一天产生的日志条数总和小于等于 `total`，则不需要启动限流机制，返回 `-1`。
请返回 `limit` 的最大值或者 `-1`。

## 二、输入描述

- 第一行为系统某一天运行的单位时间数 $N$，$1 \le N \le 10^5$。
- 第二行为表示这一天每单位时间产生的日志数量的数组 `records`，$0 \le records[i] \le 10^5$。
- 第三行为系统一天可以保存的总日志条数 `total`，$1 \le total \le 10^9$。

## 三、输出描述

每单位时间内最大可保存的日志条数 `limit`，如果不需要启动限流机制，返回 `-1`。

## 四、用例

### 用例1
**输入**：
```text
6
3 3 8 7 10 15
40
```
**输出**：
```text
9
```
**说明**：
总日志量为 $3+3+8+7+10+15 = 46$。$46 > 40$，需要限流。
- 如果 `limit = 9`：保存日志为 $3, 3, 8, 7, 9, 9$，总和为 $39 \le 40$。
- 如果 `limit = 10`：保存日志为 $3, 3, 8, 7, 10, 10$，总和为 $41 > 40$。
因此最大 `limit` 为 9。

### 用例2
**输入**：
```text
4
1 2 3 4
15
```
**输出**：
```text
-1
```
**说明**：
总日志量为 $1+2+3+4 = 10 \le 15$，不需要限流，输出 -1。

## 五、备注
1. $N$ 最大为 $10^5$。
2. $records[i]$ 最大为 $10^5$。
3. $total$ 最大为 $10^9$。
4. 建议使用二分查找提高搜索效率。

## 六、题目解析

### 1. 题目分析
本题要求找到一个整数 `limit`，使得经过限流后的日志总数（即 $\sum \min(records[i], limit)$）不超过 `total`，且 `limit` 尽可能大。如果原始总和已经小于等于 `total`，则直接返回 `-1`。

### 2. 解题思路
1. **判断是否需要限流**：首先计算数组 `records` 的总和。如果总和 $\le total$，直接输出 `-1`。
2. **确定搜索范围**：由于 `limit` 最小可以是 0（极端情况下 `total` 很小），最大不会超过数组中的最大值（因为超过最大值后，限流就不起作用了）。所以搜索区间为 $[0, \max(records)]$。
3. **二分查找**：
   - 设定 `left = 0`, `right = max(records)`。
   - 每次取中间值 `mid` 作为尝试的 `limit`。
   - 计算限流后的总和 `sum_mid = sum(min(record, mid))`。
   - 如果 `sum_mid <= total`，说明当前的 `mid` 是可行的，尝试更大的 `limit`（`left = mid + 1`），并记录当前 `mid`。
   - 如果 `sum_mid > total`，说明当前的 `mid` 太大了，需要减小限流值（`right = mid - 1`）。
4. **输出结果**：二分查找结束后的最大可行 `mid` 即为答案。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取N
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        
        // 读取records
        int[] records = new int[n];
        long currentTotal = 0;
        int maxRecord = 0;
        for (int i = 0; i < n; i++) {
            records[i] = sc.nextInt();
            currentTotal += records[i];
            if (records[i] > maxRecord) {
                maxRecord = records[i];
            }
        }
        
        // 读取上限total
        long totalLimit = sc.nextLong();
        
        // 如果不需要限流
        if (currentTotal <= totalLimit) {
            System.out.println("-1");
            return;
        }
        
        // 二分查找 limit
        int left = 0;
        int right = maxRecord;
        int ans = 0;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (check(records, mid, totalLimit)) {
                ans = mid; // 记录当前可行的最大limit
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        System.out.println(ans);
    }
    
    /**
     * 校验在指定 limit 下，总日志条数是否不超过 totalLimit
     */
    private static boolean check(int[] records, int limit, long totalLimit) {
        long sum = 0;
        for (int record : records) {
            sum += Math.min(record, limit);
            // 提前退出防止溢出（虽然long够用，但这也是好习惯）
            if (sum > totalLimit) return false;
        }
        return sum <= totalLimit;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N \log M)$，其中 $N$ 是单位时间数（数组长度），$M$ 是 `records` 中的最大值（本题最大为 $10^5$）。二分查找次数为 $\log M$，每次 check 函数需要遍历数组。
- **空间复杂度**：$O(N)$，用于存储 `records` 数组。
