## 一、 题目描述

部门在进行需求开发时需要进行人力安排。当前部门需要完成 $N$ 个需求，需求用 `requirements[i]` 表示，`requirements[i]` 表示第 $i$ 个需求的工作量大小，单位：人月。这部分需求需要在 $M$ 个月内完成开发，进行人力安排后每个月的人力是固定的。

目前要求每个月最多有 2 个需求开发，并且每个月需要完成的需求不能超过部门人力。请帮部门评估在满足需求开发进度的情况下，每个月需要的最小人力是多少？

## 二、输入描述

输入第一行为 $M$，第二行为 `requirements`，长度为 $N$。

*   $M$ 表示需要开发时间要求。
*   `requirements` 表示每个需求工作量大小。
*   $N$ 为 `requirements` 长度。

**数据范围：**
*   $1 \le N / 2 \le M \le N \le 10000$
*   $1 \le requirements[i] \le 10^9$

## 三、输出描述

对于每一组测试数据，输出部门需要人力需求，行末无多余的空格。

## 四、用例

### 用例1
**输入**
```
3
3 5 3 4
```
**输出**
```
6
```
**说明**
当选择人力为 6 时，2 个需求量为 3 的工作可以在 1 个月里完成，其他 2 个工作各需要 1 个月完成。可以在 3 个月内完成所有需求。
当选择人力为 5 时，4 个工作各需要 1 个月完成，一共需要 4 个月才能完成所有需求。
因此 6 是部门最小的人力需求。

### 用例2
**输入**
```
4
1 2 3 4 5
```
**输出**
```
5
```
**说明**
人力为 5 时，(1,4) 一个月，(2,3) 一个月，(5) 一个月，共需 3 个月，满足 $\le 4$。如果人力更小则无法完成。

## 五、备注
题目要求每个月最多只能处理 2 个需求。

## 六、题目解析

### 1. 题目分析
本题是一个典型的“最小化最大值”问题。我们需要在满足约束（每月最多2个需求，总时长不超过 $M$）的前提下，寻找一个最小的月人力值。

关键约束：
1. 每个月最多完成 **2 个需求**。
2. 每个月的工作量总和不能超过 **月人力限制**。
3. 总月数不能超过 **$M$**。

### 2. 解题思路
由于人力需求越大，完成任务所需的时间就越短（具备单调性），因此可以使用 **二分查找 (Binary Search)** 来确定最小人力。

1.  **二分边界确定**：
    *   `left`（最小值）：由于每个月至少要能独立完成最大的那个需求，所以 `left` 初始化为 `requirements` 中的最大值。
    *   `right`（最大值）：最极端的情况下，可能需要把两个最大的需求放在一个月（虽然 $M \ge N/2$ 保证了通常不需要这么大），我们可以设置为所有需求中最大的两个之和，或者更简单地设为 $2 \times 10^9$。

2.  **`check(limit)` 函数逻辑**：
    *   判断在月人力为 `limit` 的情况下，是否能在 $M$ 个月内完成。
    *   为了使每个月的人力利用率最高，应采用 **贪心算法 + 双指针**：
        *   先对 `requirements` 进行升序排序。
        *   使用左指针 `L` 指向最小需求，右指针 `R` 指向最大需求。
        *   如果 `req[L] + req[R] <= limit`：说明这两个需求可以在一个月内完成。`L++`, `R--`，月份计数 `count++`。
        *   如果 `req[L] + req[R] > limit`：说明 `req[R]` 必须单独占用一个月。`R--`，月份计数 `count++`。
        *   最后判断 `count` 是否 $\le M$。

### 3. 代码实现

#### Java8实现

```java
import java.util.Arrays;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取 M
        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();
        
        // 读取 requirements
        // 注意：由于题目没有给出N的具体输入方式，通常是通过 sc.next() 读取一行或者循环读取
        // 这里的处理方式是读取剩下的所有整数
        sc.nextLine(); // 消耗掉换行符
        String line = sc.nextLine();
        String[] parts = line.split(" ");
        long[] reqs = new long[parts.length];
        long maxVal = 0;
        long sumVal = 0;
        for (int i = 0; i < parts.length; i++) {
            reqs[i] = Long.parseLong(parts[i]);
            maxVal = Math.max(maxVal, reqs[i]);
            sumVal += reqs[i];
        }

        // 排序，为双指针贪心做准备
        Arrays.sort(reqs);

        // 二分查找最小人力
        long left = maxVal; // 至少要能装下最大的那个需求
        long right = maxVal * 2; // 两个最大需求之和肯定够了
        long ans = right;

        while (left <= right) {
            long mid = left + (right - left) / 2;
            if (check(reqs, m, mid)) {
                ans = mid;
                right = mid - 1; // 尝试更小的人力
            } else {
                left = mid + 1; // 人力不足，需要增加
            }
        }

        System.out.println(ans);
    }

    /**
     * 检查在 capacity 限制下，能否在 m 个月内完成
     */
    private static boolean check(long[] reqs, int m, long capacity) {
        int count = 0;
        int l = 0;
        int r = reqs.length - 1;

        while (l <= r) {
            if (l == r) {
                // 只剩一个需求
                count++;
                break;
            }
            if (reqs[l] + reqs[r] <= capacity) {
                // 两个需求可以拼在一个月
                l++;
                r--;
            } else {
                // 最大的需求只能单独放一个月
                r--;
            }
            count++;
        }
        return count <= m;
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：
    *   排序：$O(N \log N)$。
    *   二分查找：二分次数为 $\log(\text{max\_sum})$。每次 `check` 函数使用双指针，复杂度为 $O(N)$。
    *   总复杂度为 $O(N \log N + N \log(\text{max\_sum}))$。由于 $N=10000$，此复杂度完全可以接受。
*   **空间复杂度**：$O(1)$ 或 $O(N)$（取决于存储输入的数组空间）。
