## 二、 题目描述

如果一个数组中出现次数最多的元素出现大于等于 K 次，被称为 **K-优雅数组**，K 也可以被称为优雅阈值。

例如，数组 `[1, 2, 3, 1, 2, 3, 1]`，它是一个 3-优雅数组，因为元素 1 出现次数为 3 次（大于等于 3 次）；数组 `[1, 2, 3, 1, 2]` 就不是一个 3-优雅数组，因为其中出现次数最多的元素是 1 和 2，都只出现了 2 次。

给定一个长度为 $n$ 的数组 $A$ 和优雅阈值 $k$，请求出 $A$ 有多少个子数组是 K-优雅子数组。

**子数组** 是数组中一个或多个连续元素组成的数组。
例如，数组 `[1, 2, 3]` 包含 6 个子数组，分别是：`[1], [1, 2], [1, 2, 3], [2], [2, 3], [3]`。

## 二、输入描述

- 第一行输入两个数字，以空格隔开，含义是：A 数组长度 $n$，$k$ 值。
- 第二行输入 A 数组元素，以空格隔开。

**数据范围：**
- $1 \le n \le 10^5$
- $1 \le k \le n$
- 数组元素为正整数。

## 三、输出描述

- 输出一个整数，表示 A 有多少个子数组是 K-优雅子数组。

## 四、用例

### 用例1

**输入：**
```text
7 3
1 2 3 1 2 3 1
```

**输出：**
```text
1
```

**说明：**
只有子数组 `[1, 2, 3, 1, 2, 3, 1]` 中 1 出现了 3 次，符合条件。其他子数组中没有任何元素出现次数 $\ge 3$。

### 用例2

**输入：**
```text
5 2
1 2 3 1 2
```

**输出：**
```text
3
```

**说明：**
1-优雅子数组（频率 $\ge 2$）有：
`[1, 2, 3, 1]`（1出现2次）
`[1, 2, 3, 1, 2]`（1出现2次，2出现2次）
`[2, 3, 1, 2]`（2出现2次）
`[1, 2, 3, 1, 2]` 的另一个子区间（前面已统计，注意是连续子数组计数）。
符合条件的子数组有：`A[0..3], A[0..4], A[1..4]`。
*注：逻辑上需要覆盖所有起始点 $i$ 和终点 $j$。*

## 五、备注

- 本题关键在于如何高效统计满足条件的连续子数组数量。
- 暴力解法的时间复杂度为 $O(n^2)$，在 $n=10^5$ 的量级下会超时，需要使用滑动窗口。

## 六、题目解析

### 1. 题目分析

- **核心定义**：子数组中至少有一个元素出现的频率 $f \ge k$。
- **单调性性质**：如果子数组 $A[i \dots j]$ 是 K-优雅数组，那么任何包含该区间的更大子数组 $A[i-x \dots j+y]$ 也一定是 K-优雅数组。

### 2. 解题思路

我们可以使用**滑动窗口（双指针）**来解决：
1. 维护两个指针 `left` 和 `right`，表示当前的窗口 $[left, right]$。
2. 移动右指针 `right`，并使用一个哈希表（或数组，如果范围已知）记录窗口内每个元素出现的次数。
3. 每当新加入 `A[right]` 导致某个元素的计数达到 $k$ 时：
   - 此时，所有以当前 `right` 为结尾，且左端点在 `0` 到 `left` 之间的子数组**不一定**都满足条件。
   - 实际上，如果窗口 $[left, right]$ 满足条件，那么对于固定的 `right`，所有左端点 $i \le left$ 的子数组 $A[i \dots right]$ 都满足条件。
   - 我们收缩左指针 `left`，直到窗口不再满足 K-优雅条件（即刚才达到 $k$ 的那个元素频率减小到 $k-1$）。
   - 在收缩过程中，每移动一次 `left`，就代表找到了一个新的“临界点”。

**更简单的统计方式：**
对于每一个固定的 `right`，如果存在一个最大的 `left` 使得子数组 $A[left \dots right]$ 满足条件，那么以 `right` 为结尾的 K-优雅子数组共有 `left + 1` 个（左起点可以选 $0, 1, \dots, left$）。

### 3. 代码实现

#### Java8实现

```java
import java.util.HashMap;
import java.util.Map;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNext()) return;
        
        int n = sc.nextInt();
        int k = sc.nextInt();
        int[] nums = new int[n];
        for (int i = 0; i < n; i++) {
            nums[i] = sc.nextInt();
        }

        long count = 0;
        int left = 0;
        // 使用Map记录窗口内元素出现的次数
        Map<Integer, Integer> map = new HashMap<>();

        for (int right = 0; right < n; right++) {
            int rNum = nums[right];
            map.put(rNum, map.getOrDefault(rNum, 0) + 1);

            // 如果当前加入的元素使得窗口满足了K-优雅条件
            // 我们尝试移动left，直到窗口刚刚好不满足条件为止
            while (map.get(rNum) >= k) {
                // 此时，以right为右端点，左端点在 0 到 left 之间的所有子数组都满足条件
                // 但为了不重复计算，我们需要理解：
                // 当窗口[left, right]满足条件时，后续所有包含这个[left, right]的右边界都会满足
                // 这里的逻辑是：对于当前的left，它能贡献 (n - right) 个子数组
                // 因为 [left, right], [left, right+1], ..., [left, n-1] 全都满足
                count += (n - right);
                
                // 缩小窗口
                int lNum = nums[left];
                map.put(lNum, map.get(lNum) - 1);
                left++;
            }
        }

        System.out.println(count);
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(n)$。`right` 指针遍历一次数组，`left` 指针最多也遍历一次数组，哈希表的插入和查找平均为 $O(1)$。
- **空间复杂度**：$O(n)$。最坏情况下哈希表需要存储数组中的所有不同元素。
