## 一、 题目描述

给出 $n$ 个牌数，每个牌的数值在 -100 到 100 之间，求按照特定规则翻牌后的最大得分。

**规则如下：**
1. 连续翻牌。
2. 如果选择翻当前牌，则当前总得分 = 上一次翻牌后的总得分 + 当前牌的数字。
3. 如果当前总得分（翻牌后）**小于**它前三次的总得分，则此次**不翻牌**，并且当前总得分等于它前三次的得分。
4. 前 1 到 3 次翻牌时，如果计算出的总得分小于 0，则总得分取 0。

## 二、输入描述

一组数字，以空格分隔（或者逗号，视具体题目格式而定，通常为一行或 $n$ 个数）。
例如：`1 -5 -6 4 7 2 -2`

## 三、输出描述

输出最终的总得分。

## 四、用例

### 用例1
**输入：**
`1 -5 -6 4 7 2 -2`

**输出：**
`11`

**说明：**
1. 第一张牌 1：$0 + 1 = 1$，大于 0，翻牌，总分 1。
2. 第二张牌 -5：$1 + (-5) = -4$，小于 0，不翻牌，总分 0。
3. 第三张牌 -6：$0 + (-6) = -6$，小于 0，不翻牌，总分 0。
4. 第四张牌 4：$0 + 4 = 4$，大于前三次（第一张牌后）的总分 1，翻牌，总分 4。
5. 第五张牌 7：$4 + 7 = 11$，大于前三次（第二张牌后）的总分 0，翻牌，总分 11。
6. 第六张牌 2：$11 + 2 = 13$，大于前三次（第三张牌后）的总分 0，翻牌，总分 13。
7. 第七张牌 -2：$13 + (-2) = 11$，大于前三次（第四张牌后）的总分 4，翻牌，总分 11。

### 用例2
**输入：**
`10 -20 5 10`

**输出：**
`15`

## 五、备注
1. $n$ 的范围通常在 1 到 1000 之间。
2. 分数对比的对象是“第 $i-3$ 次操作后”的状态。

## 六、题目解析

### 1. 题目分析
这是一个典型的线性动态规划问题。我们需要维护一个数组 `dp`，其中 `dp[i]` 表示处理完第 $i$ 张牌后的总得分。

### 2. 解题思路
- **状态定义**：`dp[i]` 为处理第 $i$ 个数字后的总分。
- **转移方程**：
    - 对于 $i < 3$（前三次）：
        - 尝试翻牌：`temp = (i == 0 ? 0 : dp[i-1]) + cards[i]`
        - 如果 `temp < 0`，则 `dp[i] = 0`，否则 `dp[i] = temp`。
    - 对于 $i \ge 3$：
        - 尝试翻牌：`temp = dp[i-1] + cards[i]`
        - 如果 `temp < dp[i-3]`，说明翻这张牌不如回到前三次的状态划算，根据规则：`dp[i] = dp[i-3]`。
        - 否则，`dp[i] = temp`。
- **初始值**：可以虚拟 `dp[-1], dp[-2], dp[-3]` 为 0，方便统一计算。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 处理输入，题目可能以空格或逗号分隔
        if (!sc.hasNextLine()) return;
        String line = sc.nextLine();
        String[] parts = line.split("[,\\s]+");
        int n = parts.length;
        int[] cards = new int[n];
        for (int i = 0; i < n; i++) {
            cards[i] = Integer.parseInt(parts[i]);
        }

        // dp[i] 表示处理完第 i 张牌后的总得分
        int[] dp = new int[n];

        for (int i = 0; i < n; i++) {
            if (i == 0) {
                // 第一张牌
                dp[i] = Math.max(0, cards[i]);
            } else if (i < 3) {
                // 第二、三张牌：对比 0
                int currentTry = dp[i - 1] + cards[i];
                dp[i] = Math.max(0, currentTry);
            } else {
                // 第四张牌及以后：对比 dp[i-3]
                int currentTry = dp[i - 1] + cards[i];
                if (currentTry < dp[i - 3]) {
                    dp[i] = dp[i - 3];
                } else {
                    dp[i] = currentTry;
                }
            }
        }

        // 输出最后一次操作后的总得分
        System.out.println(dp[n - 1]);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(n)$，只需遍历一次数组。
- **空间复杂度**：$O(n)$，用于存储每个位置的 `dp` 值。如果为了节省空间，可以使用滚动数组的思想将空间优化到 $O(1)$，因为当前状态只依赖于前一个状态和前三个状态。
