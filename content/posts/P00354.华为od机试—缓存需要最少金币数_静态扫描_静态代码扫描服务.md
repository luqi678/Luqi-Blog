## 一、 题目描述

静态扫描可以快速识别源代码的缺陷，静态扫描的结果以扫描报告作为输出：

1. 文件扫描的成本和文件大小相关，如果文件大小为 $N$，则扫描成本为 $N$ 个金币。
2. 扫描报告的缓存成本和文件大小无关，每缓存一个报告需要 $M$ 个金币。
3. 扫描报告缓存后，后续再碰到该文件则不需要扫描成本，直接获取缓存结果。

给出源代码文件标识序列和文件大小序列，求解采用合理的缓存策略，最少需要的金币数。

## 二、输入描述

- 第一行为缓存一个报告金币数 $M$，$1 \le M \le 100$。
- 第二行为文件标识序列：$F_1, F_2, F_3, \dots, F_n$。
- 第三行为文件大小序列：$S_1, S_2, S_3, \dots, S_n$。

**备注：**
- $1 \le N \le 10000$ (文件序列长度)
- $1 \le F_i \le 1000$ (文件标识)
- $1 \le S_i \le 10$ (文件大小)

## 三、输出描述

采用合理的缓存策略，需要的最少金币数。

## 四、用例

### 用例1

**输入：**
```text
5
1 2 2 1 2 3 4
1 1 1 1 1 1 1
```

**输出：**
```text
7
```

**说明：**
文件大小相同，扫描成本均为 1 个金币。缓存任意文件（成本 1+5=6）均不如重复扫描（最多扫描 3 次，成本 3），因此最少成本为 $1+1+1+1+1+1+1 = 7$ 金币。

### 用例2

**输入：**
```text
5
2 2 2 2 2 5 2 2 2
3 3 3 3 3 1 3 3 3
```

**输出：**
```text
9
```

**说明：**
- 文件 2 出现了 8 次，大小为 3。如果不缓存，成本为 $8 \times 3 = 24$；如果缓存，成本为 $3 + 5 = 8$。显然缓存更划算。
- 文件 5 出现了 1 次，大小为 1。如果不缓存，成本为 1；如果缓存，成本为 $1 + 5 = 6$。不缓存更划算。
- 总成本为 $8 + 1 = 9$。

## 五、备注

- 相同文件标识的文件，其大小是一致的。
- 缓存策略是针对每个独立的文件 ID 决定的。

## 六、题目解析

### 1. 题目分析

本题的核心在于对每一个独立的文件 $F_i$ 做决策：
- **方案 A：不缓存。** 每次遇到该文件都支付其大小 $S_i$ 的扫描费用。如果该文件出现了 $K$ 次，总成本为 $K \times S_i$。
- **方案 B：使用缓存。** 第一次遇到该文件时支付扫描费用 $S_i$，并支付缓存费用 $M$。此后无论出现多少次，都不再产生费用。总成本为 $S_i + M$。

### 2. 解题思路

1. **统计信息：** 使用一个 Map（哈希表）来存储每个文件 ID 出现的次数，以及该文件的大小（由于题目保证相同 ID 大小相同，只需记录一次）。
2. **局部最优决策：** 遍历 Map 中的每一个文件 ID，计算方案 A 和 方案 B 的成本，取其最小值：$\min(K \times S_i, S_i + M)$。
3. **累加求和：** 将所有文件 ID 的最小成本累加，即为最终结果。

### 3. 代码实现

#### Java8实现

```java
import java.util.HashMap;
import java.util.Map;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 1. 读取缓存成本 M
        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();
        sc.nextLine(); // 消耗换行符

        // 2. 读取文件标识序列
        String[] fileIdsStr = sc.nextLine().trim().split("\\s+");
        // 3. 读取文件大小序列
        String[] fileSizesStr = sc.nextLine().trim().split("\\s+");

        int n = fileIdsStr.length;
        int[] fileIds = new int[n];
        int[] fileSizes = new int[n];

        for (int i = 0; i < n; i++) {
            fileIds[i] = Integer.parseInt(fileIdsStr[i]);
            fileSizes[i] = Integer.parseInt(fileSizesStr[i]);
        }

        // 4. 统计每个文件的出现次数和大小
        // key: 文件ID, value: [出现次数, 文件大小]
        Map<Integer, int[]> stats = new HashMap<>();
        for (int i = 0; i < n; i++) {
            int id = fileIds[i];
            int size = fileSizes[i];
            if (!stats.containsKey(id)) {
                stats.put(id, new int[]{1, size});
            } else {
                stats.get(id)[0]++;
            }
        }

        // 5. 计算最小总金币数
        long totalCost = 0;
        for (int[] val : stats.values()) {
            int count = val[0];
            int size = val[1];
            
            // 不缓存的总成本
            long noCacheCost = (long) count * size;
            // 缓存的总成本 (第一次扫描 + 缓存费用)
            long cacheCost = (long) size + m;
            
            totalCost += Math.min(noCacheCost, cacheCost);
        }

        System.out.println(totalCost);
    }
}
```

### 4. 复杂度分析

- **时间复杂度：** $O(N)$。我们需要遍历一遍文件序列来统计次数，然后再遍历一遍哈希表（最多 $N$ 个元素）来计算总成本。
- **空间复杂度：** $O(U)$，其中 $U$ 是不同文件 ID 的数量。最坏情况下所有文件 ID 都不同，空间复杂度为 $O(N)$。
