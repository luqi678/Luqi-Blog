## 二、 题目描述

某网上商场举办优惠活动，发布了满减、打折、无门槛3种优惠券，分别为：
1. **满减券**：每满 100 元优惠 10 元，无使用数限制。如 100~199 元可以使用 1 张减 10 元，200~299 可使用 2 张减 20 元，以此类推；
2. **打折券**：92 折券，1 次限使用 1 张。如 100 元，则优惠后为 92 元；
3. **无门槛券**：5 元优惠券，无使用数限制，直接减 5 元。

**优惠券使用限制：**
* 每次最多使用 **2 种** 优惠券，2 种优惠可以叠加（优惠叠加时以优惠后的价格计算）。
* 同一类型优惠券使用多张时必须一次性使用，不能分多次拆开使用。
* 以最优惠价格为准。如果价格相同，使用的优惠券数量越少越好。
* 可以允许某次购物不使用优惠券。
* 优惠后价格如果是小数，则**向下取整**。

请设计实现一种解决方法，帮助购物者以最少的优惠券获得最优的优惠价格。

## 二、 输入描述

1. 第一行：三个整数，代表每个人拥有的优惠券数量，顺序为：**满减券数量、打折券数量、无门槛券数量**（数量取值范围为 `[0, 10]`）。
2. 第二行：一个整数 $n$，表示购物的人数 ($1 \le n \le 10000$)。
3. 接下来 $n$ 行：每一行表示某个人优惠前的购物总价格（价格范围 `(0, 1000]`，都为整数）。

## 三、 输出描述

1. 每行输出每个人每次购物优惠后的最低价格以及使用的优惠券总数量。
2. 每行的输出顺序和输入的顺序保持一致。

## 四、 用例

### 用例 1
**输入**
```text
3 2 5
3
100
200
400
```
**输出**
```text
65 6
155 7
338 4
```

### 用例 2
**输入**
```text
10 10 10
1

```
**输出**
```text

```
*(注：99元无法满减，使用无门槛10张减50元，得49元，用券10张)*

## 五、 备注
1. 优惠券数量、人数、单价均符合描述中的范围。
2. 结果向下取整。

## 六、 题目解析

### 1. 题目分析
题目要求在三种优惠券（满减 A、打折 B、无门槛 C）中，**最多选择两种类型**进行组合，求出“价格最低”且“用券最少”的方案。

*   **满减券 (A)**：每满100减10。使用的张数 $k = \min(\text{持有数}, \text{当前价格}/100)$。
*   **打折券 (B)**：92折。只能用1张（即使持有2张也只能用1张）。
*   **无门槛 (C)**：每张减5。使用的张数 $k = \min(\text{持有数}, \text{当前价格}/5)$，但由于题目说无门槛直接减，且券数有限，通常直接把手中的 C 券全用掉（直到价格为0）。

### 2. 解题思路
由于只允许使用最多两种类型的券，我们可以穷举所有可能的使用顺序组合：
1.  仅使用 A
2.  仅使用 B
3.  仅使用 C
4.  先 A 后 B
5.  先 B 后 A
6.  先 A 后 C
7.  先 C 后 A
8.  先 B 后 C
9.  先 C 后 B
10. 不使用券

对每种组合计算最终价格和总用券数。对于每位顾客，比较这些组合，选出最优解。

**更新最优解的逻辑：**
*   如果 `当前方案价格 < 历史最优价格`：更新最优价格和最优券数。
*   如果 `当前方案价格 == 历史最优价格`：取 `券数更少` 的方案。

### 3. 代码实现

#### Java 实现

```java
import java.util.Scanner;

public class Main {
    static int maxA, maxB, maxC;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        maxA = sc.nextInt();
        maxB = sc.nextInt();
        maxC = sc.nextInt();

        int n = sc.nextInt();
        for (int i = 0; i < n; i++) {
            int price = sc.nextInt();
            solve(price);
        }
    }

    private static void solve(int price) {
        // [finalPrice, totalCount]
        int[] res = {price, 0};

        // 定义可能的组合
        // 0: A, 1: B, 2: C, 3: AB, 4: BA, 5: AC, 6: CA, 7: BC, 8: CB
        update(res, applyA(price));
        update(res, applyB(price));
        update(res, applyC(price));
        
        update(res, applyB(applyA(price)));
        update(res, applyA(applyB(price)));
        
        update(res, applyC(applyA(price)));
        update(res, applyA(applyC(price)));
        
        update(res, applyC(applyB(price)));
        update(res, applyB(applyC(price)));

        System.out.println(res[0] + " " + res[1]);
    }

    private static void update(int[] currentBest, int[] nextOption) {
        if (nextOption[0] < currentBest[0]) {
            currentBest[0] = nextOption[0];
            currentBest[1] = nextOption[1];
        } else if (nextOption[0] == currentBest[0]) {
            currentBest[1] = Math.min(currentBest[1], nextOption[1]);
        }
    }

    // 满减：每满100减10
    private static int[] applyA(int p) {
        int count = Math.min(maxA, p / 100);
        return new int[]{p - count * 10, count};
    }

    private static int[] applyA(int[] state) {
        int p = state[0];
        int count = Math.min(maxA, p / 100);
        return new int[]{p - count * 10, state[1] + count};
    }

    // 打折：92折，限1张
    private static int[] applyB(int p) {
        if (maxB >= 1) {
            return new int[]{(int) (p * 0.92), 1};
        }
        return new int[]{p, 0};
    }

    private static int[] applyB(int[] state) {
        if (maxB >= 1) {
            return new int[]{(int) (state[0] * 0.92), state[1] + 1};
        }
        return state;
    }

    // 无门槛：每张减5
    private static int[] applyC(int p) {
        int count = Math.min(maxC, (int) Math.ceil(p / 5.0)); 
        // 实际上可以直接用完，或者用到0为止
        int actualCount = Math.min(maxC, p / 5);
        // 根据题目示例和规则，通常直接扣减，若价格不足则扣到0
        int used = maxC;
        int nextP = Math.max(0, p - used * 5);
        return new int[]{nextP, used};
    }

    private static int[] applyC(int[] state) {
        int used = maxC;
        int nextP = Math.max(0, state[0] - used * 5);
        return new int[]{nextP, state[1] + used};
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：对于每个购物者，我们需要计算固定数量（9种）的组合方案。因此时间复杂度为 $O(n)$，其中 $n$ 是人数。
*   **空间复杂度**：$O(1)$，仅使用了常数空间存储中间变量。

## 七、 总结
本题属于典型的**业务逻辑模拟题**。核心难点在于理清优惠券叠加的顺序性（AB 还是 BA 结果不同）以及正确处理题目中的各种限制条件（如向下取整、最多两种券、同价取券少者）。通过显式枚举所有合法的排列组合，可以确保不遗漏最优解。
