## 一、 题目描述

给定一个无向图，包含 $M$ 个节点和 $N$ 条边。现在需要给图中的节点进行染色，可以填入红色或黑色两种颜色。
染色的规则是：**必须保证相邻的两个节点不能同时被染成红色**。
请输出一共有多少种不同的染色方案。

注意：
1. 不要求所有节点都是连通的。
2. 节点的编号范围通常在 $1$ 到 $M$ 之间。

## 二、输入描述

第一行输入两个整数：$M$ 和 $N$。
- $M$ 表示图中节点数（$1 \le M \le 15$）。
- $N$ 表示边的数量（$0 \le N \le M \times 3$）。

接下来的 $N$ 行，每行包含两个整数 $V1$ 和 $V2$，表示节点 $V1$ 到 $V2$ 之间有一条无向边。

## 三、输出描述

输出一个整数，表示满足条件的染色方案总数。

## 四、用例

### 用例1

**输入：**
```text
4 4
1 2
2 4
3 4
1 3
```

**输出：**
```text
7
```

**说明：**
4个节点，4条边形成一个环：1-2, 2-4, 4-3, 3-1。
合法的染色方案如下（R代表红色，B代表黑色）：
1. BBBB
2. RBBB
3. BRBB
4. BBRB
5. BBBR
6. RBBR (1号和4号不相邻)
7. BRRB (2号和3号不相邻)
总计 7 种。

### 用例2

**输入：**
```text
3 2
1 2
2 3
```

**输出：**
```text
5
```

**说明：**
1-2-3 的链式结构。合法方案：BBB, RBB, BRB, BBR, RBR。

## 五、备注

数据范围 $M \le 15$，意味着总的状态空间为 $2^{15} = 32768$。这是一个非常小的数据规模，可以使用深度优先搜索（DFS）或者二进制枚举（状态压缩）来解决。

## 六、题目解析

### 1. 题目分析
本题属于典型的图论染色问题。约束条件非常明确：**相邻节点不能同时为红色**。
黑色节点没有约束，即：
- 黑色节点旁边可以是黑色，也可以是红色。
- 红色节点旁边必须全部是黑色。

### 2. 解题思路
由于节点数 $M$ 最大只有 15，我们可以采用 **回溯法 (DFS)**：
1. **邻接矩阵/邻接表存储**：首先将输入的边信息存储到邻接矩阵或邻接表中，方便判断节点间的连接关系。
2. **递归搜索**：从第 1 个节点开始尝试染色。
3. **颜色尝试**：
   - 尝试染成 **黑色**：黑色节点不违反任何约束，直接递归处理下一个节点。
   - 尝试染成 **红色**：在染红色之前，需要检查该节点的所有邻居是否已经染成了红色。如果没有任何邻居是红色，则可以将当前节点染成红色，并递归处理下一个节点。
4. **计数**：当处理完所有 $M$ 个节点时，说明找到了一种可行方案，计数器加 1。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static int M, N;
    static boolean[][] adj; // 邻接矩阵存储边关系
    static int[] colors;    // 0: 黑色, 1: 红色
    static int count = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        M = sc.nextInt();
        N = sc.nextInt();

        adj = new boolean[M + 1][M + 1];
        colors = new int[M + 1];

        for (int i = 0; i < N; i++) {
            int u = sc.nextInt();
            int v = sc.nextInt();
            // 考虑节点编号可能从1开始
            if (u <= M && v <= M) {
                adj[u][v] = true;
                adj[v][u] = true;
            }
        }

        // 从第1个节点开始深度优先搜索
        dfs(1);

        System.out.println(count);
    }

    /**
     * @param u 当前处理的节点编号
     */
    private static void dfs(int u) {
        // 如果所有节点都处理完了，方案数+1
        if (u > M) {
            count++;
            return;
        }

        // 场景1：将当前节点 u 染成 黑色 (0)
        // 黑色节点无限制，直接进入下一层
        colors[u] = 0;
        dfs(u + 1);

        // 场景2：将当前节点 u 染成 红色 (1)
        // 需要检查邻居是否已经是红色
        boolean canBeRed = true;
        for (int v = 1; v < u; v++) { // 只需要检查已经染过色的编号较小的邻居
            if (adj[u][v] && colors[v] == 1) {
                canBeRed = false;
                break;
            }
        }

        if (canBeRed) {
            colors[u] = 1;
            dfs(u + 1);
            // 回溯还原（本题中非必须，因为每次进入都会覆盖colors[u]）
            colors[u] = 0;
        }
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(2^M)$。每个节点有两种染色选择，总共有 $2^M$ 种状态。对于 $M=15$，最大计算量约为 $3.2 \times 10^4$，完全符合机考时间要求。
- **空间复杂度**：$O(M^2)$ 用于存储邻接矩阵，或者 $O(M)$ 用于递归栈。在本题范围内，空间占用极小。
