## 一、 题目描述

手里有一副手牌，数字范围从 0-9，颜色有 `r` (红色), `g` (绿色), `b` (蓝色), `y` (黄色) 四种。

出牌规则如下：
1. 每次打出的牌必须跟上一张发出的牌的**数字相同**或者**颜色相同**。
2. 如果不满足上述条件，则不能继续出牌。
3. 每一张牌只能使用一次。

请问如何选择出牌顺序，才能使得出牌的数量最大？输出这个最大出牌数量。

## 二、输入描述

第一行：牌的数值序列，用空格分隔。
第二行：牌的颜色序列（r, g, b, y），用空格分隔。

两行输入的牌数量相同。

## 三、输出描述

输出一个整数，表示最大出牌数量。

## 四、用例

### 用例1

**输入**
```text
1 4 3 4 5
r y b b r
```

**输出**
```text
3
```

**说明**
手牌情况：`(1, r), (4, y), (3, b), (4, b), (5, r)`
- 路径1：`(1, r) -> (5, r)`，长度为 2。
- 路径2：`(4, y) -> (4, b) -> (3, b)`，长度为 3。
最大长度为 3。

### 用例2

**输入**
```text
1 2 3 4
r g b y
```

**输出**
```text
1
```

**说明**
所有牌的数字和颜色都不相同，无论从哪张开始，都只能打出一张。

## 五、备注

1. 牌的数量 $N$ 通常在 10-20 之间（根据华为机试常规难度估算）。
2. 数字范围 $0 \le n \le 9$。
3. 颜色范围 $\{r, g, b, y\}$。

## 六、题目解析

### 1. 题目分析
本题要求在给定的手牌集合中，按照“数字相同或颜色相同”的规则寻找最长的序列。这是一个典型的**图中求最长路径**的问题。
- 每一个手牌可以看作图中的一个节点。
- 如果两个节点满足“数字相同”或“颜色相同”，则它们之间存在一条边。
- 我们需要找到一条路径，使得路径上的节点不重复，且节点数最多。

### 2. 解题思路
由于每张牌只能用一次，且需要尝试所有可能的出牌顺序以求最大值，我们可以使用 **深度优先搜索（DFS）+ 回溯** 的方法：

1. **数据预处理**：将输入的数字和颜色组合成对象（或结构体）。
2. **枚举起点**：因为不知道从哪张牌开始能得到最长序列，所以需要遍历每一张牌作为出牌序列的第一张。
3. **DFS 过程**：
   - 记录当前已经使用的牌（使用 `visited` 数组）。
   - 遍历剩余未使用的牌，检查是否与当前最后一张牌的颜色或数字相同。
   - 如果满足条件，则标记为已使用，进入下一层递归。
   - 递归返回后，进行**回溯**（将标记重置为未使用），尝试其他可能性。
4. **记录最大值**：在递归过程中不断更新全局最大出牌数量。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static class Card {
        int num;
        String color;

        Card(int num, String color) {
            this.num = num;
            this.color = color;
        }
    }

    static int maxCount = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        if (!sc.hasNextLine()) return;
        String[] numStr = sc.nextLine().split(" ");
        if (!sc.hasNextLine()) return;
        String[] colorStr = sc.nextLine().split(" ");

        int n = numStr.length;
        Card[] cards = new Card[n];
        for (int i = 0; i < n; i++) {
            cards[i] = new Card(Integer.parseInt(numStr[i]), colorStr[i]);
        }

        boolean[] visited = new boolean[n];

        // 尝试从每一张牌作为起点
        for (int i = 0; i < n; i++) {
            visited[i] = true;
            dfs(cards, visited, i, 1);
            visited[i] = false;
        }

        System.out.println(maxCount);
    }

    /**
     * DFS 寻找最长路径
     * @param cards 手牌数组
     * @param visited 访问标记
     * @param lastIdx 上一张出牌的索引
     * @param currentCount 当前出牌数量
     */
    private static void dfs(Card[] cards, boolean[] visited, int lastIdx, int currentCount) {
        maxCount = Math.max(maxCount, currentCount);

        for (int i = 0; i < cards.length; i++) {
            if (!visited[i]) {
                // 检查是否满足颜色相同或数字相同
                if (cards[i].num == cards[lastIdx].num || cards[i].color.equals(cards[lastIdx].color)) {
                    visited[i] = true;
                    dfs(cards, visited, i, currentCount + 1);
                    // 回溯
                    visited[i] = false;
                }
            }
        }
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(N \cdot N!)$。在最坏情况下（所有牌都可以互相连接），这是一个全排列问题。但由于实际情况中颜色和数字的限制，搜索树会被大量剪枝。对于 $N \le 15$ 左右的数据量，DFS 能够快速得出结果。
- **空间复杂度**：$O(N)$。主要用于存储 `visited` 数组和递归调用栈。
