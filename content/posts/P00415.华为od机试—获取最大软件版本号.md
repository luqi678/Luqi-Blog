## 二、 题目描述

Maven 版本号定义：`<主版本>.<次版本>.<增量版本>-<里程碑版本>`。

举例：`3.1.4-beta`。
其中：
- **主版本** 和 **次版本** 是必须的。
- 主版本、次版本、增量版本由多位数字组成，可能包含前导零。
- 里程碑版本由字符串组成（可选）。

比较规则：
1. **基于数字比较**：主版本、次版本、增量版本按数字大小比较（忽略前导零）。
2. **基于字符串比较**：里程碑版本采用字典序比较。
3. **比较顺序**：从左到右，依次比较主版本、次版本、增量版本、里程碑版本。
4. 如果某版本号没有增量版本或里程碑版本，则在比较时相应位置视为更小（或在数字比较中视为0，具体看实现逻辑，但通常 Maven 规则中缺失项较小）。

输入两个版本号，输出其中较大的版本号。

## 二、输入描述

输入两个版本号，按行分割。每个版本号的长度小于 50。

## 三、输出描述

输出较大的版本号。如果两个版本号完全相同，输出第一个。

## 四、用例

### 用例1
**输入：**
```text
2.5.1-C
1.4.2-D
```
**输出：**
```text
2.5.1-C
```

### 用例2
**输入：**
```text
1.05.1
1.5.01
```
**输出：**
```text
1.05.1
```
*(注：主版本1=1，次版本05=5，增量版本1=01，完全相等输出第一个)*

### 用例3
**输入：**
```text
1.5
1.5.0
```
**输出：**
```text
1.5.0
```

## 五、备注
- 注意处理前导零。
- 注意可能缺失增量版本或里程碑版本的情况。

## 六、题目解析

### 1. 题目分析
本题的核心是实现一个自定义的版本号比较器。我们需要将版本号字符串拆分为四个部分：主版本（Major）、次版本（Minor）、增量版本（Patch）、里程碑版本（Milestone）。

### 2. 解题思路
1. **结构拆分**：
   - 首先通过连字符 `-` 将版本号拆分为“数字部分”和“里程碑部分”。
   - 接着通过点号 `.` 将“数字部分”拆分为主版本、次版本和增量版本。
2. **数值转换**：
   - 对于前三个数字部分，将字符串转换为整数（Integer）进行比较，这样可以自动忽略前导零。
3. **分段比较**：
   - 第一步：比较主版本。
   - 第二步：比较次版本。
   - 第三步：比较增量版本（若缺失则视为0）。
   - 第四步：比较里程碑版本。
     - 如果两个都有里程碑，按字典序比较。
     - 如果一个有一一个没有，通常有里程碑的被认为是在开发中，逻辑上可能更小，但根据题目描述“按从左到右顺序依次比较”，若数字部分全等，有里程碑的字符串存在则比没有的长，需明确题目中对“缺失”的定义。按常规机试逻辑：存在即大于不存在。
4. **输出结果**：根据比较结果输出原字符串。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) return;
        String v1 = sc.nextLine();
        if (!sc.hasNextLine()) return;
        String v2 = sc.nextLine();

        System.out.println(compareVersion(v1, v2) >= 0 ? v1 : v2);
    }

    public static int compareVersion(String v1, String v2) {
        // 拆分数字部分和里程碑部分
        String[] part1 = v1.split("-");
        String[] part2 = v2.split("-");

        // 处理数字部分
        String[] nums1 = part1[0].split("\\.");
        String[] nums2 = part2[0].split("\\.");

        // 比较 主、次、增量 版本
        int maxLen = Math.max(nums1.length, nums2.length);
        for (int i = 0; i < maxLen; i++) {
            long n1 = i < nums1.length ? Long.parseLong(nums1[i]) : 0;
            long n2 = i < nums2.length ? Long.parseLong(nums2[i]) : 0;
            if (n1 > n2) return 1;
            if (n1 < n2) return -1;
        }

        // 数字部分完全一致，比较里程碑部分
        String m1 = part1.length > 1 ? part1[1] : "";
        String m2 = part2.length > 1 ? part2[1] : "";

        return m1.compareTo(m2);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：O(N)，N 为版本号字符串的长度，主要是字符串分割和转换操作。
- **空间复杂度**：O(N)，存储分割后的字符串数组。
