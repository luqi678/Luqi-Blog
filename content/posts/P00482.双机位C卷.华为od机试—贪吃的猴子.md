## 二、题目描述

一只贪吃的猴子，来到一个果园，发现许多串香蕉排成一行，每串香蕉上有若干根香蕉。每串香蕉的根数由数组 `numbers` 给出。猴子获取香蕉，每次都只能从行的开头或者末尾获取，并且只能获取 $N$ 次，求猴子最多能获取多少根香蕉。

## 三、输入描述

1. 第一行为数组 `numbers` 的长度 $L$。
2. 第二行为数组 `numbers` 的值，每个数字通过空格分开。
3. 第三行输入为 $N$，表示获取的次数。

**数据范围：**
- $1 \le L \le 10^5$
- $1 \le N \le L$
- 每串香蕉根数 $1 \le numbers[i] \le 1000$

## 四、输出描述

按照题目要求能获取的最大数值。

## 五、用例

### 用例1
**输入**
```text
7
1 2 2 7 3 6 1
3
```
**输出**
```text
10
```

### 用例2
**输入**
```text
3
1 2 3
3
```
**输出**
```text
6
```
**说明**：全部获取所有的香蕉，因此最终根数为 1+2+3 = 6。

### 用例3
**输入**
```text
4
4 2 2 3
2
```
**输出**
```text
7
```
**说明**：第一次获取香蕉为行的开头（4），第二次获取为行的末尾（3），因此最终根数为 4+3 = 7。

## 六、题目解析

### 1. 题目分析
本题要求从一个数组的两端（头部或尾部）进行 $N$ 次取值，使得取出的数值之和最大。这是一个典型的窗口选择问题。

### 2. 解题思路
直接模拟从两端取 $N$ 个数的情况比较多（比如左边取 0 个右边取 $N$ 个，左边取 1 个右边取 $N-1$ 个...），我们可以转换思路：
- 既然要让取出的 $N$ 个数之和最大，那么**剩下的 $L - N$ 个连续的数之和必须最小**。
- 剩余的部分必须是**连续**的，因为我们只能从两端取数，取走的数分布在两端，剩下的数必然在中间且是连续的一段。
- 因此，问题转化为：在一个长度为 $L$ 的数组中，找到一个长度为 $M = L - N$ 的**连续子数组**，使其和最小。
- 最终结果 = 数组总和 - 长度为 $(L-N)$ 的连续子数组的最小和。

**特殊情况：**
- 如果 $N = L$，则猴子取走了所有香蕉，最大值即为数组总和。

### 3. 代码实现

#### Java实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 1. 读取数组长度
        if (!sc.hasNextInt()) return;
        int L = sc.nextInt();

        // 2. 读取数组内容并计算总和
        int[] numbers = new int[L];
        long totalSum = 0;
        for (int i = 0; i < L; i++) {
            numbers[i] = sc.nextInt();
            totalSum += numbers[i];
        }

        // 3. 读取获取次数 N
        int N = sc.nextInt();

        // 如果获取次数等于数组长度，直接输出总和
        if (N == L) {
            System.out.println(totalSum);
            return;
        }

        // 4. 计算剩余子数组的长度 M
        int M = L - N;

        // 5. 使用滑动窗口寻找长度为 M 的子数组的最小和
        long currentWindowSum = 0;
        // 初始化第一个窗口的和
        for (int i = 0; i < M; i++) {
            currentWindowSum += numbers[i];
        }

        long minWindowSum = currentWindowSum;

        // 滑动窗口向右移动
        for (int i = M; i < L; i++) {
            // 加上新进入窗口的元素，减去离开窗口的元素
            currentWindowSum += numbers[i] - numbers[i - M];
            minWindowSum = Math.min(minWindowSum, currentWindowSum);
        }

        // 6. 最大获取值 = 总和 - 最小剩余和
        System.out.println(totalSum - minWindowSum);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(L)$。我们需要遍历一遍数组计算总和，再遍历一遍数组进行滑动窗口计算，其中 $L$ 是数组的长度。
- **空间复杂度**：$O(L)$。用于存储输入的数组。如果考虑空间优化，可以在读取数据的同时处理前缀和，或者仅保留窗口数据，但题目中给出的 $L$ 范围是 $10^5$，存储数组是完全可接受的。
