## 二、 题目描述

一个跳数 `jump`，及幸存数量 `left`。给一个正整数数列 `nums`，运算过程为：
1. 从索引 0 的位置开始向后跳，中间跳过了 `jump` 个数字，命中索引为 `jump + 1` 的数字，该数被敲出。
2. 从被敲出数字的下一个点起跳（即当前位置），以此类推，直到幸存 `left` 个数为止。
3. 最后返回幸存数之和。

**约束：**
1. 0 是第一个起跳点。
2. 起跳点和命中点之间间隔 `jump` 个数字，已被敲出的数字不计入在内。
3. 跳到末尾时无缝从头开始（循环查找），并可以多次循环。
4. 若起始时 `left >= len(nums)`，则无需进行跳数处理过程。

**方法设计：**
```java
/**
 * @param nums 正整数数列
 * @param jump 跳数
 * @param left 幸存数量
 * @return 幸存数之和
 */
int sumOfLeft(int[] nums, int jump, int left)
```

## 二、 输入描述

1. 第一行输入正整数数列（以逗号分隔）。
2. 第二行输入跳数 `jump`。
3. 第三行输入幸存数量 `left`。

## 三、 输出描述

输出幸存数之和。

## 四、 用例

### 用例1
**输入**
```
1,2,3,4,5,6,7,8,9
4
3
```
**输出**
```
13
```
**说明**
- 初始列表：`[1, 2, 3, 4, 5, 6, 7, 8, 9]`
- 从 1 (索引 0) 开始起跳，跳过 4 个数（2, 3, 4, 5），命中 6。删除 6。
- 剩余列表：`[1, 2, 3, 4, 5, 7, 8, 9]`，从原 6 的位置（现 7 所在位置）起跳。
- 依次删除：6, 2, 8, 5, 4, 7。
- 剩余：1, 3, 9。
- 和为：1 + 3 + 9 = 13。

### 用例2
**输入**
```
1,2,3
1
4
```
**输出**
```
6
```
**说明**
幸存数量 `left` 大于等于数列长度，无需删除，直接求和 1+2+3=6。

## 五、 备注
- `nums` 长度范围 [1, 10000]
- `jump` 范围 [1, 10000]
- `left` 范围 [0, 10000]

## 六、 题目解析

### 1. 题目分析
本题是一个经典的**约瑟夫环（Josephus Problem）**变体。核心在于模拟删除过程：
- 关键点在于计算每一次删除的索引位置。
- **第一次跳跃**：从索引 0 开始，跳过 `jump` 个数，命中的索引是 `(0 + jump + 1) % currentSize`。
- **后续跳跃**：从被删除的位置开始，跳过 `jump` 个数（包括起始点那个位置），命中的索引是 `(currentIndex + jump) % currentSize`。

### 2. 解题思路
1. **边界处理**：如果 `left >= nums.length`，直接计算原数组之和。
2. **数据结构**：使用 `ArrayList` 存储数列，因为它支持按索引删除，且在数据量为 10000 时，删除操作的耗时在可接受范围内。
3. **循环逻辑**：
   - 初始化 `index = 0`。
   - 第一次执行：`index = (index + jump + 1) % list.size()`。
   - 删除该元素。
   - 之后只要 `list.size() > left`，重复：`index = (index + jump) % list.size()`，删除元素。
4. **求和**：遍历最终的列表求和。

### 3. 代码实现

#### Java 实现

```java
import java.util.*;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 处理第一行输入：正整数数列
        if (!sc.hasNextLine()) return;
        String line = sc.nextLine();
        int[] nums = Arrays.stream(line.split(","))
                           .mapToInt(Integer::parseInt)
                           .toArray();
        
        // 处理第二行输入：跳数
        int jump = sc.nextInt();
        
        // 处理第三行输入：幸存数量
        int left = sc.nextInt();
        
        System.out.println(sumOfLeft(nums, jump, left));
    }

    public static int sumOfLeft(int[] nums, int jump, int left) {
        int n = nums.length;
        
        // 若幸存数量大于等于总数，直接返回总和
        if (left >= n) {
            int sum = 0;
            for (int num : nums) sum += num;
            return sum;
        }
        
        // 使用 ArrayList 模拟，虽然删除是 O(N)，但 10000 数据规模下可以通过
        List<Integer> list = new ArrayList<>();
        for (int num : nums) {
            list.add(num);
        }
        
        int currentIndex = 0;
        
        // 第一次跳跃：从索引0开始跳过jump个数，命中索引为 jump + 1
        currentIndex = (currentIndex + jump + 1) % list.size();
        list.remove(currentIndex);
        
        // 后续跳跃：直到只剩下 left 个数
        // 注意：删除一个元素后，currentIndex 已经指向了下一个元素的位置
        // 所以后续跳跃只需要再增加 jump 即可实现“跳过 jump 个数字”
        while (list.size() > left) {
            currentIndex = (currentIndex + jump) % list.size();
            list.remove(currentIndex);
        }
        
        // 求和
        int sum = 0;
        for (int val : list) {
            sum += val;
        }
        return sum;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N^2)$。在最坏情况下，我们需要删除 $N - left$ 个元素，每次删除操作在 `ArrayList` 中平均耗时 $O(N)$。对于 $N=10000$，$N^2 = 10^8$，在 Java 的 `System.arraycopy`（`ArrayList.remove` 的底层实现）优化下，通常可以在机考限时内通过。
- **空间复杂度**：$O(N)$。需要一个额外的列表来存储当前的幸存数。
