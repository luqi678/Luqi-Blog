## 一、 题目描述

信号传播过程中会出现一些误码，不同的数字表示不同的误码ID，取值范围为1~65535。用一个数组记录误码出现的情况，每个误码出现的次数代表误码频度。

请找出记录中包含**频度最高**误码的**最小子数组**长度。

注意：
- 频度最高的误码可能不止一个。
- 子数组必须包含该误码出现的所有位置。例如，如果误码 `2` 出现了 3 次，分别在索引 1, 2, 4，那么包含误码 `2` 的子数组必须至少覆盖索引 [1, 4]，长度为 4。

## 二、输入描述

1. 第一行：误码总数目 $N$，取值范围为 0~255。（注：根据题目背景描述，数组长度实际可能达到 1000，请以实际输入为准）。
2. 第二行：误码出现频率数组，误码ID范围为 1~65535，数组长度为 1~1000。

## 三、输出描述

包含频率最高的误码最小子数组长度。

## 四、用例

### 用例1
**输入**
```text
5
1 2 2 4 1
```
**输出**
```text
2
```
**说明**
- 误码 1 出现 2 次，索引为 [0, 4]，长度为 $4 - 0 + 1 = 5$。
- 误码 2 出现 2 次，索引为 [1, 2]，长度为 $2 - 1 + 1 = 2$。
- 误码 4 出现 1 次。
- 频度最高为 2，对应的误码有 1 和 2。其中最短长度为 2。

### 用例2
**输入**
```text
7
1 2 2 4 2 1 1
```
**输出**
```text
4
```
**说明**
- 误码 1 出现 3 次，出现位置 [0, 5, 6]，覆盖长度 $6 - 0 + 1 = 7$。
- 误码 2 出现 3 次，出现位置 [1, 2, 4]，覆盖长度 $4 - 1 + 1 = 4$。
- 频度最高为 3，对应的误码有 1 和 2。最短长度为 4。

## 五、备注
- 数组长度较小（最大1000），误码ID范围较大（65535），适合使用哈希表。

## 六、题目解析

### 1. 题目分析
本题的核心是寻找“众数”以及该“众数”在原数组中分布的跨度。
- **频度最高**：即出现次数最多的数字。
- **最小子数组长度**：对于某个误码 ID，要包含它“所有”出现的记录，最短的长度就是它最后一次出现的索引减去第一次出现的索引再加 1。

### 2. 解题思路
1. **统计信息**：使用一个哈希表（Map）来记录每个误码 ID 出现的：
   - 次数 (`count`)
   - 第一次出现的索引 (`firstIndex`)
   - 最后一次出现的索引 (`lastIndex`)
2. **计算最高频度**：在遍历数组的过程中，实时更新并维护全局最高频度 `maxCount`。
3. **计算结果**：遍历哈希表，找出所有 `count` 等于 `maxCount` 的 ID，计算它们的 `lastIndex - firstIndex + 1`，取其中的最小值。

### 3. 代码实现

#### Java8实现

```java
import java.util.HashMap;
import java.util.Map;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        if (n == 0) {
            System.out.println(0);
            return;
        }

        // key: 误码ID, value: [出现次数, 第一次出现索引, 最后一次出现索引]
        Map<Integer, int[]> map = new HashMap<>();
        int maxFreq = 0;

        for (int i = 0; i < n; i++) {
            int errId = sc.nextInt();
            if (!map.containsKey(errId)) {
                // 初始化：次数1，首索引i，尾索引i
                map.put(errId, new int[]{1, i, i});
            } else {
                int[] info = map.get(errId);
                info[0]++; // 频度+1
                info[2] = i; // 更新最后一次出现的索引
            }
            // 更新最大频度
            maxFreq = Math.max(maxFreq, map.get(errId)[0]);
        }

        int minLen = Integer.MAX_VALUE;

        // 遍历Map，寻找频度等于maxFreq的ID中，跨度最小的
        for (int[] info : map.values()) {
            if (info[0] == maxFreq) {
                int currentLen = info[2] - info[1] + 1;
                minLen = Math.min(minLen, currentLen);
            }
        }

        System.out.println(minLen);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N)$。只需遍历一次数组存入哈希表，再遍历一次哈希表（最多 $N$ 个元素）寻找结果。
- **空间复杂度**：$O(K)$，其中 $K$ 是数组中不同误码 ID 的个数，最坏情况下为 $O(N)$。
