## 一、 题目描述

一线运维人员在对通话流量进行监控，每一段时间内都会出现流量的高峰，流量有高有低形成一个个波峰波谷。运维人员想找到流量变化最快的波峰，你可以帮助他吗？

给定一个整数数组 `nums`，代表采样点的流量值，请找到满足以下条件的三元组 $(i, j, k)$：

- $i < j < k$
- $nums[i] < nums[j]$ 且 $nums[j] > nums[k]$ （即 $j$ 是峰顶）

并在所有满足条件的三元组中，找到 $(k - i)$ 的**最小值**。

## 二、输入描述

第一行为 $n$ 个整数，表示数组中的 $n$ 个元素。
（注：$0 \le n \le 100,000$）

## 三、输出描述

返回所有满足条件的三元组中 $(k - i)$ 的最小值。若不存在满足条件的三元组，返回 `-1`。

## 四、用例

### 用例1

**输入：**
```text
3 5 4 7 2 1
```

**输出：**
```text
2
```

**说明：**
满足条件的三元组有：
- `(0, 1, 2)`，对应值 `[3, 5, 4]`，$k - i = 2 - 0 = 2$。
- `(2, 3, 4)`，对应值 `[4, 7, 2]`，$k - i = 4 - 2 = 2$。
最小值为 2。

### 用例2

**输入：**
```text
4 3 2 1
```

**输出：**
```text
-1
```

**说明：**
数组单调递减，无法找到满足 $nums[i] < nums[j] > nums[k]$ 的三元组，返回 -1。

## 五、备注

- 数组长度 $n$ 可能达到 $100,000$，因此算法复杂度应控制在 $O(n)$ 或 $O(n \log n)$。

## 六、题目解析

### 1. 题目分析

本题要求在一个序列中寻找一个“山峰”结构：左侧有一个比峰顶小的值，右侧也有一个比峰顶小的值。我们的目标是让这个结构的跨度 $(k-i)$ 最小。

对于每一个下标 $j$ 作为潜在的峰顶：
- 我们需要在其左侧 $(0 \dots j-1)$ 找到一个下标 $i$，使得 $nums[i] < nums[j]$。为了让 $k-i$ 最小，我们希望 $i$ 尽可能**靠近** $j$（即 $i$ 越大越好）。
- 我们需要在其右侧 $(j+1 \dots n-1)$ 找到一个下标 $k$，使得 $nums[k] < nums[j]$。为了让 $k-i$ 最小，我们希望 $k$ 尽可能**靠近** $j$（即 $k$ 越小越好）。

### 2. 解题思路

寻找“最近的比当前值小”的元素，是典型的**单调栈**应用场景。

1.  **预处理左侧最近更小值**：
    遍历数组，利用单调栈维护一个寻找左侧最近的 $nums[i] < nums[j]$ 的索引数组 `left[]`。
    - 如果 $nums[j-1] < nums[j]$，那么 $j-1$ 就是我们要找的 $i$。
    - 如果 $nums[j-1] \ge nums[j]$，我们需要继续往左找。单调栈可以帮我们在 $O(1)$ 平均时间内找到这个位置。

2.  **预处理右侧最近更小值**：
    同理，从右向左遍历，维护一个寻找右侧最近的 $nums[k] < nums[j]$ 的索引数组 `right[]`。

3.  **计算最小值**：
    遍历每一个索引 $j$（从 $1$ 到 $n-2$），如果 `left[j]` 和 `right[j]` 都存在，计算 `right[j] - left[j]`，取所有结果中的最小值。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
        // 处理输入，因为没有给出n，使用Scanner读取整行并分割
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) {
            System.out.println("-1");
            return;
        }
        String line = sc.nextLine().trim();
        if (line.isEmpty()) {
            System.out.println("-1");
            return;
        }
        String[] parts = line.split("\\s+");
        int n = parts.length;
        int[] nums = new int[n];
        for (int i = 0; i < n; i++) {
            nums[i] = Integer.parseInt(parts[i]);
        }

        if (n < 3) {
            System.out.println("-1");
            return;
        }

        // left[j] 存储 j 左侧最近的 i，使得 nums[i] < nums[j]
        int[] left = new int[n];
        // right[j] 存储 j 右侧最近的 k，使得 nums[k] < nums[j]
        int[] right = new int[n];
        Arrays.fill(left, -1);
        Arrays.fill(right, -1);

        // 使用单调栈寻找左侧最近的小于当前值的索引
        Deque<Integer> stack = new ArrayDeque<>();
        for (int j = 0; j < n; j++) {
            while (!stack.isEmpty() && nums[stack.peek()] >= nums[j]) {
                stack.pop();
            }
            if (!stack.isEmpty()) {
                left[j] = stack.peek();
            }
            stack.push(j);
        }

        // 清空栈，寻找右侧最近的小于当前值的索引
        stack.clear();
        for (int j = n - 1; j >= 0; j--) {
            while (!stack.isEmpty() && nums[stack.peek()] >= nums[j]) {
                stack.pop();
            }
            if (!stack.isEmpty()) {
                right[j] = stack.peek();
            }
            stack.push(j);
        }

        int minDistance = Integer.MAX_VALUE;
        boolean found = false;

        // 遍历每一个可能的峰顶 j
        for (int j = 1; j < n - 1; j++) {
            if (left[j] != -1 && right[j] != -1) {
                minDistance = Math.min(minDistance, right[j] - left[j]);
                found = true;
            }
        }

        if (found) {
            System.out.println(minDistance);
        } else {
            System.out.println("-1");
        }
    }
}
```

### 4. 复杂度分析

-   **时间复杂度**：$O(n)$。我们对数组进行了三次线性扫描（一次读入，一次从左往右单调栈，一次从右往左单调栈），单调栈操作中每个元素最多进栈一次、出栈一次。
-   **空间复杂度**：$O(n)$。使用了 `left`、`right` 数组及栈来存储索引。
