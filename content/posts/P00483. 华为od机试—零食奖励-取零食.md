## 一、 题目描述

小朋友考试得第一名就可以得到零食奖励。

现在有 `N` 种零食，每种零食有：
-   价格 `Price`
-   数量 `Quantity` (限购数量)
-   喜爱度 `Favorability`

小朋友手中有 `x` 元钱。请返回选取 `x` 元零食可以达到的**最大喜爱度**。

## 二、输入描述

第一行输入为 `x` 和 `N`，`x` 为可使用的钱的总额，`N` 为零食种类数。

**约束：**
-   $0 \le x \le 1000$
-   $0 \le N \le 100$

第二行开始为零食属性，共 `N` 行，每行有三个整型数值，分别代表零食的：
1.  价格：$(0, 100)$
2.  数量：$[0, 10000]$
3.  喜爱度：$[0, 10000]$

## 三、输出描述

输出一个整数，表示在预算范围内能获得的最大喜爱度。

## 四、用例

| 输入                                                                                 | 输出   | 说明                                                                                              |
| :--------------------------------------------------------------------------------- | :--- | :---------------------------------------------------------------------------------------------- |
| `10 2`<br>`2 4 3`<br>`3 3 4`                                                       | `14` | **零食1**：价2，存4，喜3。<br>**零食2**：价3，存3，喜4。<br>方案：买2个零食1（花费4，喜6）+ 买2个零食2（花费6，喜8）。<br>总花费 10，总喜爱度 14。 |
| `6 7`<br>`3 1 8`<br>`4 1 2`<br>`3 1 1`<br>`9 1 7`<br>`4 1 1`<br>`5 1 8`<br>`4 1 4` | `9`  | 6元钱最佳策略是购买第一个零食（3元，喜8）和第三个零食（3元，喜1）。<br>总花费6元，总喜爱度 9。                                           |

## 五、备注

-   **问题类型**：这是一个标准的**多重背包问题 (Bounded Knapsack Problem)**。
-   **特点**：每种物品有有限的数量限制，既不是只能选1个（0/1背包），也不是可以无限选（完全背包）。
-   **数据规模**：
    -   钱（背包容量） $x \le 1000$
    -   物品种类 $N \le 100$
    -   单品数量可达 $10000$
    -   直接暴力拆解（把10000个物品看作10000个独立物品）会导致 $N \times Q \times x \approx 10^9$ 的复杂度，可能会超时。
    -   需要使用**二进制拆分**优化，或者基于单调队列优化。考虑到 $x$ 较小，二进制拆分是最稳妥且易实现的方案。

## 六、题目解析

### 核心思路

1.  **问题转化**：
    题目要求在有限容量（金额）下获得最大价值（喜爱度），且每种物品有数量限制。这是多重背包问题。

2.  **二进制拆分优化**：
    对于第 $i$ 种零食，数量为 $C_i$，价格 $P_i$，喜爱度 $V_i$。
    我们不直接一个个取，而是将 $C_i$ 拆分成若干组，每组看作一个新的“0/1背包”物品。
    拆分系数为：$1, 2, 4, 8, \dots, 2^k, \text{remainder}$。
    
    *例如*：某零食数量为 13。
    拆分为：
    -   1个 (价值 $1V$，重 $1P$)
    -   2个 (价值 $2V$，重 $2P$)
    -   4个 (价值 $4V$，重 $4P$)
    -   剩余 6个 (13 - 1 - 2 - 4 = 6) -> 价值 $6V$，重 $6P$。
    
    这样，0~13 之间的任意数量都可以由这 4 组物品组合而成。我们将物品总数从 $N \times C$ 降低到了 $N \times \log C$。

3.  **DP 状态定义**：
    `dp[j]` 表示恰好或最多花费 `j` 元所能获得的最大喜爱度。

4.  **状态转移**：
    将拆分后的所有新物品（设为 List）进行标准的 0/1 背包遍历。
    对于每个新物品（价格 `w`，喜爱度 `v`）：
    `dp[j] = Math.max(dp[j], dp[j - w] + v)`，其中 `j` 从 `x` 逆序遍历到 `w`。

### 算法实现 (Java)

```java
import java.util.ArrayList;
import java.util.List;
import java.util.Scanner;

public class Main {
    // 定义一个内部类来存储拆分后的物品
    static class Item {
        int price;
        int favor;

        public Item(int price, int favor) {
            this.price = price;
            this.favor = favor;
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取总金额 x 和 零食种类 N
        if (!sc.hasNextInt()) return;
        int x = sc.nextInt();
        int n = sc.nextInt();
        
        // 存储二进制拆分后的所有物品
        List<Item> items = new ArrayList<>();
        
        for (int i = 0; i < n; i++) {
            int price = sc.nextInt();
            int quantity = sc.nextInt();
            int favor = sc.nextInt();
            
            // 如果价格为0，相当于白送，直接加上喜爱度即可（或者在背包中作为重量0处理）
            // 题目约束价格(0,100)，通常意味着 price > 0。
            // 严谨起见，进行二进制拆分
            if (quantity > 0) {
                int k = 1;
                while (quantity >= k) {
                    items.add(new Item(k * price, k * favor));
                    quantity -= k;
                    k *= 2;
                }
                // 处理剩余的数量
                if (quantity > 0) {
                    items.add(new Item(quantity * price, quantity * favor));
                }
            }
        }
        
        // 动态规划求解 0/1 背包
        // dp[j] 表示容量为 j 时的最大价值
        int[] dp = new int[x + 1];
        
        for (Item item : items) {
            // 0/1 背包必须逆序遍历，防止同一物品被重复使用
            for (int j = x; j >= item.price; j--) {
                dp[j] = Math.max(dp[j], dp[j - item.price] + item.favor);
            }
        }
        
        System.out.println(dp[x]);
    }
}
```

### 复杂度分析
-   **时间复杂度**：$O(x \cdot N \cdot \log(\text{Quantity}))$。
    -   $N=100$，Quantity=10000，$\log(10000) \approx 14$。
    -   物品总数约为 $100 \times 14 = 1400$。
    -   背包容量 $x=1000$。
    -   总运算次数 $\approx 1.4 \times 10^6$，远小于 $10^8$ 的限制，非常安全。
-   **空间复杂度**：$O(x)$，用于存储 DP 数组。
