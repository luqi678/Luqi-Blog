## 二、 题目描述

有一套系统需升级，为减小系统升级期间的影响，需根据系统过去一段时间内的每小时平均访问数据，来预测最佳升级时间窗。

现给长度为 168（7 * 24）的整数数组，表示一个周期（假设从周一 00:00 到周日 24:00）的每小时历史数据，最佳升级时间窗选择规则如下：

1. 时间窗内**累计用户访问量**必须**小于或等于**给定的容忍值（注：原题干描述可能有歧义，根据示例及实际业务逻辑，通常指低峰期，即访问量之和不超过容忍值）。
2. 时间窗必须是连续的 $x$ 个小时，最大的 $x$ 即为最佳升级时间窗，且 $x$ 不超过 7 * 24。
3. 时间窗允许跨周期，例如当前周期的第 167 小时到下一周期的第 166 小时，是一个长度为 168 的时间窗。

请计算最佳升级时间窗，并返回其开始时间和结束时间的数组下标。如果存在多个最佳升级时间窗（长度 $x$ 相同），返回开始时间下标最小的一个。

## 三、 输入描述

第一行为整数 $n$，表示给定的升级影响的容忍值，取值范围：$[0, 2^{31}-1]$。
第二行为 168 个整数，表示一个周期（7 * 24）的每个小时用户访问量，每个值的范围：$[0, 2^{31}-1]$。

## 四、 输出描述

两个整数，分别表示所计算出的最佳升级时间窗的开始时间下标（包含）和结束时间下标（包含），不存在满足条件的时间窗时返回 `-1 -1`。

### 用例 1

**输入**


**输出**

```text

```

**说明**
在该序列中，下标 22 到 25 的数据为 `2 1 1 2`，其和为 $2+1+1+2=6$，满足 $\leq 6$ 的条件。经计算这是长度最大的且开始下标最小的窗口。

## 六、 题目解析

### 1. 题目分析

*   **目标**：在 168 小时的环形数据中，找到一个连续的子数组，其元素之和 $\leq$ 容忍值 $n$，且该子数组的长度 $x$ 最大。
*   **约束条件**：
    1.  累计访问量 $\leq n$。
    2.  长度 $x \leq 168$。
    3.  环形处理：可以从周日跨越到周一。
    4.  优先级：最大长度 > 最小起始下标。
*   **核心难点**：
    1.  **环形处理**：通常可以通过将原数组复制一份接在后面（变为 336 长度）来简化环形窗口的计算。
    2.  **数据范围**：容忍值和数组元素可达 $2^{31}-1$，求和时会溢出 `int`，必须使用 `long`。

### 2. 解题思路

由于数组长度固定为 168，这是一个非常小的常数级规模。我们可以直接采用枚举起点或滑动窗口的方法：

1.  **枚举起点法 ($O(N^2)$)**：
    *   遍历每一个可能的起点 $i \in [0, 167]$。
    *   对于每个起点 $i$，向后累加元素，直到累加和超过容忍值 $n$ 或长度达到 168。
    *   记录过程中发现的最大长度 `maxLen` 及其对应的 `start` 和 `end`。
    *   由于我们是从 $i=0$ 开始遍历，当 `len > maxLen` 时才更新，自然满足“起始下标最小”的规则。

2.  **滑动窗口法 ($O(N)$)**：
    *   将数组扩展为 2 倍长度。
    *   使用左右指针 `left` 和 `right`。
    *   当窗口和 $\leq n$ 时，尝试扩大 `right`，并更新 `maxLen`。
    *   当窗口和 $> n$ 或窗口长度 $> 168$ 时，缩小 `left`。
    *   注意：在处理“最小起始下标”时需要额外判断。

鉴于 $N=168$ 极小，$O(N^2)$ 逻辑更简单且不容易出错。

### 3. 代码实现

#### Java 实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 1. 读取容忍值
        if (!sc.hasNextLong()) return;
        long tolerance = sc.nextLong();

        // 2. 读取168小时的数据
        int N = 168;
        long[] data = new long[N];
        for (int i = 0; i < N; i++) {
            data[i] = sc.nextLong();
        }

        int maxLen = 0;
        int bestStart = -1;
        int bestEnd = -1;

        // 3. 枚举每一个可能的起始位置
        for (int i = 0; i < N; i++) {
            long currentSum = 0;
            // 尝试从 i 开始延伸，长度最多为 168
            for (int len = 1; len <= N; len++) {
                // 利用取模处理环形逻辑
                int currentIndex = (i + len - 1) % N;
                currentSum += data[currentIndex];

                // 如果当前和在容忍范围内
                if (currentSum <= tolerance) {
                    // 如果当前长度比之前记录的长，更新结果
                    // 因为 i 是从小到大遍历，len > maxLen 保证了长度相同时取最小 i
                    if (len > maxLen) {
                        maxLen = len;
                        bestStart = i;
                        bestEnd = currentIndex;
                    }
                } else {
                    // 一旦超过容忍值，该起点 i 的后续长度都不再考虑
                    break;
                }
            }
        }

        // 4. 输出结果
        if (bestStart == -1) {
            System.out.println("-1 -1");
        } else {
            System.out.println(bestStart + " " + bestEnd);
        }
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(N^2)$，其中 $N=168$。总计算量约为 $168 \times 168 = 28,224$，远小于机考 $10^7$ 的限制，性能极佳。
*   **空间复杂度**：$O(N)$，用于存储 168 个小时的流量数据。

## 七、 备注

*   注意题目中隐含的条件：如果一个数字本身就大于容忍值，该位置无法作为任何时间窗的一部分。
*   一定要使用 `long` 类型，否则在处理 $2^{31}$ 级别的数据累加时会产生负数。
