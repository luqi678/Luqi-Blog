## 一、 题目描述

评估一个网络的信号质量，其中一个做法是将网络划分为栅格，然后对每个栅格的信号质量计算。路测的时候，希望选择一条信号最好的路线（彼此相连的栅格集合）进行演示。

现给出 $R$ 行 $C$ 列的整数数组 `Cov`。每个单元格的数值 $S$ 即为该栅格的信号质量（已归一化，无单位，值越大信号越好）。

要求从 `[0, 0]` 到 `[R - 1, C - 1]` 设计一条最优路测路线。返回该路线得分。

**规则：**
1. 路测路线可以 **上下左右** 四个方向移动，不能走对角线。
2. 路线的评分是以 **路线上信号最差的栅格** 为准的。例如路径 `8 -> 4 -> 5 -> 9` 的值为 `4`。
3. **线路最优** 表示该条线路的评分最高。

## 二、输入描述

1. 第一行表示栅格的行数 $R$；
2. 第二行表示栅格的列数 $C$；
3. 第三行开始，每一行表示栅格地图一行的信号值，每个单元格的数值为 $S$。

**约束：**
- $1 < R, C < 20$
- $1 < S < 65535$

## 三、输出描述

输出最优路线的得分（即所有可能路径中，最小信号值的最大可能值）。

## 四、用例

### 用例1

**输入：**
```
3
3
5 4 5
1 2 6
7 4 6
```

**输出：**
```
4
```

**说明：**
其中一条路线为：`5 -> 4 -> 5 -> 6 -> 6`。该路线上最小值为 4。没有比 4 更大的最小值的路径了。

### 用例2

**输入：**
```
6
5
3 4 6 3 4
0 2 1 1 7
8 8 3 2 7
3 2 4 9 8
4 1 2 0 0
4 6 5 4 3
```

**输出：**
```
3
```

## 五、备注

本题的核心在于“最大化最小值”，这是经典的二分查找应用场景。

## 六、题目解析

### 1. 题目分析

本题要求在 $R \times C$ 的网格中，寻找一条从左上角 $(0,0)$ 到右下角 $(R-1, C-1)$ 的路径。每条路径的得分由该路径上信号强度**最小**的那个格子决定。我们的目标是让这个**最小值尽可能大**。

这是一个典型的 **最大化最小值 (Maximize the Minimum Value)** 问题。

### 2. 解题思路

这类问题通常有几种解法：
1. **二分查找 + BFS/DFS**：
   - 信号强度的范围是固定的（根据题目 $1 < S < 65535$）。
   - 我们可以二分枚举可能的得分 `mid`。
   - 对于每一个 `mid`，检查是否存在一条路径，使得路径上所有格子的信号值都 $\ge mid$。
   - 检查过程可以使用 BFS（广度优先搜索）或 DFS。
2. **改造后的 Dijkstra 算法**：
   - 使用优先队列（最大堆），每次选取当前路线上最小信号值最大的点进行扩展。
3. **并查集 (Union-Find)**：
   - 将格点按信号强度从大到小排序。
   - 逐步将格点加入图中，并与相邻已加入的格点合并。
   - 当 $(0,0)$ 和 $(R-1, C-1)$ 连通时，当前格点的信号值即为答案。

由于本题数据范围非常小（$R, C < 20$），上述任何一种方法都能轻松通过。考虑到代码实现的简洁性，**二分查找 + BFS** 是最标准的选择。

### 3. 代码实现

#### Java实现

```java
import java.util.*;

public class Main {
    static int R, C;
    static int[][] grid;
    static int[][] dirs = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        R = sc.nextInt();
        if (!sc.hasNextInt()) return;
        C = sc.nextInt();
        
        grid = new int[R][C];
        int minS = 65535, maxS = 0;
        
        for (int i = 0; i < R; i++) {
            for (int j = 0; j < C; j++) {
                grid[i][j] = sc.nextInt();
                minS = Math.min(minS, grid[i][j]);
                maxS = Math.max(maxS, grid[i][j]);
            }
        }

        // 二分查找可能的得分范围 [minS, maxS]
        // 实际上信号质量范围在起点和终点的最小值之间
        int left = 1;
        int right = Math.min(grid[0][0], grid[R - 1][C - 1]);
        int ans = right;

        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (check(mid)) {
                ans = mid; // 如果满足，尝试更大的最小值
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        System.out.println(ans);
    }

    /**
     * 检查是否存在一条路径，使得所有格子信号值 >= threshold
     */
    private static boolean check(int threshold) {
        // 如果起点或终点本身就小于阈值，直接返回false
        if (grid[0][0] < threshold || grid[R - 1][C - 1] < threshold) return false;

        Queue<int[]> queue = new LinkedList<>();
        queue.offer(new int[]{0, 0});
        boolean[][] visited = new boolean[R][C];
        visited[0][0] = true;

        while (!queue.isEmpty()) {
            int[] curr = queue.poll();
            int r = curr[0];
            int c = curr[1];

            if (r == R - 1 && c == C - 1) return true;

            for (int[] d : dirs) {
                int nr = r + d[0];
                int nc = c + d[1];

                if (nr >= 0 && nr < R && nc >= 0 && nc < C 
                    && !visited[nr][nc] && grid[nr][nc] >= threshold) {
                    visited[nr][nc] = true;
                    queue.offer(new int[]{nr, nc});
                }
            }
        }
        return false;
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(R \times C \times \log(\text{maxS}))$。
    - 二分查找的范围是信号强度的范围，约 $\log(65535) \approx 16$ 次。
    - 每次二分执行一次 BFS，复杂度为 $O(R \times C)$。
    - 给定 $R, C < 20$，总计算次数约 $16 \times 400 = 6400$，远低于机考限制。
- **空间复杂度**：$O(R \times C)$。
    - 需要一个 `visited` 数组和 BFS 队列来存储网格状态。
