## 二、 题目描述

高铁城市圈对人们的出行、经济的拉动效果明显。
每年都会规划新的高铁城市圈建设。在给定城市数量，可建设高铁的两城市间的修建成本列表、以及结合城市商业价值会固定建设的两城市建高铁。
请你设计算法，达到修建城市高铁的最低成本。注意，需要满足城市圈内城市间两两互联可达 (通过其他城市中转可达也属于满足条件）。

## 二、输入描述

1. 第一行，包含此城市圈中城市的数量 `N`、可建设高铁的两城市间修建成本列表数量 `M`、必建高铁的城市列表数量 `K`。三个数字用空格间隔。
2. 可建设高铁的两城市间的修建成本列表，为 `M` 行输入数据，格式为 `3` 个数字 `u v cost`，用空格分隔，长度不超过 1000。
3. 固定要修建的高铁城市列表，是上面参数 2 的子集，共有 `K` 行，每行输入为 `2` 个数字 `u v`，以空格分隔。

**参数范围：**
- 城市 id 从 1 开始编号。
- 建设成本为正整数，取值范围均不会超过 1000。
- 城市数量 $N \le 1000$，成本列表数量 $M \le 1000$。

## 三、输出描述

修建城市圈高铁的最低成本，正整数。如果城市圈内存在两城市之间无法互联，则返回 -1。

## 四、用例

### 用例1

**输入**
```
3 3 0
1 2 10
1 3 100
2 3 50
```

**输出**
```
60
```

**解释**
城市圈数量为 3，城市 id 为 1,2,3。
可选路径：(1,2)成本10，(1,3)成本100，(2,3)成本50。
必建路径为0。
为了让 1,2,3 互联，最优选择 (1,2) 和 (2,3)，成本为 10 + 50 = 60。

### 用例2

**输入**
```
3 3 1
1 2 10
1 3 100
2 3 50
1 3
```

**输出**
```
110
```

**解释**
必建 (1,3)，成本 100。
此时 1 和 3 已连通，还需连接 2。可选 (1,2) 成本 10 或 (2,3) 成本 50。
选 (1,2) 最优，总成本 100 + 10 = 110。

## 五、备注

- 考察点：图论、最小生成树（MST）、并查集。
- 注意：必建的高铁必须包含在总成本中，且由于是必建，无论其成本多高都必须先加入连通分量。

## 六、题目解析

### 1. 题目分析

本题要求在满足所有城市互联的前提下，求最小建设成本。这是一个典型的 **最小生成树 (Minimum Spanning Tree, MST)** 问题。
不同之处在于，题目给出了“必建”路径。这些必建路径必须首先被选中，然后再从剩余的可选路径中挑选成本最低的，直到所有城市连通。

### 2. 解题思路

1.  **数据结构选择**：
    *   使用 **并查集 (Union-Find)** 来维护城市的连通性。
    *   使用一个类或结构体 `Edge` 存储高铁路径的起点、终点和成本。
2.  **处理必建路径**：
    *   首先读取所有的 $M$ 条可选路径，可以用一个 Map 记录 `(u, v)` 对应的成本，方便查找必建路径的费用。
    *   遍历 $K$ 条必建路径，获取它们的成本并累加到 `totalCost`。
    *   在并查集中将必建路径的两个城市进行 `union` 操作。
3.  **Kruskal 算法流程**：
    *   将所有可选路径（排除已经作为必建路径处理过的，或者全部路径均可，只要在 Kruskal 过程中跳过已连通的即可）按成本从小到大排序。
    *   遍历排序后的路径，如果路径连接的两个城市在并查集中属于不同集合，则合并它们，并将成本累加到 `totalCost`。
4.  **连通性检查**：
    *   最后检查并查集中连通分量的个数。如果连通分量个数为 1（即所有城市都在一个集合中），则输出 `totalCost`；否则输出 -1。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static class Edge {
        int u, v, cost;
        public Edge(int u, int v, int cost) {
            this.u = u;
            this.v = v;
            this.cost = cost;
        }
    }

    static int[] parent;
    static int count; // 连通分量数量

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;

        int n = sc.nextInt(); // 城市数
        int m = sc.nextInt(); // 可选路径数
        int k = sc.nextInt(); // 必建路径数

        // 初始化并查集
        parent = new int[n + 1];
        for (int i = 1; i <= n; i++) parent[i] = i;
        count = n;

        // 存储所有可选边，方便根据 u,v 查 cost
        // 注意：题目说必建路径是参数2的子集
        Map<String, Integer> edgeMap = new HashMap<>();
        List<Edge> allEdges = new ArrayList<>();

        for (int i = 0; i < m; i++) {
            int u = sc.nextInt();
            int v = sc.nextInt();
            int cost = sc.nextInt();
            allEdges.add(new Edge(u, v, cost));
            // 保持 u < v 方便查找
            String key = Math.min(u, v) + "_" + Math.max(u, v);
            edgeMap.put(key, cost);
        }

        long totalCost = 0;
        // 记录哪些边已经被必建占用了，防止重复计算（虽然 Kruskal 会通过 find 过滤，但必建的钱必须先收）
        // 实际上题目说必建是子集，我们要先处理这些
        Set<String> mandatorySet = new HashSet<>();
        for (int i = 0; i < k; i++) {
            int u = sc.nextInt();
            int v = sc.nextInt();
            String key = Math.min(u, v) + "_" + Math.max(u, v);
            mandatorySet.add(key);
            
            // 累加必建成本并合并
            if (union(u, v)) {
                // 如果这次 union 成功，说明连接了新的组件
            }
            // 无论是否已经连通，必建的成本都要加（题目逻辑：固定建设）
            // 注意：如果同一个必建边输入多次，按题意理解只需加一次成本
            // 这里假设每种 u,v 组合在必建中只出现一次且对应 M 中的一条边
            totalCost += edgeMap.getOrDefault(key, 0);
        }

        // Kruskal 算法
        // 按成本升序排序
        allEdges.sort(Comparator.comparingInt(e -> e.cost));

        for (Edge edge : allEdges) {
            String key = Math.min(edge.u, edge.v) + "_" + Math.max(edge.u, edge.v);
            // 如果这条边不是必建边，且当前不连通，则选择它
            if (!mandatorySet.contains(key)) {
                if (union(edge.u, edge.v)) {
                    totalCost += edge.cost;
                }
            }
        }

        // 检查最终是否所有城市都连通
        if (count == 1) {
            System.out.println(totalCost);
        } else {
            System.out.println("-1");
        }
    }

    static int find(int i) {
        if (parent[i] == i) return i;
        return parent[i] = find(parent[i]);
    }

    static boolean union(int i, int j) {
        int rootI = find(i);
        int rootJ = find(j);
        if (rootI != rootJ) {
            parent[rootI] = rootJ;
            count--;
            return true;
        }
        return false;
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：
    *   读取 $M$ 条边并存入 Map：$O(M)$。
    *   处理 $K$ 条必建路径：$O(K \cdot \alpha(N))$，其中 $\alpha$ 是阿克曼函数的反函数，接近常数。
    *   排序 $M$ 条边：$O(M \log M)$。
    *   Kruskal 遍历：$O(M \cdot \alpha(N))$。
    *   总复杂度：$O(M \log M)$。对于 $M=1000$ 的数据规模，性能绰绰有余。
*   **空间复杂度**：
    *   存储边的列表及并查集数组：$O(M + N)$。
