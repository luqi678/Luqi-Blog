## 一、 题目描述

某块业务芯片最小容量单位为 1.25G，总容量为 $M \times 1.25G$，对该芯片资源编号为 $1, 2, \dots, M$。
该芯片支持 3 种不同的配置，分别为 A、B、C。
- 配置 A: 占用容量为 $1.25 \times 1 = 1.25G$
- 配置 B: 占用容量为 $1.25 \times 2 = 2.5G$
- 配置 C: 占用容量为 $1.25 \times 8 = 10G$

某块板卡上集成了 $N$ 块上述芯片，对芯片编号为 $1, 2, \dots, N$。各个芯片之间彼此独立，不能跨芯片占用资源。
给定板卡上芯片数量 $N$、每块芯片容量 $M$、用户按次序配置后，请输出芯片资源占用情况。

**资源分配规则：**
1. 按照芯片编号从小到大分配所需资源。
2. 每一个配置总是尝试从第一块芯片开始尝试放入，如果当前芯片剩余空间不足，则尝试下一块芯片。
3. 芯片上资源如果被占用标记为 1，没有被占用标记为 0。
4. 用户配置是按次序依次配置到芯片中。
5. 如果用户配置序列中某个配置超过了芯片总容量 $M$，丢弃该配置，继续遍历用户后续配置。
6. 如果一个配置的大小 $\le M$，但在所有芯片中都找不到足够的连续空闲空间（本题中由于是单位化管理，只需考虑剩余总量），则该配置也应被丢弃（根据通用逻辑及样例推导）。

## 二、输入描述

第一行一个整数 $M$：每块芯片容量为 $M \times 1.25G$，取值范围为: $1 \sim 256$。
第二行一个整数 $N$：板卡包含的芯片数量，取值范围为 $1 \sim 32$。
第三行为一个字符串，表示用户配置序列：例如 `ACABA`，长度不超过 1000。

## 三、输出描述

输出 $N$ 行，表示每块芯片的占用情况。每行由 '1' 和 '0' 组成的字符串，长度为 $M$。

## 四、用例

### 用例1
**输入**
```text
8
2 
ACABA
```
**输出**
```text
11111000
11111111
```

### 用例2
**输入**
```text
8
2 
ACBCB
```
**输出**
```text
11111000
11111111
```

## 五、备注
- A 占用 1 个单位，B 占用 2 个单位，C 占用 8 个单位。
- 优先分配编号小的芯片（First Fit 策略）。
- 最终输出时，每块芯片已占用的单位用 '1' 表示（且全部靠左），剩余单位用 '0' 表示。

## 六、题目解析

### 1. 题目分析
本题属于典型的**装箱问题（Bin Packing Problem）**的变体。由于题目明确了“按照芯片编号从小到大分配”，这对应了贪心算法中的 **First Fit (首次适应算法)**。
- 芯片相当于箱子，容量为 $M$。
- 配置 A, B, C 相当于体积为 1, 2, 8 的物品。
- 规则：对于每一个物品，从第 1 个箱子开始检查，哪一个箱子能装下就放哪，放进去后更新箱子剩余容量。

### 2. 解题思路
1. **初始化**：创建一个长度为 $N$ 的数组 `chips`，记录每块芯片已使用的容量，初始均为 0。
2. **解析配置**：遍历输入的配置字符串。
   - 'A' -> `need = 1`
   - 'B' -> `need = 2`
   - 'C' -> `need = 8`
3. **合法性检查**：如果 `need > M`，直接丢弃该配置。
4. **分配资源**：从 $i = 0$ 到 $N-1$ 遍历芯片：
   - 如果 `chips[i] + need <= M`，则 `chips[i] += need`，分配完成，跳出当前配置的芯片搜索循环。
   - 如果所有芯片都无法容纳，丢弃该配置（虽然题目主要强调了超过 $M$ 丢弃，但实际操作中放不下也只能跳过）。
5. **构造输出**：遍历 `chips` 数组，对于每一个芯片，输出 `chips[i]` 个 '1' 和 `M - chips[i]` 个 '0'。

### 3. 代码实现

#### Java 实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 1. 输入处理
        if (!sc.hasNextInt()) return;
        int M = sc.nextInt(); // 每块芯片容量
        if (!sc.hasNextInt()) return;
        int N = sc.nextInt(); // 芯片数量
        if (!sc.hasNext()) return;
        String configs = sc.next(); // 用户配置序列

        // 2. 记录每块芯片已占用的容量
        int[] chipUsage = new int[N];

        // 3. 遍历配置序列进行分配 (First Fit 策略)
        for (char c : configs.toCharArray()) {
            int need = 0;
            if (c == 'A') need = 1;
            else if (c == 'B') need = 2;
            else if (c == 'C') need = 8;
            else continue; // 非法字符

            // 如果单个配置需求超过单块芯片总容量，丢弃
            if (need > M) {
                continue;
            }

            // 从第一块芯片开始尝试放置
            for (int i = 0; i < N; i++) {
                if (chipUsage[i] + need <= M) {
                    chipUsage[i] += need;
                    break; // 分配成功，处理下一个配置
                }
            }
        }

        // 4. 输出结果
        for (int i = 0; i < N; i++) {
            StringBuilder sb = new StringBuilder();
            // 已占用部分填 '1'
            for (int j = 0; j < chipUsage[i]; j++) {
                sb.append('1');
            }
            // 剩余部分填 '0'
            for (int j = 0; j < M - chipUsage[i]; j++) {
                sb.append('0');
            }
            System.out.println(sb.toString());
        }
        
        sc.close();
    }
}
```

#### Python 实现
```python
import sys

def solve():
    try:
        line1 = sys.stdin.readline()
        if not line1: return
        m = int(line1.strip())
        
        line2 = sys.stdin.readline()
        if not line2: return
        n = int(line2.strip())
        
        line3 = sys.stdin.readline()
        if not line3: return
        configs = line3.strip()
    except EOFError:
        return

    # 记录芯片已使用的容量
    chip_usage = [0] * n
    
    # 配置映射
    size_map = {'A': 1, 'B': 2, 'C': 8}
    
    for char in configs:
        if char not in size_map:
            continue
        need = size_map[char]
        
        # 超过总容量丢弃
        if need > m:
            continue
            
        # First Fit 分配
        for i in range(n):
            if chip_usage[i] + need <= m:
                chip_usage[i] += need
                break
                
    # 输出结果
    for usage in chip_usage:
        print('1' * usage + '0' * (m - usage))

if __name__ == "__main__":
    solve()
```

### 4. 复杂度分析
- **时间复杂度**：$O(L \times N)$，其中 $L$ 是配置字符串的长度（最高 1000），$N$ 是芯片数量（最高 32）。总计算量约为 $3.2 \times 10^4$，远低于机考限制。
- **空间复杂度**：$O(N + M)$，用于存储芯片状态和输出字符串。
