## 二、题目描述

给定一个数组，里面有 6 个整数，求这个数组能够表示的最大 24 进制的时间是多少，输出这个时间，无法表示输出 invalid。

## 三、输入描述

输入为一个整数数组，数组内有六个整数。
输入整数数组长度为 6，不需要考虑其它长度，元素值为 0 或者正整数，6 个数字每个数字只能使用一次。
注：输入格式可能包含方括号和逗号，如 `[0,2,3,0,5,6]`。

## 四、输出描述

输出为一个 24 进制格式的时间（HH:MM:SS），或者字符串 ”invalid“。

## 五、用例

### 用例1
**输入**：
`[0,2,3,0,5,6]`

**输出**：
`23:56:00`

### 用例2
**输入**：
`[9,9,9,9,9,9]`

**输出**：
`invalid`

## 六、备注

1. 输出时间格式为 `HH:MM:SS` 格式。
2. 小时取值范围：[00, 23]。
3. 分钟取值范围：[00, 59]。
4. 秒数取值范围：[00, 59]。

## 七、题目解析

### 1. 题目分析
题目要求从给定的 6 个数字中，选出所有数字并排列组合成一个合法的最大时间。时间的格式是 `HH:MM:SS`，其中：
- `HH` 必须在 `00` 到 `23` 之间。
- `MM` 必须在 `00` 到 `59` 之间。
- `SS` 必须在 `00` 到 `59` 之间。

### 2. 解题思路
由于只有 6 个数字，其全排列的总数是 $6! = 720$ 种，数量非常小。我们可以通过**全排列**的方法穷举所有可能的组合，然后对每一种组合进行合法性校验，并记录其中的最大值。

1. **解析输入**：由于输入可能带有 `[` 和 `]` 以及 `,`，需要先提取出这 6 个整数。
2. **全排列生成**：利用回溯算法（Backtracking）生成 6 个数字的所有排列。
3. **合法性检查**：对于每一种排列 $(d_1, d_2, d_3, d_4, d_5, d_6)$：
   - 计算 $HH = d_1 \times 10 + d_2$
   - 计算 $MM = d_3 \times 10 + d_4$
   - 计算 $SS = d_5 \times 10 + d_6$
   - 判断 $0 \le HH < 24$ 且 $0 \le MM < 60$ 且 $0 \le SS < 60$。
4. **寻找最大值**：如果合法，将其转换为总秒数或直接进行字符串比较，记录最大的一组。
5. **格式化输出**：如果找到了最大时间，按 `HH:MM:SS` 格式输出（注意补零）；否则输出 `invalid`。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static String maxTime = "";
    static int maxTotalSeconds = -1;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String input = sc.nextLine();
        
        // 解析输入，提取数字
        input = input.replace("[", "").replace("]", "");
        String[] parts = input.split(",");
        int[] nums = new int[6];
        for (int i = 0; i < 6; i++) {
            nums[i] = Integer.parseInt(parts[i].trim());
        }

        // 进行全排列搜索
        boolean[] used = new boolean[6];
        int[] path = new int[6];
        backtrack(nums, used, path, 0);

        if (maxTotalSeconds == -1) {
            System.out.println("invalid");
        } else {
            System.out.println(maxTime);
        }
    }

    private static void backtrack(int[] nums, boolean[] used, int[] path, int depth) {
        if (depth == 6) {
            checkAndUpdateTime(path);
            return;
        }

        for (int i = 0; i < 6; i++) {
            if (!used[i]) {
                used[i] = true;
                path[depth] = nums[i];
                backtrack(nums, used, path, depth + 1);
                used[i] = false;
            }
        }
    }

    private static void checkAndUpdateTime(int[] p) {
        int hour = p[0] * 10 + p[1];
        int minute = p[2] * 10 + p[3];
        int second = p[4] * 10 + p[5];

        if (hour >= 0 && hour < 24 && minute >= 0 && minute < 60 && second >= 0 && second < 60) {
            int totalSeconds = hour * 3600 + minute * 60 + second;
            if (totalSeconds > maxTotalSeconds) {
                maxTotalSeconds = totalSeconds;
                maxTime = String.format("%02d:%02d:%02d", hour, minute, second);
            }
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N!)$，其中 $N=6$。$6! = 720$，计算量非常小，完全满足机考性能要求。
- **空间复杂度**：$O(N)$，递归深度为 6，用于存储排列路径和标记数组。
