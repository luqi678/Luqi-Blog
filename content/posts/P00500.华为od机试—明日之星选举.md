## 一、 题目描述

给定一组选票 `votes`。`votes[i]` 代表第 $i$ 张选票的内容，包含一个字符串 `"zhangsan,hanmei"`，表示推荐 `zhangsan` 为明日之星，反对 `hanmei` 成为明日之星。赞成或反对使用逗号 (`,`) 分割。

给定一个数组 $n$，表示最后将筛选出赞成票最多的 $n$ 个人。
排序规则如下：
1. 如果赞成票数量不同，则赞成票越多，排序越靠前；
2. 如果赞成票相同，则反对票越少，排序越靠前；
3. 如果赞成票和反对票都相同，则按照姓名字典序升序排序。

**注意：**
1. $1 \le votes.length \le 500$
2. $1 \le votes[i].length \le 20$
3. `votes` 由小写字母和逗号组成，且最多只有一个逗号。
4. `votes` 中赞成票必选（即逗号前必定有名字），反对票可选（可能没有逗号）。
5. $n$ 的取值范围为 $[1, \text{不同姓名的数量}]$。

## 二、输入描述

第一行输入为 $M$，表示选票的数量，其中 $1 \le M \le 500$。
接下来 $M$ 行为选票具体内容。
最后一行输入为 $N$，表示最终当选明日之星的人员个数，$1 \le N \le \text{不同姓名的数量}$。

## 三、输出描述

输出一个字符串，返回当选的 $N$ 个人的姓名，按规则顺序排列，姓名之间使用逗号分割。

## 四、用例

### 用例1

**输入：**
```text
6
zhangsan,hanmei
zhangsan,lisi
lisi
lisi
wangwu
hanmei
2
```
**输出：**
```text
zhangsan,lisi
```

### 用例2

**输入：**
```text
4
zhangsan,lisi
lisi,wangwu
wangwu,qianliu
qianliu,zhangsan
2
```
**输出：**
```text
lisi,qianliu
```
**说明：**
每个人的情况都是：1赞成，1反对。
由于票数相同，按照字典序升序排序：`hanmei`, `lisi`, `qianliu`, `wangwu`, `zhangsan`。
前两个是 `lisi, qianliu`。（注：示例中名字可能不全，按字母序排 lisi < qianliu）。

## 六、题目解析

### 1. 题目分析
本题是一道典型的“统计+排序”类问题。核心在于：
1.  **解析字符串**：每行输入可能包含一个名字（赞成），也可能包含两个名字（第一个赞成，第二个反对）。
2.  **数据统计**：需要记录每个候选人的赞成票数和反对票数。使用 `Map<String, Candidate>` 结构非常合适。
3.  **多级排序**：需要根据赞成票（降序）、反对票（升序）、姓名（字典序升序）三个维度进行排序。

### 2. 解题思路
1.  创建一个类 `Candidate`，包含 `name` (String), `favor` (int), `oppose` (int)。
2.  读取 $M$ 行输入，根据是否存在逗号分割出赞成者和反对者。
3.  维护一个 `HashMap<String, Candidate>`：
    *   遇到赞成者，若 Map 中不存在则创建，`favor` 加 1。
    *   遇到反对者，若 Map 中不存在则创建，`oppose` 加 1。
4.  将 Map 的 `values` 转换成一个 `List<Candidate>`。
5.  对 List 进行排序：
    *   `o1.favor != o2.favor` ? `o2.favor - o1.favor` (降序)
    *   `o1.oppose != o2.oppose` ? `o1.oppose - o2.oppose` (升序)
    *   否则 `o1.name.compareTo(o2.name)` (字典序升序)
6.  取出排序后的前 $N$ 个名字，拼接成字符串输出。

## 七、代码实现

### Java8实现

```java
import java.util.*;
import java.util.stream.Collectors;

public class Main {
    static class Candidate {
        String name;
        int favor = 0;
        int oppose = 0;

        Candidate(String name) {
            this.name = name;
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int m = Integer.parseInt(sc.nextLine());

        Map<String, Candidate> map = new HashMap<>();

        for (int i = 0; i < m; i++) {
            String line = sc.nextLine();
            if (line == null || line.isEmpty()) continue;

            String[] parts = line.split(",");
            // 处理赞成票
            String favorName = parts[0];
            map.putIfAbsent(favorName, new Candidate(favorName));
            map.get(favorName).favor++;

            // 处理反对票（如果有）
            if (parts.length > 1) {
                String opposeName = parts[1];
                map.putIfAbsent(opposeName, new Candidate(opposeName));
                map.get(opposeName).oppose++;
            }
        }

        int n = Integer.parseInt(sc.nextLine());

        // 排序逻辑
        List<Candidate> list = new ArrayList<>(map.values());
        list.sort((a, b) -> {
            if (a.favor != b.favor) {
                return b.favor - a.favor; // 赞成票降序
            }
            if (a.oppose != b.oppose) {
                return a.oppose - b.oppose; // 反对票升序
            }
            return a.name.compareTo(b.name); // 姓名字典序升序
        });

        // 取前N个并输出
        List<String> result = new ArrayList<>();
        for (int i = 0; i < Math.min(n, list.size()); i++) {
            result.add(list.get(i).name);
        }

        System.out.println(String.join(",", result));
    }
}
```

## 八、复杂度分析

1.  **时间复杂度**：
    *   读取 $M$ 张选票并存入 Map：$O(M \times L)$，其中 $L$ 为名字平均长度。
    *   List 排序：假设不同姓名数量为 $K$（$K \le 2M$），排序复杂度为 $O(K \log K)$。
    *   整体复杂度为 $O(M \times L + K \log K)$，在 $M=500$ 的数据规模下性能极佳。
2.  **空间复杂度**：
    *   使用 HashMap 存储所有候选人的票数信息，空间复杂度为 $O(K \times L)$。
