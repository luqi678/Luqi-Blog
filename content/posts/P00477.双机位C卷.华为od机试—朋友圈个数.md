## 二、 题目描述

给定一组朋友关系，统计一下该朋友关系网中的朋友圈个数。

**朋友圈的定义**：
一个朋友圈至少由 3 个朋友组成，且要求同一个朋友圈中的任意两个人都具有直接的朋友关系（即构成一个完全图）。

**不考虑子朋友圈**：
即当发现 `[A, B, C, D]` 组成一个朋友圈时，其子集如 `[A, B, C]`、`[A, B, D]` 等子朋友圈不单独计数。只有无法被进一步扩展的完全子图（极大团）才被计为一个朋友圈。

## 二、 输入描述

1. 先输入一个数字 $N$ 代表关系的总数。
2. 后面每条关系一行，两个成员以逗号分隔（例如 `A,B`）。

**约束条件**：
- $1 \le$ 关系数 $\le 10^3$
- 总朋友人数 $\le 100$
- 输入保证 $Friend[i][0]$ 和 $Friend[i][1]$ 一定不一样。
- 同一关系不会反向输入。

## 三、 输出描述

输出一个整数，表示整个关系网中朋友圈（满足条件的极大团）的个数。

## 四、 用例

### 用例1
**输入：**
```
3
A,B
A,C
B,D
```
**输出：**
```
0
```
**说明：** 没有任何三个或以上的人两两互为朋友。

### 用例2
**输入：**
```
7
A,B
A,C
B,C
A,D
D,E
B,D
A,E
```
**输出：**
```
3
```
**说明：** 存在三个极大团：`{A, B, C}`, `{A, B, D}`, `{A, D, E}`。它们都满足至少3人且两两相连，且不互为包含关系。

### 用例3
**输入：**
```
6
A,B
A,C
B,C
A,D
B,D
D,C
```
**输出：**
```
1
```
**说明：** `{A, B, C, D}` 四个人两两相连，构成一个大小为 4 的极大团。子集不计入，故输出 1。

## 五、 备注
本题的核心在于寻找无向图中所有**大小 $\ge 3$ 的极大团（Maximal Clique）**。

## 六、 题目解析

### 1. 题目分析
题目要求的“朋友圈”在图论中对应的是**团 (Clique)** 的概念：
- **团**：一个顶点集合，其中任意两个顶点之间都有边相连。
- **极大团 (Maximal Clique)**：一个团，如果它不能通过增加另一个顶点来扩展成更大的团。
- **题目要求**：
    1. 团的大小必须 $\ge 3$。
    2. 只统计极大团（不考虑子朋友圈）。

### 2. 解题思路
寻找图中所有极大团的经典算法是 **Bron-Kerbosch 算法**。

**Bron-Kerbosch 算法流程**：
维护三个集合：
- $R$：当前已确定的团中的节点。
- $P$：可能可以加入团的候选节点。
- $X$：已经处理完毕，且不能再加入团的节点（用于排除重复和非极大团）。

递归函数 `BronKerbosch(R, P, X)`：
1. 如果 $P$ 和 $X$ 都为空，说明 $R$ 是一个极大团。若 $|R| \ge 3$，结果计数加一。
2. 遍历 $P$ 中的节点 $v$：
    - 递归调用 `BronKerbosch(R ∪ {v}, P ∩ N(v), X ∩ N(v))`。
    - 将 $v$ 从 $P$ 中移除，加入 $X$。

由于人数 $\le 100$，该算法虽然在最坏情况下是指数级的，但在该规模下效率极高。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static int resultCount = 0;
    static boolean[][] adj;
    static int numPeople = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        sc.nextLine();

        List<String[]> relations = new ArrayList<>();
        Set<String> names = new TreeSet<>();

        for (int i = 0; i < n; i++) {
            String line = sc.nextLine();
            String[] parts = line.split(",");
            relations.add(parts);
            names.add(parts[0]);
            names.add(parts[1]);
        }

        // 映射名字到索引
        List<String> nameList = new ArrayList<>(names);
        Map<String, Integer> nameToIndex = new HashMap<>();
        for (int i = 0; i < nameList.size(); i++) {
            nameToIndex.put(nameList.get(i), i);
        }

        numPeople = nameList.size();
        adj = new boolean[numPeople][numPeople];

        for (String[] rel : relations) {
            int u = nameToIndex.get(rel[0]);
            int v = nameToIndex.get(rel[1]);
            adj[u][v] = true;
            adj[v][u] = true;
        }

        // Bron-Kerbosch 准备
        Set<Integer> R = new HashSet<>();
        Set<Integer> P = new HashSet<>();
        Set<Integer> X = new HashSet<>();

        for (int i = 0; i < numPeople; i++) {
            P.add(i);
        }

        bronKerbosch(R, P, X);

        System.out.println(resultCount);
    }

    /**
     * Bron-Kerbosch 算法寻找极大团
     */
    public static void bronKerbosch(Set<Integer> R, Set<Integer> P, Set<Integer> X) {
        if (P.isEmpty() && X.isEmpty()) {
            // 找到一个极大团
            if (R.size() >= 3) {
                resultCount++;
            }
            return;
        }

        Iterator<Integer> it = P.iterator();
        while (it.hasNext()) {
            int v = it.next();

            // R ∪ {v}
            Set<Integer> nextR = new HashSet<>(R);
            nextR.add(v);

            // P ∩ N(v)
            Set<Integer> nextP = new HashSet<>();
            for (int p : P) {
                if (adj[v][p]) nextP.add(p);
            }

            // X ∩ N(v)
            Set<Integer> nextX = new HashSet<>();
            for (int x : X) {
                if (adj[v][x]) nextX.add(x);
            }

            bronKerbosch(nextR, nextP, nextX);

            // P := P \ {v}, X := X ∪ {v}
            it.remove();
            X.add(v);
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：Bron-Kerbosch 算法的最坏情况复杂度为 $O(3^{V/3})$，其中 $V$ 是顶点数。在本题 $V \le 100$ 的情况下，实际运行时间远小于此上限，因为图通常是稀疏的，且递归树深度受限于团的大小。
- **空间复杂度**：$O(V^2)$ 用于存储邻接矩阵，以及递归调用栈中的集合存储。对于 $V=100$，空间开销约为 $100 \times 100$ 的布尔矩阵，内存消耗极低。
