## 一、 题目描述

给定两个字符串 `str1` 和 `str2`。如果 `str1` 的某种**排列组合**（即包含相同字符及其数量的字符串）能够在 `str2` 里面找到，则返回 `str2` 中第一个匹配子串的起始字符下标。

如果不存在，则返回 -1。

## 二、输入描述

输入为一行，包含两个字符串 `str1` 和 `str2`，中间用空格分隔。

- `str1` 和 `str2` 仅包含小写英文字母（根据示例推断，通常此类题目会限制字符集）。

## 三、输出描述

输出一个整数，表示第一个匹配的起始下标。如果未找到，输出 -1。

## 四、用例

### 用例1
**输入：**
```text
abc qwcab
```
**输出：**
```text
2
```
**说明：** `str1` 为 "abc"，`str2` 中下标 2 开始的子串为 "cab"，它是 "abc" 的一个排列，因此返回 2。

### 用例2
**输入：**
```text
abc qcwab
```
**输出：**
```text
-1
```
**说明：** `str2` 中没有任何长度为 3 的子串是 "abc" 的排列。

## 五、备注
1. `str1` 的长度通常小于或等于 `str2` 的长度。
2. 匹配是指字符种类和每个字符出现的次数完全一致。

## 六、题目解析

### 1. 题目分析
本题要求判断 `str1` 的**全排列**是否是 `str2` 的子串。全排列匹配的核心在于：在 `str2` 中寻找一个长度与 `str1` 相等的连续子串，使得该子串中每个字符出现的频率与 `str1` 完全相同。

### 2. 解题思路
这是一个典型的**固定长度滑动窗口**问题：
1. **统计频率**：首先统计 `str1` 中每个字符出现的次数（可以使用长度为 26 的数组或 HashMap）。
2. **初始化窗口**：在 `str2` 上取前 `len(str1)` 个字符，统计其字符频率。
3. **滑动比较**：
   - 比较当前窗口的频率统计与 `str1` 的频率统计是否一致。
   - 如果一致，直接返回当前窗口的左边界下标。
   - 如果不一致，将窗口向右滑动一位：移出左侧字符，加入右侧新字符，更新频率统计。
4. **结束条件**：如果遍历完 `str2` 仍未找到，返回 -1。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNext()) return;
        
        String str1 = sc.next();
        String str2 = sc.next();
        
        int n1 = str1.length();
        int n2 = str2.length();
        
        if (n1 > n2) {
            System.out.println("-1");
            return;
        }
        
        // 统计 str1 的字符频率
        int[] count1 = new int[26];
        for (char c : str1.toCharArray()) {
            count1[c - 'a']++;
        }
        
        // 初始化第一个窗口（长度为 n1）
        int[] count2 = new int[26];
        for (int i = 0; i < n1; i++) {
            count2[str2.charAt(i) - 'a']++;
        }
        
        // 开始滑动窗口
        for (int i = 0; i <= n2 - n1; i++) {
            // 检查当前窗口是否匹配
            if (Arrays.equals(count1, count2)) {
                System.out.println(i);
                return;
            }
            
            // 窗口右移：移出下标 i 的字符，移入下标 i + n1 的字符
            if (i < n2 - n1) {
                count2[str2.charAt(i) - 'a']--;
                count2[str2.charAt(i + n1) - 'a']++;
            }
        }
        
        System.out.println("-1");
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N_2 \times \Sigma)$，其中 $N_2$ 是 `str2` 的长度，$\Sigma$ 是字符集大小（本题为 26）。每次移动窗口需要比较两个数组。如果优化比较过程（通过记录匹配字符的个数），可以达到 $O(N_2)$。
- **空间复杂度**：$O(\Sigma)$，需要两个长度为 26 的数组来存储字符频率。
