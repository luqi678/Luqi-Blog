## 一、 题目描述
给定一个非负整数数组 `nums`，重新排列这些整数的顺序，使得它们拼接在一起形成的数字最大。
由于结果可能非常大，所以你需要返回一个字符串而不是整数。

## 二、输入描述
一个非负整数数组 `nums`。
- `0 <= nums.length <= 100`
- `0 <= nums[i] <= 10^9`

## 三、输出描述
拼接后形成的最大的数字字符串。

## 四、用例
### 用例1
输入：`nums = [10, 2]`
输出：`"210"`

### 用例2
输入：`nums = [3, 30, 34, 5, 9]`
输出：`"9534330"`

## 五、备注
1. 如果最终拼接的结果开头是 "0"，说明数组中全部是 0，应当返回 "0"。
2. 排序规则是本题的核心。

## 六、题目解析

### 1. 题目分析
本题的本质是一个**排序问题**。我们需要确定一种排序准则，使得排序后的元素按顺序拼接能得到最大值。
直观上，我们可能会想按照字典序从大到小排列，但这会遇到问题：例如 `[3, 30]`。
- 按字典序排列：`"30" > "3"`，拼接结果是 `"303"`。
- 实际最大结果：`"330"`。
显然，传统的字典序无法直接解决问题。

### 2. 解题思路
*   **自定义比较规则**：
    对于数组中的两个数字 $a$ 和 $b$，我们将它们转换为字符串 $S_a$ 和 $S_b$。
    - 如果 $S_a + S_b > S_b + S_a$，则说明在拼接时，$a$ 应该排在 $b$ 的前面。
    - 如果 $S_a + S_b < S_b + S_a$，则说明在拼接时，$b$ 应该排在 $a$ 的前面。
*   **贪心策略证明**：
    这种比较准则具有传递性，符合全序关系的定义，因此可以使用标准的排序算法。通过这种局部最优的选择（每次选出当前能让拼接结果更大的前缀），最终可以达成全局最优。
*   **特殊情况处理**：
    排序完成后，如果数组中最大的元素是 `0`（即排序后的第一个元素是 `"0"`），那么整个拼接结果必然是 `"0"`，而不能是 `"00...0"`。

### 3. 代码实现

#### Java8实现
```java
import java.util.Arrays;
import java.util.stream.Collectors;

public class Solution {
    public String largestNumber(int[] nums) {
        // 1. 将数字数组转换为字符串数组
        String[] strs = new String[nums.length];
        for (int i = 0; i < nums.length; i++) {
            strs[i] = String.valueOf(nums[i]);
        }

        // 2. 自定义排序规则
        // 如果 (b + a) 的字典序大于 (a + b)，则 b 应该排在 a 前面
        Arrays.sort(strs, (a, b) -> (b + a).compareTo(a + b));

        // 3. 特殊情况：如果排序后第一个元素是 "0"，说明整个数组全是 0
        if (strs[0].equals("0")) {
            return "0";
        }

        // 4. 拼接字符串并返回
        StringBuilder sb = new StringBuilder();
        for (String s : strs) {
            sb.append(s);
        }
        return sb.toString();
    }

    public static void main(String[] args) {
        Solution sol = new Solution();
        System.out.println(sol.largestNumber(new int[]{10, 2})); // 输出: "210"
        System.out.println(sol.largestNumber(new int[]{3, 30, 34, 5, 9})); // 输出: "9534330"
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(n \log n \cdot k)$。其中 $n$ 是数组的长度，$k$ 是数字的平均长度。排序需要 $O(n \log n)$ 次比较，每次比较涉及字符串拼接和比较，耗时 $O(k)$。
*   **空间复杂度**：$O(n \cdot k)$。需要额外的空间存储字符串数组和最终生成的字符串。
