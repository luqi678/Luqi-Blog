## 二、 题目描述

考古问题：假设以前的石碑被打碎成了很多块，每块上面都有一个或若干个字符。请你写个程序来把之前石碑上文字可能的组合全部写出来，按升序进行排列。

## 二、输入描述

- 第一行输入 $n$，$n$ 表示石碑碎片的个数（$1 \le n \le 10$）。
- 第二行依次输入石碑碎片上的文字内容 $s$，共有 $n$ 组。

## 三、输出描述

- 输出石碑文字的组合（按照升序排列），行末无多余空格。

## 四、用例

### 用例1
**输入**
```text
3
a b c
```
**输出**
```text
abc
acb
bac
bca
cab
cba
```

### 用例2
**输入**
```text
3
a b a
```
**输出**
```text
aab
aba
baa
```

## 五、备注
1. 石碑碎片的个数 $n$ 较小（通常 $\le 10$），说明可以使用回溯法。
2. 碎片本身可能包含多个字符。
3. 必须处理重复碎片导致的重复组合（去重）。
4. 输出结果需要按字典序升序排列。

## 六、题目解析

### 1. 题目分析
本题的本质是**带重复元素的全排列问题**。
- 我们有 $n$ 个字符串碎片，需要将它们拼接成一个完整的字符串。
- 所有的碎片都必须使用且仅使用一次。
- 不同的排列顺序可能会产生相同的字符串（例如两个 `a` 碎片），需要去重。
- 最终结果需要按字典序排序输出。

### 2. 解题思路
1.  **预处理**：首先对输入的 $n$ 个碎片数组进行字典序排序。排序有两个目的：
    - 方便在回溯过程中进行剪枝去重。
    - 保证生成全排列的过程本身就是按字典序进行的。
2.  **回溯算法 (DFS)**：
    - 使用一个布尔数组 `used` 来记录每个碎片是否已被使用。
    - 使用一个临时容器（如 `StringBuilder` 或 `String[]`）记录当前路径。
    - **去重关键逻辑**：在每一层递归中，如果当前元素与上一个元素相同（`arr[i] == arr[i-1]`），并且上一个元素没有被使用（`!used[i-1]`），则跳过当前元素。这保证了相同元素在同一层只会被选择一次。
3.  **输出**：当路径长度达到 $n$ 时，将当前拼接的字符串输出。

### 3. 代码实现

#### Java8实现

```java
import java.util.Arrays;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        String[] fragments = new String[n];
        for (int i = 0; i < n; i++) {
            fragments[i] = sc.next();
        }

        // 1. 先对碎片进行排序，这是实现去重和按升序输出的基础
        Arrays.sort(fragments);

        boolean[] used = new boolean[n];
        StringBuilder path = new StringBuilder();
        
        // 2. 开始回溯
        backtrack(fragments, used, 0, n, new String[n]);
    }

    /**
     * @param fragments 原始碎片数组
     * @param used      标记数组
     * @param depth     当前递归深度
     * @param n         碎片总数
     * @param result    存储当前排列的路径
     */
    private static void backtrack(String[] fragments, boolean[] used, int depth, int n, String[] result) {
        // 终止条件：所有碎片都已使用
        if (depth == n) {
            System.out.println(String.join("", result));
            return;
        }

        for (int i = 0; i < n; i++) {
            // 如果该碎片已使用，跳过
            if (used[i]) continue;

            // 剪枝去重：
            // 如果当前元素和前一个元素相同，且前一个元素在这一层未被使用过，则跳过
            if (i > 0 && fragments[i].equals(fragments[i - 1]) && !used[i - 1]) {
                continue;
            }

            // 选择当前碎片
            used[i] = true;
            result[depth] = fragments[i];
            
            // 递归进入下一层
            backtrack(fragments, used, depth + 1, n, result);

            // 回溯，撤销选择
            used[i] = false;
        }
    }
}
```

### 4. 复杂度分析

-   **时间复杂度**：$O(n! \cdot L)$，其中 $n$ 是碎片的个数，$L$ 是拼接后的字符串长度。全排列的数量最多为 $n!$，每次拼接和输出需要 $O(L)$ 的时间。鉴于 $n \le 10$，这个复杂度在可接受范围内。
-   **空间复杂度**：$O(n)$。主要开销为递归调用的栈空间以及记录路径的数组和标记数组。
