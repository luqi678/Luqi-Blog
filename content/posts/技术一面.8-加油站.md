## 一、 题目描述
在一条环路上有 $N$ 个加油站，其中第 $i$ 个加油站有汽油 `gas[i]` 升。
你有一辆油箱容量无限的汽车，从第 $i$ 个加油站开往第 $i+1$ 个加油站需要消耗汽油 `cost[i]` 升。你从其中的一个加油站出发，开始时油箱为空。
如果你可以绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1。

**说明：**
- 如果题目有解，该答案即为唯一答案。
- 输入数组均为非空数组，且长度相同。
- 输入数组中的元素均为非负整数。

## 二、输入描述
- `gas`: 一个整数数组，表示每个加油站拥有的汽油量。
- `cost`: 一个整数数组，表示从一个加油站到下一个加油站的成本。

## 三、输出描述
- 如果存在一个起始加油站，可以环形巡逻所有加油站并回到起点，则返回该加油站的索引。
- 如果不存在这样的加油站，则返回 -1。

## 四、用例
### 用例1
输入: 
gas = [1,2,3,4,5]
cost = [3,4,5,1,2]
输出: 3
解释:
从 3 号加油站 (索引为 3) 出发，可获得 4 升汽油。此时油箱有 = 0 + 4 = 4 升。
开往 4 号加油站，消耗 1 升，剩余 4 - 1 = 3 升。
开往 0 号加油站，获得 5 升，消耗 2 升，剩余 3 + 5 - 2 = 6 升。
开往 1 号加油站，获得 1 升，消耗 3 升，剩余 6 + 1 - 3 = 4 升。
开往 2 号加油站，获得 2 升，消耗 4 升，剩余 4 + 2 - 4 = 2 升。
开往 3 号加油站，获得 3 升，消耗 5 升，剩余 2 + 3 - 5 = 0 升。
刚好回到起点，因此返回 3。

### 用例2
输入: 
gas = [2,3,4]
cost = [3,4,3]
输出: -1
解释:
你不能从 0 号或 1 号加油站出发，因为没有足够的汽油可以到达下一个加油站。
如果从 2 号加油站出发，可以获得 4 升汽油。 此时油箱有 = 0 + 4 = 4 升。
开往 0 号加油站，消耗 3 升，剩余 4 - 3 = 1 升。
开往 1 号加油站，获得 2 升，消耗 3 升，剩余 1 + 2 - 3 = 0 升。
开往 2 号加油站，获得 3 升，消耗 4 升，剩余 0 + 3 - 4 = -1 升。
你无法返回 2 号加油站，因为在开往 2 号的路上油箱已经空了。

## 五、备注
该题目是经典的贪心算法题。关键在于理解：如果总油量大于等于总消耗量，则一定存在解。

## 六、题目解析
### 1. 题目分析
本题考察对环形数组的理解以及贪心策略的应用。核心逻辑是判断在每一个站点时，当前的累计剩余油量是否足以支撑到下一个站点。

### 2. 解题思路
我们可以通过以下两个核心结论来简化问题：
1. **全局油量平衡**：如果 $\sum gas[i] < \sum cost[i]$，无论从哪个点出发，都不可能绕行一圈。
2. **贪心局部选择**：从 `0` 号加油站出发，计算到第 `i` 站时的剩余油量 `curSum`。如果在到达第 `i+1` 站时 `curSum < 0`，说明从 `0` 到 `i` 之间的任何一个站作为起点都无法越过 `i` 站。此时，我们应当将起点设为 `i+1`，并将 `curSum` 清零重新计算。

### 3. 代码实现
#### Java8实现
```java
public class Solution {
    public int canCompleteCircuit(int[] gas, int[] cost) {
        int totalSum = 0; // 记录总剩余油量
        int curSum = 0;   // 记录当前路程的油量平衡情况
        int start = 0;    // 起始位置索引

        for (int i = 0; i < gas.length; i++) {
            int diff = gas[i] - cost[i];
            totalSum += diff;
            curSum += diff;

            // 如果当前剩余油量小于0，说明从当前记录的start到i之间任何一点出发都行不通
            if (curSum < 0) {
                // 更新起始点为下一个站点
                start = i + 1;
                // 重置当前平衡
                curSum = 0;
            }
        }

        // 如果总油量小于总消耗量，直接返回-1
        if (totalSum < 0) {
            return -1;
        }

        return start;
    }

    public static void main(String[] args) {
        Solution sol = new Solution();
        int[] gas = {1, 2, 3, 4, 5};
        int[] cost = {3, 4, 5, 1, 2};
        System.out.println(sol.canCompleteCircuit(gas, cost)); // 输出 3
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N)$，其中 $N$ 是加油站的数量。我们只需要遍历一遍数组。
- **空间复杂度**：$O(1)$，只使用了常数个额外变量来存储油量总和与起始索引。
