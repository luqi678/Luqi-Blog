## 二、题目描述

有一堆长方体积木，它们的长度和宽度都相同，但长度不一。

小橙想把这堆积木叠成一面墙，墙的每层可以放一个积木，也可以将两个积木拼接起来，要求每层的长度相同。

若必须用完这些积木，叠成的墙最多为多少层？

如下是叠成的一面墙的图示，积木仅按宽和高所在的面进行拼接。

![积木拼接示意图](https://img-blog.csdnimg.cn/img_convert/79491763a19b8893d503612544252654.png)

## 二、输入描述

输入为一行，为各个积木的长度，数字为正整数，并由空格分隔。
积木的数量和长度都不超过 5000。

## 三、输出描述

输出一个数字，为墙的最大层数，如果无法按要求叠成每层长度一致的墙，则输出-1。

## 四、用例

### 用例1

**输入**
```text
3 6 6 3
```

**输出**
```text
3
```

**解释**：
以 6 为每层长度：
- 第一层：6
- 第二层：6
- 第三层：3 + 3
总共 3 层。如果尝试其他长度（如 9），只能叠出 2 层（6+3, 6+3）。题目要求“最多”层数，故输出 3。

### 用例2

**输入**
```text
1 4 2 3 6
```

**输出**
```text
-1
```

**解释**：
总长度为 16。可能的每层长度（必须是 16 的约数且不小于最大值 6）有 8, 16。
- 若长度为 8：可以组合 (6+2)，但剩余 (1, 4, 3) 无法组成 8（虽然 1+4+3=8，但每层最多只能用 2 块积木）。
- 若长度为 16：只能用 (6+4+3+2+1) 组成，超过了每层最多 2 块积木的限制。
因此无法满足要求，输出 -1。

## 五、备注
1. 积木必须全部用完。
2. 每层只能放 1 个或 2 个积木。
3. 墙的每层长度必须相同。

## 六、题目解析

### 1. 题目分析

这道题的关键约束有两个：
1. **必须用完所有积木**。
2. **每层最多只能由 1 个或 2 个积木拼接**。

设所有积木的总长度为 $S$，每层的固定长度为 $L$，墙的总层数为 $H$。
则有数学关系：$S = H \times L$。

由此可以推导出：
- $L$ 必须是 $S$ 的约数。
- $L \ge \max(积木长度)$（因为任何一个积木都得能放下）。
- 由于每层最多 2 个积木，积木总数 $N$ 与层数 $H$ 满足：$H \le N \le 2H$。

### 2. 解题思路

1. **求和与排序**：首先计算所有积木的总长度 $S$，并将积木长度数组 `nums` 进行升序排序。
2. **确定 $L$ 的范围**：
   - 最小可能的 $L$ 是 `nums` 中的最大值 `nums[n-1]`。
   - 最大可能的 $L$ 是 $S$（此时只有 1 层，但由于每层最多 2 个积木，除非积木总数 $\le 2$，否则不可能）。
3. **枚举约数**：
   - 从最小可能的 $L$ 开始枚举 $S$ 的所有约数。
   - 为了求“最大层数”，即求“最小的合法长度 $L$”。
4. **验证合法性**：
   对于每一个可能的 $L$，使用**双指针法**验证是否能将所有积木按 1 个或 2 个一组凑成 $L$：
   - 指针 `left` 指向开头，`right` 指向末尾。
   - 如果 `nums[right] == L`：该积木独立成一层，`right--`，层数计数加 1。
   - 如果 `nums[right] + nums[left] == L`：两个积木凑成一层，`left++`, `right--`，层数计数加 1。
   - 否则：无法按要求凑成长度 $L$，此 $L$ 非法。
   - 验证结束后，如果所有积木都用完了，且总层数 $count$ 确实等于 $S/L$，则该 $L$ 合法。

### 3. 代码实现

#### Java 实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) return;
        String line = sc.nextLine();
        String[] parts = line.trim().split("\\s+");
        int n = parts.length;
        int[] nums = new int[n];
        long sum = 0;
        int maxLen = 0;
        for (int i = 0; i < n; i++) {
            nums[i] = Integer.parseInt(parts[i]);
            sum += nums[i];
            maxLen = Math.max(maxLen, nums[i]);
        }

        // 排序，方便后续双指针校验
        Arrays.sort(nums);

        // 我们要找最大的层数 H = sum / L
        // 也就是要找最小的合法每层长度 L
        // L 必须是 sum 的约数，且 L >= maxLen
        
        int resultH = -1;

        // 枚举每层长度 L
        // 为了让层数最大，L 应该从小到大枚举
        for (int L = maxLen; L <= sum; L++) {
            // 只有当 L 是总长度的约数时才考虑
            if (sum % L == 0) {
                int H = (int) (sum / L);
                
                // 校验在每层长度为 L 的情况下，是否能恰好用完所有积木（每层1-2块）
                if (check(nums, L, H)) {
                    resultH = H;
                    // 找到了最小的 L，对应的就是最大的 H
                    break;
                }
            }
        }

        System.out.println(resultH);
    }

    /**
     * 校验函数：使用双指针判断是否能将 nums 分成 H 组，每组和为 L，每组 1~2 个元素
     */
    private static boolean check(int[] nums, int L, int H) {
        int left = 0;
        int right = nums.length - 1;
        int count = 0;

        while (left <= right) {
            if (nums[right] == L) {
                // 如果最大的积木已经等于 L，单独一层
                right--;
                count++;
            } else if (left < right && nums[left] + nums[right] == L) {
                // 如果最大和最小的能凑成 L，合并一层
                left++;
                right--;
                count++;
            } else {
                // 无法组成 L，说明该 L 不可行
                return false;
            }
        }
        
        // 如果组成的层数等于预期的 H，说明合法
        return count == H;
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：
    1. 排序：$O(N \log N)$，其中 $N \le 5000$。
    2. 计算总和及枚举约数：$S$ 的最大值为 $5000 \times 5000 = 2.5 \times 10^7$。虽然 $S$ 很大，但约数的数量相对较少。
    3. 校验过程：对于每个候选的 $L$，双指针校验只需 $O(N)$。
    4. 整体复杂度大约为 $O(N \log N + \text{约数个数} \times N)$，在 $N=5000$ 的情况下性能绰绰有余。

- **空间复杂度**：$O(N)$，用于存储积木长度数组。
