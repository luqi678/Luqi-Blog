## 一、 题目描述

现在你要竞选一个县的县长。你去对每一个选民进行了调查。你已经知道每一个人要选的人是谁，以及要花多少钱才能让这个人选你。

现在你想要花最少的钱使得你当上县长。你当选的条件是你的票数比任何一个其它候选人的多（严格的多，不能和他们中最多的相等）。

请计算一下最少要花多少钱。你是 0 号候选人。

## 二、输入描述

- 第一行有一个整数 $n$ ($1 \le n \le 10^5$)，表示这个县的选民数目。
- 接下来有 $n$ 行，每一行有两个整数 $a_i$ 和 $b_i$ ($0 \le a_i \le 10^5; 0 \le b_i \le 10^4$)，表示第 $i$ 个选民选的是第 $a_i$ 号候选人，想要让他选择自己就要花 $b_i$ 的钱。
- 如果一个选民原本就选你（$a_i = 0$），则 $b_i$ 必定为 0。

## 三、输出描述

- 输出一个整数表示花费的最少的钱。

## 四、用例

### 用例1

**输入：**
```text
5
1 2
1 2
1 2
2 1
0 0
```

**输出：**
```text
3
```

**说明：**
原本 0 号候选人 1 票，1 号 3 票，2 号 1 票。
为了赢，如果 0 号想拿到 3 票：
1. 从 1 号那里买 1 个人（花费 2），此时 0 号 2 票，1 号 2 票。还不满足“严格大于”。
2. 再从 2 号那里买 1 个人（花费 1），此时 0 号 3 票，1 号 2 票，2 号 0 票。
总花费 $2 + 1 = 3$。

## 五、备注

$n \le 10^5$，收买费用 $b_i \le 10^4$。需要考虑高效的算法，直接暴力枚举收买组合会超时。

## 六、题目解析

### 1. 题目分析

本题的核心是找到一个目标票数 $x$，使得 0 号候选人获得 $x$ 票时，其他所有候选人的票数都小于 $x$。
- 如果我们固定 0 号候选人最终的票数为 $x$：
    1. 所有当前票数 $\ge x$ 的候选人，必须被收买走一部分投票者，直到他们的票数降到 $x-1$。为了省钱，我们肯定收买这些候选人支持者中最便宜的那些。
    2. 执行完上述步骤后，如果 0 号候选人的总票数还不足 $x$，我们需要从剩余的所有选民（尚未被收买的）中挑选最便宜的，补齐到 $x$ 票。
- 由于最终票数 $x$ 可能的范围是 $1$ 到 $n$，我们可以枚举 $x$。

### 2. 解题思路

如果直接枚举 $x$ 从 $1$ 到 $n$ 并每次重新计算，复杂度是 $O(n^2)$，会超时。我们需要优化：
1. **逆向处理**：考虑 $x$ 从 $n$ 递减到 $1$。
2. **分类维护**：
    - 使用一个数组 `List<Integer>[] candidateVoters` 存储每个候选人的所有支持者的收买价格，并从小到大排序。
    - 使用一个数据结构（如**权值线段树**或**树状数组**）来维护当前“未被强制收买”的所有选民的价格。线段树可以快速查询：前 $k$ 个最便宜的选民的总花费。
3. **状态更新**：
    - 当目标票数 $x$ 减小时，原本不需要强制收买的选民（因为所属候选人票数未达标）现在可能需要被强制收买。
    - 强制收买的选民从线段树中移除。
4. **计算代价**：
    - 强制收买代价 = 满足“所有其他候选人票数 $< x$”所需的最小花费。
    - 总代价 = 强制收买代价 + 线段树中补齐到 $x$ 票所需的最小花费。

### 3. 代码实现 (Java)

```java
import java.io.*;
import java.util.*;

public class Main {
    static class SegmentTree {
        int n;
        long[] count;
        long[] sum;

        SegmentTree(int n) {
            this.n = n;
            count = new long[4 * n + 1];
            sum = new long[4 * n + 1];
        }

        void update(int pos, int val, int node, int l, int r) {
            if (l == r) {
                count[node] += val;
                sum[node] += (long) val * l;
                return;
            }
            int mid = (l + r) / 2;
            if (pos <= mid) update(pos, val, 2 * node, l, mid);
            else update(pos, val, 2 * node + 1, mid + 1, r);
            count[node] = count[2 * node] + count[2 * node + 1];
            sum[node] = sum[2 * node] + sum[2 * node + 1];
        }

        long query(long k, int node, int l, int r) {
            if (k <= 0) return 0;
            if (l == r) return k * l;
            int mid = (l + r) / 2;
            if (count[2 * node] >= k) {
                return query(k, 2 * node, l, mid);
            } else {
                return sum[2 * node] + query(k - count[2 * node], 2 * node + 1, mid + 1, r);
            }
        }
    }

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine());

        List<Integer>[] voters = new List[100001];
        for (int i = 0; i <= 100000; i++) voters[i] = new ArrayList<>();

        int baseVotes = 0;
        int maxPrice = 10000;
        SegmentTree st = new SegmentTree(maxPrice);

        for (int i = 0; i < n; i++) {
            String[] parts = br.readLine().split(" ");
            int a = Integer.parseInt(parts[0]);
            int b = Integer.parseInt(parts[1]);
            if (a == 0) {
                baseVotes++;
            } else {
                voters[a].add(b);
                st.update(b, 1, 1, 0, maxPrice);
            }
        }

        // 记录每个候选人的票数，并排序候选人
        List<List<Integer>> activeCandidates = new ArrayList<>();
        for (int i = 1; i <= 100000; i++) {
            if (!voters[i].isEmpty()) {
                Collections.sort(voters[i]);
                activeCandidates.add(voters[i]);
            }
        }
        
        // 按照选民数量分组，便于枚举目标票数 x
        List<Integer>[] sizeGroups = new List[n + 1];
        for (int i = 0; i <= n; i++) sizeGroups[i] = new ArrayList<>();
        for (int i = 0; i < activeCandidates.size(); i++) {
            sizeGroups[activeCandidates.get(i).size()].add(i);
        }

        long minTotalCost = Long.MAX_VALUE;
        long mandatoryCost = 0;
        int mandatoryCount = 0;
        
        // candidateNextIdx[i] 表示第 i 个候选人已经被强制收买了多少最便宜的投票者
        int[] candidateNextIdx = new int[activeCandidates.size()];
        // 记录哪些候选人当前票数 >= x
        Set<Integer> largeCandidates = new HashSet<>();

        // x 从 n 降到 1
        for (int x = n; x >= 1; x--) {
            // 所有票数达到 x 的候选人，必须收买掉一个最便宜的
            for (int candIdx : sizeGroups[x]) {
                largeCandidates.add(candIdx);
            }
            
            for (int candIdx : largeCandidates) {
                List<Integer> prices = activeCandidates.get(candIdx);
                // 如果该候选人当前票数 >= x，则需要收买到只剩 x-1 票
                // 随着 x 减小，需要强制收买的选民会增加
                while (prices.size() - candidateNextIdx[candIdx] >= x) {
                    int price = prices.get(candidateNextIdx[candIdx]);
                    mandatoryCost += price;
                    mandatoryCount++;
                    st.update(price, -1, 1, 0, maxPrice);
                    candidateNextIdx[candIdx]++;
                }
            }

            int currentVotes = baseVotes + mandatoryCount;
            long currentTotalCost = mandatoryCost;
            if (currentVotes < x) {
                currentTotalCost += st.query(x - currentVotes, 1, 0, maxPrice);
            }
            minTotalCost = Math.min(minTotalCost, currentTotalCost);
        }

        System.out.println(minTotalCost);
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：
    - 排序每个候选人的支持者：$O(n \log n)$。
    - 枚举目标票数 $x$ 从 $n$ 到 1：$O(n)$。
    - 线段树更新与查询：$O(\log(\max\_price))$。
    - 每个选民最多被移出线段树一次，所以强制收买部分的总复杂度是 $O(n \log(\max\_price))$。
    - 总复杂度 $O(n \log n + n \log(\max\_price))$，在 $10^5$ 规模下可以轻松通过。
- **空间复杂度**：
    - $O(n + \max\_price)$ 用于存储选民信息和线段树。
