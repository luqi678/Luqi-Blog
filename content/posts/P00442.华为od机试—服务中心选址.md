## 一、 题目描述

一家快递公司希望在一条街道建立新的服务中心。公司统计了该街道中所有区域在地图上的位置，并希望能够以此为依据为新的服务中心选址：使服务中心到所有区域的距离的总和最小。

给你一个数组 `positions`，其中 `positions[i] = [left, right]` 表示第 $i$ 个区域在街道上的位置，其中 `left` 代表区域的左侧起点，`right` 表示区域的右侧终点。设选择服务中心的位置为 `location`。

*   如果第 $i$ 个区域的右侧终点 `right` 满足 `right < location`，则第 $i$ 个区域到服务中心的距离为 `location - right`；
*   如果第 $i$ 个区域的左侧起点 `left` 满足 `left > location`，则第 $i$ 个区域到服务中心的距离为 `left - location`；
*   如果第 $i$ 个区域满足 `left <= location <= right`，则第 $i$ 个区域到服务中心的距离为 0。

选择最佳的服务中心位置 `location`，请返回最佳的服务中心位置到所有区域的距离总和的最小值。

## 二、输入描述

第一行，一个整数 $N$ 表示区域个数。
后面 $N$ 行，每行两个整数，表示区域的左右起点终点。

## 三、输出描述

运行结果输出一个整数，表示服务中心位置到所有区域的距离总和的最小值。

## 四、用例

### 用例1

**输入**
```
3
1 2
3 4
10 20
```

**输出**
```
8
```

### 用例2

**输入**
```
2
1 5
10 15
```

**输出**
```
5
```

## 五、备注
1. $N \le 10^5$
2. 坐标范围在 $[-10^9, 10^9]$ 之间。

## 六、题目解析

### 1. 题目分析

本题的核心是寻找一个点 $x$（即服务中心的位置），使得它到 $N$ 个闭区间 $[L_i, R_i]$ 的距离之和最小。
距离定义为：
- 若 $x < L_i$，距离为 $L_i - x$
- 若 $x > R_i$，距离为 $x - R_i$
- 若 $L_i \le x \le R_i$，距离为 $0$

这个函数可以表示为 $f_i(x) = \max(0, L_i - x, x - R_i)$。
目标是最小化 $F(x) = \sum_{i=1}^{N} f_i(x)$。

### 2. 解题思路

1.  **凸函数性质**：
    每个 $f_i(x)$ 都是一个凸函数（Convex Function）。多个凸函数之和 $F(x)$ 仍然是一个凸函数。
    对于凸函数，我们可以使用**三分法（Ternary Search）**来寻找其最小值。由于题目要求的坐标范围非常大，且可能是连续的值，三分法非常适合。

2.  **三分法的范围**：
    服务中心的最优位置一定在所有区间覆盖的范围之内，即 $[\min(L_i), \max(R_i)]$。

3.  **关于整数解**：
    虽然距离函数在区间端点处斜率会变化，但由于 $F(x)$ 是分段线性的且每一段的斜率都是整数，最小值点处 $x$ 虽然可能是某个区间，但总存在一个整数位置使距离和达到最小。我们可以直接对整数坐标进行三分，或者对浮点数三分后检查周围整数。

4.  **更高效的观察**：
    如果我们将每个区间视为两个点 $L_i$ 和 $R_i$，并将所有 $2N$ 个端点排序。可以证明，最优的 $x$ 一定在这些端点之间。实际上，这个问题是“货仓选址”问题（中位数问题）的一个变种。

### 3. 代码实现

我们将采用**三分法**来寻找最小值，这种方法逻辑通用且容易实现。

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        long[][] positions = new long[n][2];
        
        long leftBoundary = Long.MAX_VALUE;
        long rightBoundary = Long.MIN_VALUE;
        
        for (int i = 0; i < n; i++) {
            positions[i][0] = sc.nextLong(); // left
            positions[i][1] = sc.nextLong(); // right
            leftBoundary = Math.min(leftBoundary, positions[i][0]);
            rightBoundary = Math.max(rightBoundary, positions[i][1]);
        }
        
        // 三分法寻找凸函数最小值
        long low = leftBoundary;
        long high = rightBoundary;
        
        while (high - low > 2) {
            long m1 = low + (high - low) / 3;
            long m2 = high - (high - low) / 3;
            if (calculateDistance(positions, m1) < calculateDistance(positions, m2)) {
                high = m2;
            } else {
                low = m1;
            }
        }
        
        // 在剩余范围内找最小值
        long minDistance = calculateDistance(positions, low);
        for (long i = low + 1; i <= high; i++) {
            minDistance = Math.min(minDistance, calculateDistance(positions, i));
        }
        
        System.out.println(minDistance);
    }
    
    // 计算位置 x 到所有区域的距离总和
    private static long calculateDistance(long[][] positions, long x) {
        long total = 0;
        for (long[] range : positions) {
            if (x < range[0]) {
                total += (range[0] - x);
            } else if (x > range[1]) {
                total += (x - range[1]);
            }
        }
        return total;
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(N \cdot \log(\text{Range}))$，其中 $N$ 是区域数量，Range 是坐标范围（约 $2 \times 10^9$）。$\log_{1.5}(2 \times 10^9) \approx 53$，因此计算量大约为 $53 \times 10^5$，在 1 秒限制内可以通过。
*   **空间复杂度**：$O(N)$，用于存储区域的位置信息。
