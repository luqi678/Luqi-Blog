## 一、 题目描述

给定一个整数数组 `asteroids`，表示在同一条直线上移动的小行星。对于数组中的每个小行星，其位置按从左到右排列。

小行星之间的碰撞规则如下：
1. **移动方向**：正数表示向右移动，负数表示向左移动。
2. **碰撞结果**：
   - 如果两个小行星迎面相撞（即左侧行星向右 `+`，右侧行星向左 `-`），大小更大的一方获胜，较小的一方被摧毁。
   - 如果两个小行星大小相等，那么双方都被摧毁。
   - 如果小行星移动方向相同，或者由于方向原因无法相遇（如左侧向左，右侧向右），则不会发生碰撞。

请你模拟小行星碰撞过程，并返回碰撞结束后剩余的小行星数组。

## 二、输入描述

*   一个整数数组 `asteroids`，长度范围为 `[1, 10000]`。
*   数组元素 `asteroids[i]` 范围为 `[-1000, 1000]`，且不为 `0`。

## 三、输出描述

*   返回一个表示最终剩余小行星状态的整数数组。

## 四、用例

### 用例1
输入：`asteroids = [5, 10, -5]`
输出：`[5, 10]`
解释：10 和 -5 碰撞，10 获胜，-5 消失。5 和 10 方向相同，不碰撞。

### 用例2
输入：`asteroids = [8, -8]`
输出：`[]`
解释：8 和 -8 迎面相撞，大小相等，全部摧毁。

### 用例3
输入：`asteroids = [10, 2, -5]`
输出：`[10]`
解释：2 和 -5 碰撞，-5 获胜，2 消失；接着 10 和 -5 碰撞，10 获胜，-5 消失。

### 用例4
输入：`asteroids = [-2, -1, 1, 2]`
输出：`[-2, -1, 1, 2]`
解释：左侧的向左走，右侧的向右走，永不相撞。

## 五、备注
*   碰撞只发生在 `stack` 顶为正数（向右）且当前行星为负数（向左）的情况下。

## 六、题目解析

### 1. 题目分析
本题的核心在于模拟。由于行星是从左到右依次处理的，且碰撞具有“链式反应”（比如一个向左的大行星可能连续撞碎左边多个向右的小行星），这非常符合**栈（Stack）**的特性：后进先出。

### 2. 解题思路
1.  遍历 `asteroids` 数组。
2.  对于当前行星 `ast`：
    *   如果 `ast > 0`：它是向右移动的，暂时不会和左边的行星碰撞，直接压入栈。
    *   如果 `ast < 0`：它是向左移动的。
        *   只要栈顶行星是向右移动的（`stack.peek() > 0`），就会发生碰撞。
        *   若 `stack.peek() < abs(ast)`：栈顶行星较小，被粉碎，弹出栈顶，继续与新栈顶比较。
        *   若 `stack.peek() == abs(ast)`：两者同归于尽，弹出栈顶，标记当前行星 `ast` 已消失。
        *   若 `stack.peek() > abs(ast)`：当前行星 `ast` 被粉碎，标记 `ast` 已消失。
    *   如果当前行星 `ast` 经历了一轮碰撞后依然“生存”（没有被撞碎），且没有遇到能阻挡它的（如栈为空或栈顶也是向左走的），则入栈。

### 3. 代码实现

#### Java8实现

```java
import java.util.Stack;

class Solution {
    public int[] asteroidCollision(int[] asteroids) {
        Stack<Integer> stack = new Stack<>();

        for (int ast : asteroids) {
            boolean alive = true;
            // 只有 栈顶向右(>0) 且 当前行星向左(<0) 才会碰撞
            while (alive && ast < 0 && !stack.isEmpty() && stack.peek() > 0) {
                int top = stack.peek();
                if (top < -ast) {
                    // 栈顶行星被撞碎，继续循环比较下一个栈顶
                    stack.pop();
                } else if (top == -ast) {
                    // 两者同归于尽
                    stack.pop();
                    alive = false;
                } else {
                    // 当前行星被撞碎
                    alive = false;
                }
            }
            
            // 如果经历碰撞后还活着，入栈
            if (alive) {
                stack.push(ast);
            }
        }

        // 将栈中元素转为数组
        int[] res = new int[stack.size()];
        for (int i = res.length - 1; i >= 0; i--) {
            res[i] = stack.pop();
        }
        return res;
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(N)$，其中 $N$ 是小行星的数量。每个小行星最多入栈一次，出栈一次。
*   **空间复杂度**：$O(N)$，最坏情况下所有行星都不碰撞，需要存储在栈中。
