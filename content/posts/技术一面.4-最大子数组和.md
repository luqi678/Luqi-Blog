## 一、 题目描述

给定一个整数数组 `nums` ，找到一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

**连续子数组** 是数组中的一个连续部分。

## 二、输入描述

一个整数数组 `nums`。
在面试手撕场景下，通常表现为函数入参 `int[] nums`。

## 三、输出描述

返回一个整数，表示子数组的最大和。

## 四、用例

### 用例1
输入: `[-2,1,-3,4,-1,2,1,-5,4]`  
输出: `6`  
解释: 连续子数组 `[4,-1,2,1]` 的和最大，为 `6`。

### 用例2
输入: `[1]`  
输出: `1`  

### 用例3
输入: `[5,4,-1,7,8]`  
输出: `23`  

## 五、备注

1. 数组长度范围：`1 <= nums.length <= 10^5`
2. 数组元素范围：`-10^4 <= nums[i] <= 10^4`

## 六、题目解析

### 1. 题目分析
本题是经典的“最大子序和”问题。题目要求的是**连续**的子数组。如果数组中全部是正数，那么最大和就是所有元素之和；但由于存在负数，我们需要决定在什么时候“抛弃”之前的累加结果，重新开始计算。

### 2. 解题思路
常用的解法有两种：**动态规划 (Dynamic Programming)** 和 **贪心算法 (Kadane's Algorithm)**。两者的核心思想在本质上是一致的。

*   **状态定义**：设 `dp[i]` 为以第 `i` 个元素结尾的连续子数组的最大和。
*   **转移方程**：对于 `nums[i]`，有两种选择：
    1.  加入前面的子数组：`dp[i-1] + nums[i]`
    2.  不加入前面的子数组，以当前元素作为新子数组的起始：`nums[i]`
    *   即：`dp[i] = Math.max(nums[i], dp[i-1] + nums[i])`
*   **优化**：由于 `dp[i]` 仅取决于 `dp[i-1]`，我们可以使用一个变量 `currentSum` 来记录当前最大和，从而将空间复杂度降至 $O(1)$。

### 3. 代码实现

#### Java8实现

```java
public class Solution {
    public int maxSubArray(int[] nums) {
        // 如果数组为空，根据规范处理
        if (nums == null || nums.length == 0) {
            return 0;
        }

        // 初始化结果为第一个元素
        int max_sum = nums[0];
        // 记录以当前元素结尾的最大子数组和
        int current_sum = nums[0];

        // 从第二个元素开始遍历
        for (int i = 1; i < nums.length; i++) {
            // 如果 current_sum 小于 0，说明对后面没有贡献，直接抛弃
            // 否则 current_sum = current_sum + nums[i]
            current_sum = Math.max(nums[i], current_sum + nums[i]);
            
            // 更新全局最大值
            max_sum = Math.max(max_sum, current_sum);
        }

        return max_sum;
    }

    public static void main(String[] args) {
        Solution sol = new Solution();
        int[] nums1 = {-2, 1, -3, 4, -1, 2, 1, -5, 4};
        System.out.println("Case 1: " + sol.maxSubArray(nums1)); // Expected: 6

        int[] nums2 = {5, 4, -1, 7, 8};
        System.out.println("Case 2: " + sol.maxSubArray(nums2)); // Expected: 23
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(n)$，其中 $n$ 是数组的长度。我们只需要遍历一遍数组。
*   **空间复杂度**：$O(1)$，我们只使用了常数个额外空间来存储当前最大和以及全局最大和。
