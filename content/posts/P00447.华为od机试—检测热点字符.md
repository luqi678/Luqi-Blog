## 一、 题目描述

某 web 服务器会持续接收文本字符流。已知文本字符流仅由大写英文字母 (A-Z)、小写英文字母 (a-z)、数字 (0-9) 三种组成。

为了持续观测其中的热点字符，需要每隔一段字符长度就进行一次增量统计（假设增量统计单位为 $M$，则每接收 $M$ 个字符，就要对当前已接收的所有字符串的热点字符进行统计一次）。

每次的统计结果按**出现次数从高到低**排列，对于出现次数相同的字符则按 **ASCII 码值从大到小**排列。

**注意：**
- 统计是**累加**的。第二次统计的是前 $2M$ 个字符，第三次是前 $3M$ 个字符，以此类推。
- 每次统计输出前 $N$ 个符合条件的字符。
- 输入字符串最大长度为 2,000,000。
- 区分大小写。

## 二、输入描述

每个输入数据为一行，数据之间以 1 个空格间隔。
1. 第 1 个数字 $N$：表示依次统计出现最多的 $N$ 个字符。
2. 第 2 个数字 $M$：表示每 $M$ 个字符进行一次增量统计。
3. 第 3 个是需要统计的字符串 $S$。

## 三、输出描述

输出每次统计结果合并后的字符串。

## 四、用例

### 用例1
**输入**
```text
3 10 aaagggbcce9999333213gggggaaaaa
```
**输出**
```text
gac93gga9
```

### 用例2
**输入**
```text
1 10 1234567890
```
**输出**
```text
9
```

### 用例3
**输入**
```text
5 10 1234567890
```
**输出**
```text
98765
```

## 五、备注
- 输入保证第一次统计时（即前 $M$ 个字符中）有足够的不同字符（即不同字符种类数 $\ge N$）。
- 字符集仅包含 `[a-z, A-Z, 0-9]`。

## 六、题目解析

### 1. 题目分析
题目的核心在于“增量”和“排序”。
*   **增量**：意味着统计是累计的。如果 $M=10$，第一次统计范围是 $[0, 9]$，第二次是 $[0, 19]$，第三次是 $[0, 29]$。
*   **排序规则**：
    1.  次数（Frequency）从大到小。
    2.  如果次数相同，ASCII 码从大到小。
*   **输出**：每一轮统计出的 $N$ 个字符按顺序拼接成最终的字符串。

### 2. 解题思路
1.  **数据结构**：
    *   使用一个长度为 128 的整型数组 `cnt` 来记录每个 ASCII 字符出现的次数（因为字符仅限数字和字母，128 足够）。
    *   使用一个 `List` 存储当前出现过的字符，以便进行排序。
2.  **流程控制**：
    *   读取 $N, M, S$。
    *   遍历字符串 $S$，每处理 $M$ 个字符，就触发一次统计逻辑。
    *   统计逻辑：
        *   将当前有计数的字符放入 List。
        *   利用自定义比较器（Comparator）对 List 进行排序。
        *   取前 $N$ 个字符追加到结果集中。
3.  **效率优化**：
    *   由于字符串长度可能达 $2,000,000$，使用 `Scanner` 可能较慢，建议使用 `StringTokenizer` 或直接 `split` 处理输入（对于本题 $N, M$ 较小，重点在 $S$）。
    *   字符种类固定（最多 62 种：10数字 + 26小写 + 26大写），因此每次排序的列表长度非常短，性能瓶颈主要在字符串的遍历上。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.訊);
        if (!sc.hasNext()) return;

        int n = sc.nextInt();
        int m = sc.nextInt();
        String s = sc.next();

        int len = s.length();
        int[] counts = new int[128]; // 记录字符出现次数
        StringBuilder result = new StringBuilder();

        // 每次步进 M 个字符
        for (int i = 0; i < len; i += m) {
            // 将当前窗口内的 M 个字符加入统计
            for (int j = i; j < i + m && j < len; j++) {
                counts[s.charAt(j)]++;
            }

            // 统计当前所有的字符
            List<CharInfo> charList = new ArrayList<>();
            for (int k = 0; k < 128; k++) {
                if (counts[k] > 0) {
                    charList.add(new CharInfo((char) k, counts[k]));
                }
            }

            // 按照次数降序，ASCII降序排列
            Collections.sort(charList, (a, b) -> {
                if (a.count != b.count) {
                    return b.count - a.count;
                }
                return b.ch - a.ch;
            });

            // 取前 N 个
            for (int k = 0; k < n && k < charList.size(); k++) {
                result.append(charList.get(k).ch);
            }
        }

        System.out.println(result.toString());
    }

    static class CharInfo {
        char ch;
        int count;

        CharInfo(char ch, int count) {
            this.ch = ch;
            this.count = count;
        }
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(L + \frac{L}{M} \cdot D \log D)$
    *   $L$ 是字符串长度（$2 \times 10^6$）。
    *   $\frac{L}{M}$ 是统计次数。
    *   $D$ 是不同字符的个数（最大为 62）。
    *   由于 $D$ 是常数，排序时间几乎可以忽略不计。主要的耗时在遍历字符串 $L$。
*   **空间复杂度**：$O(D + \text{ResultLength})$
    *   需要存储 128 长度的计数数组和长度为 $D$ 的列表。
    *   结果字符串的长度为 $(L/M) \times N$。
