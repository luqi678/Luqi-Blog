## 一、 题目描述

实现一个简单的绘图模块，绘图模块仅支持矩形的绘制和擦除。

* 当新绘制的矩形与已有图形重叠时，对图形取 **并集**；
* 当新擦除的矩形与已有图形重叠时，对图形取 **差集**。

给定一系列绘制和擦除操作，计算最终图形的面积。

## 二、输入描述

第一行为操作数 $N$，接下来的 $N$ 行格式为：

1.  `d x1 y1 x2 y2`：`d` 表示绘制操作，$(x1, y1)$ 为矩形左上角坐标，$(x2, y2)$ 为右下角坐标；
2.  `e x1 y1 x2 y2`：`e` 表示擦除操作，坐标含义同上。

坐标均为整数且范围在 $[-100, 100]$ 内，用例保证坐标有效性。

## 三、输出描述

输出最终图形的面积。

## 四、用例

### 示例 1

**输入**

```text
2
d 0 2 2 0
d -1 1 1 -1
```

**输出**

```text
7
```

**说明**

1.  第一个操作 `d 0 2 2 0` 绘制了一个 $2 \times 2$ 的矩形（面积 4）。
2.  第二个操作 `d -1 1 1 -1` 绘制了另一个 $2 \times 2$ 的矩形。
3.  两矩形重叠区域为 $1 \times 1$（面积 1）。
4.  最终面积 = $4 + 4 - 1 = 7$。

### 示例 2

**输入**

```text
2
d 0 2 2 0
e -1 1 1 -1
```

**输出**

```text
3
```

**说明**

1.  第一个操作 `d 0 2 2 0` 绘制面积为 4。
2.  第二个操作 `e -1 1 1 -1` 擦除了一个区域。
3.  两矩形重叠区域为 1，这部分被擦除。
4.  最终面积 = $4 - 1 = 3$。

## 五、备注

1.  坐标范围较小 $([-100, 100])$，这提示我们可以使用 **二维数组（网格）模拟** 的方法来解决问题。
2.  题目中的坐标 $(x1, y1)$ 和 $(x2, y2)$ 定义了矩形的对角线，计算时需要处理大小关系（即取 `min` 和 `max`）。

## 六、题目解析

### 1. 核心思路：像素网格模拟
由于坐标范围仅在 $[-100, 100]$ 之间，整个平面的跨度只有 200。我们可以将二维平面看作是由 $1 \times 1$ 的小方格（像素）组成的网格。

* 创建一个二维布尔数组 `grid`。
* **坐标偏移**：由于数组下标不能为负数，我们需要给所有坐标加上一个偏移量（例如 `+100`），将区间 $[-100, 100]$ 映射到 $[0, 200]$。
* **绘制 (`d`)**：将矩形覆盖范围内的 `grid` 单元格设置为 `true`。
* **擦除 (`e`)**：将矩形覆盖范围内的 `grid` 单元格设置为 `false`。
* **统计**：最后遍历整个数组，统计 `true` 的个数即为面积。

### 2. 细节处理
* **坐标转换**：
    矩形由线段组成，例如 $x$ 轴从 0 到 2，包含了区间 $[0, 1]$ 和 $[1, 2]$ 两个单位长度。对应数组下标处理应该是 `for (int i = minX; i < maxX; i++)`（注意这里是 `<` 而不是 `<=`，因为点 $x=2$ 是边界线，不包含第 3 个格子）。
* **数组大小**：
    坐标范围 $[-100, 100]$，加上偏移量 `100` 后范围是 $[0, 200]$。数组大小定义为 `205 * 205` 足够容纳。

### 3. Java 代码实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        // 读取操作数
        if (!sc.hasNext()) return;
        int n = sc.nextInt();

        // 定义网格
        // 坐标范围 -100 到 100，跨度 200
        // 使用偏移量 100，将坐标映射到 0~200
        // 数组大小设为 205 足够
        boolean[][] grid = new boolean[205][205];
        int offset = 100;

        for (int i = 0; i < n; i++) {
            String op = sc.next();
            int x1 = sc.nextInt();
            int y1 = sc.nextInt();
            int x2 = sc.nextInt();
            int y2 = sc.nextInt();

            // 1. 规范化坐标，确定循环范围
            // 无论题目给的是左上还是右下，先确定 min 和 max
            int minX = Math.min(x1, x2) + offset;
            int maxX = Math.max(x1, x2) + offset;
            int minY = Math.min(y1, y2) + offset;
            int maxY = Math.max(y1, y2) + offset;

            // 2. 根据操作类型决定填充值
            boolean isDraw = op.equals("d");

            // 3. 填充网格
            // 注意：x 坐标从 minX 到 maxX 表示长度为 maxX - minX
            // 对应的网格索引是 [minX, maxX - 1]
            for (int x = minX; x < maxX; x++) {
                for (int y = minY; y < maxY; y++) {
                    grid[x][y] = isDraw;
                }
            }
        }

        // 4. 统计面积
        int area = 0;
        for (int i = 0; i < grid.length; i++) {
            for (int j = 0; j < grid[0].length; j++) {
                if (grid[i][j]) {
                    area++;
                }
            }
        }

        System.out.println(area);
    }
}
```

### 4. 复杂度分析
* **时间复杂度**：$O(N \times W \times H)$，其中 $N$ 是操作数，$W$ 和 $H$ 是坐标跨度（本题为 200）。因为 $W$ 和 $H$ 很小，这是一个非常高效的常数级复杂度。
* **空间复杂度**：$O(W \times H)$，用于存储网格，约 $200 \times 200 = 40000$ 个 boolean 值，内存占用极小。
