## 一、 题目描述

现有一个 CPU 和一些任务需要处理，已提前获知每个任务的任务 ID、优先级、所需执行时间和到达时间。 CPU 同时只能运行一个任务，请编写一个任务调度程序，采用“可抢占优先权调度”调度算法进行任务调度，规则如下：

1. 如果一个任务到来时，CPU 是空闲的，则 CPU 可以运行该任务直到任务执行完毕。但是如果运行中有一个更高优先级的任务到来，则 CPU 必须暂停当前任务去运行这个优先级更高的任务；
2. 如果一个任务到来时，CPU 正在运行一个比他优先级更高的任务时，新到达的任务必须等待；
3. 当 CPU 空闲时，如果还有任务在等待，CPU 会从这些任务中选择一个优先级最高的任务执行，相同优先级的任务选择到达时间最早的任务。

## 二、输入描述

输入有若干行，每一行有四个数字（均小于 $10^8$），分别为任务 ID，任务优先级，执行时间，到达时间。
- 每个任务的任务 ID 不同。
- 优先级数字越大优先级越高。
- 相同优先级的任务不会同时到达。
- 输入的任务已按照到达时间从小到大排序。
- 保证在任何时间，处于等待的任务不超过 10000 个。

## 三、输出描述

按照任务执行结束的顺序，输出任务的 ID 和完成时间，中间用空格隔开。

## 四、用例

### 用例1

**输入**
```text
1 3 5 1
2 1 5 10
3 2 7 12
4 3 2 20
5 4 9 21
6 4 2 22
```

**输出**
```text
1 6
3 19
5 30
6 32
4 33
2 35
```

## 五、备注
- 时间数值可能较大，建议使用 `long` 类型。
- 重点在于如何处理“抢占”逻辑：每当一个新任务在 `newArrivalTime` 到达时，我们需要计算 CPU 从当前时间 `currentTime` 到 `newArrivalTime` 这段时间内能完成多少工作。

## 六、题目解析

### 1. 题目分析
本题属于经典的操作系统**可抢占优先权调度算法**模拟。核心点在于：
- **优先级**：数字越大，优先级越高。
- **抢占**：高优先级任务到来时，如果当前正在执行的任务优先级较低，则立即中断当前任务，转而执行高优先级任务。
- **等待队列**：当 CPU 忙碌或任务被中断时，任务进入等待队列。队列内部按优先级（大到小）和到达时间（小到大）排序。

### 2. 解题思路
1.  **数据结构**：使用一个 `PriorityQueue` (大顶堆) 来存储等待执行的任务。排序规则：优先级从大到小；优先级相同时，到达时间从小到大。
2.  **模拟过程**：
    -   记录一个当前时间 `currentTime`。
    -   依次读取每一个新任务 `newTask`：
        -   在 `newTask` 到达之前（即 `currentTime` 到 `newTask.arrivalTime` 之间），CPU 会一直尝试从 `PriorityQueue` 中取出最高优先级的任务进行执行。
        -   如果取出任务的剩余执行时间小于等于这段空闲时间，则该任务能执行完，更新 `currentTime`，输出结果。
        -   如果取出任务的剩余执行时间大于这段空闲时间，则该任务只能执行一部分，更新其剩余时间并重新放回队列，同时将 `currentTime` 更新为 `newTask.arrivalTime`。
    -   将 `newTask` 加入 `PriorityQueue`。
    -   注意：如果读取新任务时，`currentTime` 小于任务到达时间且队列为空，说明 CPU 中间有一段纯空闲时间，直接将 `currentTime` 跳到新任务到达时间。
3.  **收尾**：输入结束后，依次清空 `PriorityQueue` 中的任务，并累加时间输出。

### 3. 代码实现

#### Java实现

```java
import java.util.*;

public class Main {
    static class Task {
        int id;
        int priority;
        long remainingTime;
        long arrivalTime;

        public Task(int id, int priority, long remainingTime, long arrivalTime) {
            this.id = id;
            this.priority = priority;
            this.remainingTime = remainingTime;
            this.arrivalTime = arrivalTime;
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<Task> inputTasks = new ArrayList<>();
        
        // 读取所有输入任务
        while (sc.hasNextInt()) {
            int id = sc.nextInt();
            int priority = sc.nextInt();
            long duration = sc.nextLong();
            long arrival = sc.nextLong();
            inputTasks.add(new Task(id, priority, duration, arrival));
        }

        // 优先队列：优先级高者优先，同优先级到达早者优先
        PriorityQueue<Task> pq = new PriorityQueue<>((a, b) -> {
            if (a.priority != b.priority) {
                return b.priority - a.priority;
            }
            return Long.compare(a.arrivalTime, b.arrivalTime);
        });

        long currentTime = 0;
        
        for (Task newTask : inputTasks) {
            // 处理当前时间到新任务到达时间之间的任务执行
            while (!pq.isEmpty() && currentTime < newTask.arrivalTime) {
                Task top = pq.poll();
                long availableTime = newTask.arrivalTime - currentTime;
                
                if (top.remainingTime <= availableTime) {
                    // 能在下一个任务到达前完成
                    currentTime += top.remainingTime;
                    System.out.println(top.id + " " + currentTime);
                } else {
                    // 不能完成，执行一部分后放回
                    top.remainingTime -= availableTime;
                    currentTime = newTask.arrivalTime;
                    pq.add(top);
                }
            }
            
            // 如果 CPU 空闲且当前时间还没到新任务到达时间，跳到新任务时间
            if (currentTime < newTask.arrivalTime) {
                currentTime = newTask.arrivalTime;
            }
            
            // 将新任务加入等待队列
            pq.add(newTask);
        }

        // 处理剩余队列中的任务
        while (!pq.isEmpty()) {
            Task top = pq.poll();
            currentTime += top.remainingTime;
            System.out.println(top.id + " " + currentTime);
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N \log N)$，其中 $N$ 是任务数量。每个任务最多入队和出队一次，优先队列的操作复杂度为 $O(\log N)$。
- **空间复杂度**：$O(N)$，用于存储任务列表和优先队列（题目保证等待任务不超过 10000，但在极端情况下 $N$ 个任务可能都在队列中）。
