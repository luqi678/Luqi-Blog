## 二、题目描述

当前 IT 部门支撑了子公司颗粒化业务，该部门需要实现为子公司快速开租建站的能力。建站是指在一个全新的环境部署一套 IT 服务。

1. 每个站点开站会由一系列部署任务项构成，每个任务项部署完成时间都是固定和相等的，设为 1。
2. 部署任务项之间可能存在依赖，假如任务 2 依赖任务 1，那么等任务 1 部署完，任务 2 才能部署。
3. 任务有多个依赖任务则需要等所有依赖任务都部署完该任务才能部署。
4. 没有依赖的任务可以并行部署，优秀的员工们会做到完全并行无等待的部署。

给定一个站点部署任务项和它们之间的依赖关系，请给出一个站点的最短开站时间。

## 二、输入描述

第一行是任务数 `taskNum`。
第二行是任务的依赖关系数 `relationsNum`。
接下来 `relationsNum` 行，每行包含两个 id，描述一个依赖关系，格式为: `IDi IDj`，表示部署任务 `IDi` 部署完成了，部署任务 `IDj` 才能部署，`IDi` 和 `IDj` 值的范围为: `[0, taskNum)`。

注: 输入保证部署任务之间的依赖不会存在环。

## 三、输出描述

1 个整数，表示一个站点的最短开站时间。

## 四、用例

### 用例1
**输入**
```
5
5
0 4
1 2
1 3
2 3
2 4
```
**输出**
```
3
```
**说明**：先同时部署任务 0 和任务 1，然后部署任务 2，最后同时部署任务 3 和任务 4。最短开站时间为 3。

### 用例2
**输入**
```
5
3
0 3
0 4
1 3
```
**输出**
```
2
```
**说明**：先同时部署任务 0，任务 1，任务 2。然后再同时部署任务 3 和 4。最短开站时间为 2。

## 五、备注
- $1 < taskNum \le 100$
- $1 \le relationsNum \le 5000$

## 六、题目解析

### 1. 题目分析
本题是一道典型的图论问题。题目描述了任务之间的先后依赖关系，并且每个任务的执行耗时固定为 1。由于存在依赖且要求最短时间，这实际上是在求 **有向无环图（DAG）的层数** 或者说是 **最长路径的长度**。

关键点：
- 任务执行时间为 1，意味着同一层级的任务可以并行完成。
- 总时间取决于依赖链条中最长的那个序列。
- 题目保证无环。

### 2. 解题思路
我们可以使用 **拓扑排序（Kahn 算法）** 的变体来解决：
1. **构建图**：使用邻接表存储任务依赖关系，并记录每个任务的 **入度**（即该任务依赖的前置任务数量）。
2. **初始化队列**：将所有入度为 0 的任务（即没有任何依赖的任务）放入队列。
3. **分层遍历（BFS）**：
   - 使用一个循环，每次处理当前队列中所有的任务（这些任务在同一时间步执行）。
   - 时间计数器 `time` 加 1。
   - 遍历当前队列中的每个任务，将其指向的后续任务的入度减 1。
   - 如果某个后续任务的入度变为 0，说明它的所有前置任务已完成，将其放入下一轮的队列中。
4. **输出结果**：当所有任务处理完毕，计数器 `time` 即为最短开站时间。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取任务数
        if (!sc.hasNextInt()) return;
        int taskNum = sc.nextInt();
        
        // 读取关系数
        if (!sc.hasNextInt()) return;
        int relationsNum = sc.nextInt();
        
        // 邻接表
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < taskNum; i++) {
            adj.add(new ArrayList<>());
        }
        
        // 入度数组
        int[] inDegree = new int[taskNum];
        
        // 构建图
        for (int i = 0; i < relationsNum; i++) {
            int u = sc.nextInt();
            int v = sc.nextInt();
            adj.get(u).add(v);
            inDegree[v]++;
        }
        
        // 拓扑排序 BFS
        Queue<Integer> queue = new LinkedList<>();
        // 将所有入度为0的任务加入队列
        for (int i = 0; i < taskNum; i++) {
            if (inDegree[i] == 0) {
                queue.offer(i);
            }
        }
        
        int totalTime = 0;
        
        // 开始按层遍历
        while (!queue.isEmpty()) {
            int size = queue.size();
            totalTime++; // 进入新的一层，耗时加1
            
            // 弹出当前层级的所有任务
            for (int i = 0; i < size; i++) {
                int curr = queue.poll();
                
                // 遍历当前任务指向的后续任务
                for (int next : adj.get(curr)) {
                    inDegree[next]--;
                    // 如果入度变为0，则该任务可以开始部署
                    if (inDegree[next] == 0) {
                        queue.offer(next);
                    }
                }
            }
        }
        
        System.out.println(totalTime);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(V + E)$，其中 $V$ 是任务数 `taskNum`，$E$ 是依赖关系数 `relationsNum`。我们需要遍历所有的点和边来完成拓扑排序。
- **空间复杂度**：$O(V + E)$，用于存储邻接表、入度数组以及 BFS 队列。在题目给定范围内（V=100, E=5000），该算法非常高效。
