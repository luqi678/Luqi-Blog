## 一、 题目描述
假设你现在有 $n$ 个人，需要用救生艇将他们送走。每艘救生艇可以承载的最大重量为 `limit`。
每个人都有一个重量 `people[i]`。
**约束条件：**
1. 每艘救生艇最多只能承载 **两个人**。
2. 两个人的重量之和不能超过 `limit`。

请你计算出最少需要多少艘救生艇才能将所有人都安全地送走。

## 二、输入描述
* `people`: 一个整数数组，表示每个人的重量。
* `limit`: 一个整数，表示每艘救生艇的最大承载重量。

## 三、输出描述
* 一个整数，表示最少需要的救生艇数量。

## 四、用例
### 用例1
**输入：**
`people = [1, 2, 3, 4, 5]`, `limit = 5`
**输出：**
`3`
**解释：**
1. 第一艘救生艇载重量为 1 和 4 的人 ($1 + 4 = 5$)。
2. 第二艘救生艇载重量为 2 和 3 的人 ($2 + 3 = 5$)。
3. 第三艘救生艇载重量为 5 的人 ($5$)。
总共需要 3 艘。

### 用例2
**输入：**
`people = [3, 2, 2, 1]`, `limit = 3`
**输出：**
`3`
**解释：**
1. 第一艘救生艇载重量为 3 的人 ($3$)。
2. 第二艘救生艇载重量为 2 和 1 的人 ($2 + 1 = 3$)。
3. 第三艘救生艇载重量为 2 的人 ($2$)。
总共需要 3 艘。

## 五、备注
* $1 \le people.length \le 50,000$
* $1 \le people[i] \le limit \le 30,000$

## 六、题目解析

### 1. 题目分析
本题的核心在于如何分配人员以最大化每艘船的利用率。题目中一个关键的限制是：**每艘救生艇最多只能坐两个人**。这意味着无论一艘船剩下的载重还有多少，只要已经坐了两个人，就不能再坐第三个人。

### 2. 解题思路
由于每艘船最多坐两人，为了节省船只，我们应该尽可能让一艘船坐满两人。
贪心策略如下：
1. **排序**：首先对所有人的重量进行升序排序。
2. **双指针配合**：
   - 使用左指针 `left` 指向当前最轻的人。
   - 使用右指针 `right` 指向当前最重的人。
3. **配对逻辑**：
   - 如果最重的人和最轻的人重量之和 `people[left] + people[right] <= limit`，说明这两个人可以共用一艘船。此时 `left` 右移，`right` 左移。
   - 如果两人的重量之和大于 `limit`，说明这个最重的人无法与当前的任何一个人（即使是最轻的人）坐同一艘船。因此，最重的人必须**独自**占用一艘船。此时只有 `right` 左移。
4. **计数**：无论以上哪种情况发生，所需的救生艇数量都要加 1。
5. **循环结束**：当 `left > right` 时，所有人分配完毕。

### 3. 代码实现

#### Java8实现
```java
import java.util.Arrays;

public class Solution {
    /**
     * 计算最少需要的救生艇数量
     * 
     * @param people 每个人的重量数组
     * @param limit  救生艇的最大承载重量
     * @return 最少救生艇数
     */
    public int numRescueBoats(int[] people, int limit) {
        // 1. 对重量进行排序，方便使用贪心双指针
        Arrays.sort(people);
        
        int count = 0;
        int left = 0;                   // 指向最轻的人
        int right = people.length - 1;  // 指向最重的人
        
        // 2. 双指针遍历
        while (left <= right) {
            // 如果最轻的和最重的能坐在一起
            if (people[left] + people[right] <= limit) {
                left++; // 最轻的人上船
            }
            // 无论能不能带上最轻的，最重的人在这个逻辑分支里都会消耗一艘船
            right--; 
            count++; // 船只数量加 1
        }
        
        return count;
    }

    public static void main(String[] args) {
        Solution sol = new Solution();
        // 测试用例1
        System.out.println(sol.numRescueBoats(new int[]{1, 2, 3, 4, 5}, 5)); // 输出: 3
        // 测试用例2
        System.out.println(sol.numRescueBoats(new int[]{3, 2, 2, 1}, 3));    // 输出: 3
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(N \log N)$，其中 $N$ 是 `people` 数组的长度。复杂度主要消耗在排序（`Arrays.sort`）上，之后的双指针遍历只需要 $O(N)$ 的时间。
*   **空间复杂度**：$O(\log N)$ 或 $O(N)$，取决于排序算法的实现（Java 的 `DualPivotQuicksort` 对基本类型通常是 $O(\log N)$ 的辅助栈空间）。
