## 二、 题目描述

实现一个简单的绘图模块，绘图模块仅支持矩形的绘制和擦除。

1. 当新绘制的矩形与已有图形重叠时，对图形取**并集**；
2. 当新擦除的矩形与已有图形重叠时，对图形取**差集**。

给定一系列绘制和擦除操作，计算最终图形的面积。

## 二、输入描述

第一行为操作数 $N$，接下来的 $N$ 行格式为：

- `d x1 y1 x2 y2`：`d` 表示绘制操作，$(x1, y1)$ 为矩形左上角坐标，$(x2, y2)$ 为右下角坐标；
- `e x1 y1 x2 y2`：`e` 表示擦除操作，坐标含义同上。

坐标均为整数且范围在 $[-100, 100]$ 内，用例保证坐标有效性。

## 三、输出描述

输出最终图形的面积。

## 四、用例

### 用例1

**输入：**
```text
2  
d 0 2 2 0  
d -1 1 1 -1
```
**输出：**
```text
7
```

### 用例2

**输入：**
```text
2  
d 0 2 2 0  
e -1 1 1 -1
```
**输出：**
```text
3
```

## 五、备注
- 坐标系统采用标准笛卡尔坐标系。
- 坐标范围较小（$-100$ 到 $100$），可以利用该特性。

## 六、题目解析

### 1. 题目分析
题目要求我们处理矩形的绘制（相加）和擦除（相减）。由于坐标范围非常小（仅在 $-100$ 到 $100$ 之间），总的活动区域为 $200 \times 200$ 的网格。这意味着我们可以直接使用一个二维数组（或布尔矩阵）来模拟这个过程。

### 2. 解题思路
- **坐标偏移**：为了方便处理数组索引，我们将坐标统一加上 $100$。这样原先 $[-100, 100]$ 的范围变成了 $[0, 200]$。
- **单位网格**：我们将每一个单位长度的线段所围成的 $1 \times 1$ 小正方形看作网格中的一个点。如果一个矩形的左上角是 $(x1, y1)$，右下角是 $(x2, y2)$，那么它覆盖的 $x$ 范围是 $[\min(x1, x2), \max(x1, x2))$，覆盖的 $y$ 范围是 $[\min(y1, y2), \max(y1, y2))$。
- **模拟操作**：
    - `d` (draw): 将对应区域的网格值设为 $1$（或 `true`）。
    - `e` (erase): 将对应区域的网格值设为 $0$（或 `false`）。
- **计算面积**：遍历整个 $200 \times 200$ 的二维数组，统计值为 $1$ 的个数。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        
        // 坐标范围从-100到100，长度为200
        // 使用布尔数组表示每一个 1x1 的网格是否被覆盖
        boolean[][] canvas = new boolean[201][201];
        
        for (int i = 0; i < n; i++) {
            String op = sc.next();
            int x1 = sc.nextInt();
            int y1 = sc.nextInt();
            int x2 = sc.nextInt();
            int y2 = sc.nextInt();
            
            // 确保左边界小于右边界，下边界小于上边界
            int minX = Math.min(x1, x2);
            int maxX = Math.max(x1, x2);
            int minY = Math.min(y1, y2);
            int maxY = Math.max(y1, y2);
            
            // 将坐标偏移100，使其全部落在 [0, 200]
            int offset = 100;
            
            for (int x = minX; x < maxX; x++) {
                for (int y = minY; y < maxY; y++) {
                    if (op.equals("d")) {
                        canvas[x + offset][y + offset] = true;
                    } else if (op.equals("e")) {
                        canvas[x + offset][y + offset] = false;
                    }
                }
            }
        }
        
        // 统计最终 true 的个数，即为面积
        int area = 0;
        for (int i = 0; i < 201; i++) {
            for (int j = 0; j < 201; j++) {
                if (canvas[i][j]) {
                    area++;
                }
            }
        }
        
        System.out.println(area);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N \times W \times H)$，其中 $N$ 是操作次数，$W$ 和 $H$ 是矩形的长宽。由于坐标范围固定在 $200$ 以内，最坏情况下单次操作涉及 $200 \times 200 = 40,000$ 个点，总复杂度在可接受范围内。
- **空间复杂度**：$O(C^2)$，其中 $C$ 是坐标范围（$200$）。创建了一个 $201 \times 201$ 的二维数组，空间消耗极小。
