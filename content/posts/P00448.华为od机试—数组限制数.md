## 二、题目描述

给定一个数组，可以给数组添加一个限制数 (M)，使得数组中的每一个数和限制数 (M) 取 $\min(a_i, M)$。要求使得处理后的数组总和不超过给定的阈值 `total`，求限制数 $M$ 最高可以取多少。如果原始数组的总和本身就不超过 `total`，则不需要限制，返回 -1。

## 二、输入描述

第一行包含两个整数 $n, total$ ($1 \le n, total, a_i \le 100,000$)，分别代表数组的长度和不能超过的数组总和。

第二行包含 $n$ 个整数，代表数组的每个元素 $a_i$。

## 三、输出描述

输出一个整数，表示限制数 $M$ 的最大值。如果不需要限制，输出 -1。

## 四、用例

### 用例1
**输入**：
```text
5 12
1 2 3 4 5
```
**输出**：
```text
3
```
**说明**：
当 $M=3$ 时，数组变为 `[1, 2, 3, 3, 3]`，总和为 $1+2+3+3+3 = 12 \le 12$。
当 $M=4$ 时，数组变为 `[1, 2, 3, 4, 4]`，总和为 $1+2+3+4+4 = 14 > 12$。
因此最大 $M$ 为 3。

### 用例2
**输入**：
```text
5 15
1 2 3 4 5
```
**输出**：
```text
-1
```
**说明**：
原始数组总和为 $1+2+3+4+5 = 15$。由于 $15 \le 15$，不需要限制，返回 -1。

## 五、备注
- 数组长度 $n$ 最高 100,000。
- 数组元素 $a_i$ 最高 100,000。
- 阈值 `total` 最高 100,000。

## 六、题目解析

### 1. 题目分析
本题的核心是寻找一个临界值 $M$。随着 $M$ 的增大，经过 $\min(a_i, M)$ 处理后的数组总和 $S(M)$ 是单调递增的。
- 如果 $S(\infty) \le total$（即原始数组总和 $\le total$），直接输出 -1。
- 否则，我们需要在 $[0, \max(a_i)]$ 的范围内找到一个最大的 $M$，使得 $S(M) \le total$。

### 2. 解题思路
由于总和函数 $S(M)$ 具有单调性，我们可以采用 **二分查找 (Binary Search)** 的方法来高效寻找 $M$：
1. **计算原始总和**：遍历一遍数组，求和。如果和 $\le total$，输出 -1。
2. **确定二分边界**：左边界 `low = 0`，右边界 `high = max(a_i)`。
3. **执行二分**：
   - 取中间值 `mid = (low + high) / 2`。
   - 计算当前的 `currentSum = sum(min(a_i, mid))`。
   - 如果 `currentSum <= total`，说明当前的 `mid` 是合法的，可能还有更大的 $M$，因此记录 `ans = mid` 并尝试向右搜索 `low = mid + 1`。
   - 如果 `currentSum > total`，说明当前的 `mid` 太大了，必须减小限制，向左搜索 `high = mid - 1`。
4. **输出结果**：最终记录的 `ans` 即为所求。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        long total = sc.nextLong();
        
        int[] nums = new int[n];
        long currentSum = 0;
        int maxVal = 0;
        
        for (int i = 0; i < n; i++) {
            nums[i] = sc.nextInt();
            currentSum += nums[i];
            if (nums[i] > maxVal) {
                maxVal = nums[i];
            }
        }
        
        // 如果初始总和已经不超过total，返回-1
        if (currentSum <= total) {
            System.out.println(-1);
            return;
        }
        
        // 二分查找最大限制数 M
        int low = 0;
        int high = maxVal;
        int ans = 0;
        
        while (low <= high) {
            int mid = low + (high - low) / 2;
            if (check(nums, mid, total)) {
                ans = mid; // 记录当前可行解
                low = mid + 1; // 尝试寻找更大的限制数
            } else {
                high = mid - 1; // 限制数太大，需要减小
            }
        }
        
        System.out.println(ans);
    }
    
    /**
     * 检查当前的限制数 mid 是否满足总和不超过 total
     */
    private static boolean check(int[] nums, int mid, long total) {
        long sum = 0;
        for (int x : nums) {
            sum += Math.min(x, mid);
            // 优化：如果中间求和已经超过total，可以提前返回
            if (sum > total) return false;
        }
        return sum <= total;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(n \log(\max(A)))$。其中 $n$ 是数组长度，$\max(A)$ 是数组中的最大值。二分查找的次数为 $\log(100,000) \approx 17$ 次，每次检查需要遍历数组 $O(n)$。
- **空间复杂度**：$O(n)$。用于存储输入的数组。
