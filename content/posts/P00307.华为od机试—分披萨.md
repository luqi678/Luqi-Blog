## 二、题目描述

吃货和“馋嘴”两人到披萨店点了一份铁盘圆形披萨，并嘱咐店员将披萨按放射状切成大小相同的偶数个小块。但是粗心的服务员将披萨切成了每块大小都完全不同**奇数块**，且肉眼能分辨出大小。

由于两人都想吃到最多的披萨，他们商量了一个他们认为公平的分法：
1. 从“吃货”开始，轮流取披萨。
2. 除了第一块披萨可以任意选取外，其他都必须从**缺口**开始选（即只能选剩余块中相邻的边缘块）。
3. 他俩选披萨的思路不同：
    - “馋嘴”每次都会选**两端中最大**的那块披萨。
    - “吃货”知道“馋嘴”的想法，会通过规划获得自己能得到的最大披萨总和。

已知披萨小块的数量以及每块的大小，求“吃货”能分得的最大的披萨大小的总和。

## 二、输入描述

- 第 1 行为一个正整数奇数 $N$，表示披萨小块数量 ($3 \le N < 500$)。
- 接下来的第 2 行到第 $N + 1$ 行（共 $N$ 行），每行为一个正整数，表示第 $i$ 块披萨的大小 ($1 \le i \le N$)。
- 披萨小块按顺时针或逆时针方向依次编号。
- 每块披萨的大小范围为 $[1, 2,147,483,647]$。

## 三、输出描述

- “吃货”能分得到的最大的披萨大小的总和。

## 四、用例

### 用例1
**输入**
```text
5
8
2
10
5
7
```
**输出**
```text
19
```
**说明**
1. “吃货”先拿 10。
2. 剩下 [5, 7, 8, 2]（环形）。缺口两端是 5 和 2。“馋嘴”选大的，拿走 5。
3. 剩下 [7, 8, 2]。缺口两端是 7 和 2。“吃货”选 7。
4. 剩下 [8, 2]。“馋嘴”选大的，拿走 8。
5. 剩下 [2]。“吃货”拿走 2。
总计：10 + 7 + 2 = 19。

### 用例2
**输入**
```text
3
1
3
2
```
**输出**
```text
3
```

## 五、备注
1. $N$ 是奇数，意味着“吃货”总共拿 $\frac{N+1}{2}$ 块，“馋嘴”拿 $\frac{N-1}{2}$ 块。
2. 由于数据范围较大，总和可能超过 `int`，需使用 `long`。

## 六、题目解析

### 1. 题目分析
本题属于博弈论结合动态规划的题目。
- **环境**：环形数组。
- **吃货策略**：最优决策（求最大值）。
- **馋嘴策略**：贪心决策（固定取两端最大值）。
- **约束**：除了第一块，必须从缺口两端取。

### 2. 解题思路
由于“馋嘴”的选择是确定性的，我们可以只考虑“吃货”在每一轮的选择。

1.  **第一步**：吃货可以选 $N$ 个位置中的任意一个。我们遍历每一个位置 $i$ 作为起点。
2.  **后续过程**：一旦吃货选了第 $i$ 块，剩下的披萨形成一个区间（在环形数组中）。接下来轮到“馋嘴”选，然后“吃货”选，循环往复。
3.  **状态定义**：使用记忆化搜索 `memo[L][R]` 表示：当剩下的披萨区间为从索引 $L$ 到 $R$ 时（此时该“馋嘴”选了），吃货从这部分剩余披萨中能获得的最大值。
4.  **递归逻辑**：
    - **馋嘴操作**：比较 `arr[L]` 和 `arr[R]`。
        - 如果 `arr[L] > arr[R]`，馋嘴拿走 $L$，区间变为 `(L+1, R)`。
        - 如果 `arr[R] > arr[L]`，馋嘴拿走 $R$，区间变为 `(L, R-1)`。
    - **吃货操作**（在馋嘴拿走后）：
        - 剩下的区间如果是 `[newL, newR]`。
        - 吃货可以选 `arr[newL]`，之后进入 `solve(newL+1, newR)`。
        - 或者选 `arr[newR]`，之后进入 `solve(newL, newR-1)`。
        - 吃货取两者中的最大值。
    - **边界条件**：如果 $L == R$，馋嘴拿走这一块，吃货得 0。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    static int N;
    static long[] pizza;
    static long[][] memo;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        N = sc.nextInt();
        pizza = new long[N];
        for (int i = 0; i < N; i++) {
            pizza[i] = sc.nextLong();
        }

        long maxTotal = 0;
        // 吃货可以第一块任意选，遍历所有可能的第一块
        for (int i = 0; i < N; i++) {
            // 选了第 i 块后，剩下的区间是 [i+1, i-1] (环形)
            memo = new long[N][N];
            for (int j = 0; j < N; j++) {
                for (int k = 0; k < N; k++) {
                    memo[j][k] = -1;
                }
            }
            // 剩下的区间两端索引
            int L = (i + 1) % N;
            int R = (i - 1 + N) % N;
            
            // 当前总和 = 第一块的大小 + 在剩余区间里能拿到的最大值
            maxTotal = Math.max(maxTotal, pizza[i] + solve(L, R));
        }

        System.out.println(maxTotal);
    }

    /**
     * 计算在剩余区间 [L, R] 中，由馋嘴先选，吃货能拿到的最大值
     */
    public static long solve(int L, int R) {
        // 馋嘴拿走之后，剩下的块数是偶数。如果 L==R，说明只剩下一块，馋嘴拿走，吃货得0
        if (pizza[L] > pizza[R]) {
            L = (L + 1) % N;
        } else {
            R = (R - 1 + N) % N;
        }
        
        // 如果馋嘴拿完后没剩了（原本剩2块），吃货得0
        if (L == R) {
            return pizza[L];
        }
        
        // 记忆化
        if (memo[L][R] != -1) {
            return memo[L][R];
        }

        // 现在轮到吃货选，他可以选左边或右边
        // 选左边：当前值 + 剩下的给馋嘴先选
        long pickLeft = pizza[L] + solve((L + 1) % N, R);
        // 选右边：当前值 + 剩下的给馋嘴先选
        long pickRight = pizza[R] + solve(L, (R - 1 + N) % N);

        memo[L][R] = Math.max(pickLeft, pickRight);
        return memo[L][R];
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(N^2)$。
    - 虽然外层有一个 $O(N)$ 的循环遍历起手位置，但内部使用了记忆化搜索。对于一个固定的披萨数组，所有可能的区间状态只有 $N^2$ 个。
    - 注意：在实现中，为了方便理解，我在每个起手位置后都重置了 `memo`。优化做法是可以只初始化一次 `memo`，因为无论起手是谁，`solve(L, R)` 的含义是相同的。即使如此，$500^2$ 的计算量在 1s 内绰绰有余。
- **空间复杂度**：$O(N^2)$。用于存储记忆化搜索的状态 `memo` 数组。
