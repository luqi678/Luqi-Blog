## 二、 题目描述

有 $N$ 个快递站点用字符串标识，某些站点之间有道路连接。每个站点有一些包裹要运输，每个站点间的包裹不重复。路上有检查站会导致部分货物无法通行，计算哪些货物无法正常投递？

## 二、输入描述

1. 第一行输入 $M$ $N$，分别代表 $M$ 个包裹和 $N$ 个道路限制信息。
2. $0 \le M, N \le 100$。
3. 接下来的 $M$ 行，每行包含：`包裹名称 起点 终点`。
4. 接下来的 $N$ 行，每行包含：`站点1 站点2 检查站禁止通行的包裹名称`。其中禁止通行的包裹如果有多个，以空格分开。

## 三、输出描述

输出不能送达的包裹名称，如：`package2 package4`。
如果所有包裹都可以送达，则输出：`none`。
输出结果按照包裹名称的**升序**排列。

## 四、用例

### 用例1
**输入：**
```text
4 2
package1 A C
package2 A C
package3 B C
package4 A C
A B package1
A C package2
```
**输出：**
```text
package2
```
**说明：**
有 4 个包裹，2 个禁止通行规则。
包裹1从 A 到 C，但在 A-B 路段限制 package1，因为 AC 路线不一定经过 AB，在此题逻辑中，仅当限制的路径（站点1 站点2）与包裹的路径（起点 终点）完全一致时才认为被拦截。
包裹2从 A 到 C，A-C 路段正好限制了 package2，因此 package2 无法送达。

### 用例2
**输入：**
```text
2 1
package1 A B
package2 C D
A B package1 package2
```
**输出：**
```text
package1
```
**说明：**
package1 路径 A-B，限制 A-B 包含 package1，拦截。
package2 路径 C-D，虽然 A-B 限制了 package2，但 package2 不经过 A-B，不拦截。

## 五、备注
1. 包裹名称、站点名称均为字符串。
2. 注意处理多个禁止通行包裹的情况（空格分隔）。
3. 路径是有向的，即“站点1 站点2”的限制只针对从站点1到站点2的包裹。

---

## 六、题目解析

### 1. 题目分析
本题的核心是判断包裹的运输路径是否遇到了该路径上的拦截规则。
- 每个包裹都有明确的 `起点` 和 `终点`。
- 每个拦截规则对应一个 `特定路段（站点1-站点2）` 以及该路段禁止通过的 `包裹列表`。
- 我们需要匹配：如果一个包裹的 `起点-终点` 与某个拦截规则的 `站点1-站点2` 相同，并且该包裹在该规则的禁止名单中，则该包裹无法送达。

### 2. 解题思路
1. **存储包裹信息**：使用一个结构（如 `Map` 或对象列表）记录每个包裹及其对应的 `起点-终点` 关系。
2. **存储限制信息**：使用一个 `Map<String, Set<String>>`。
   - Key：`站点1 + "-" + 站点2`。
   - Value：该路径禁行的包裹集合（`Set` 方便查找）。
3. **匹配校验**：遍历每一个包裹，根据其 `起点` 和 `终点` 生成 Key，去限制信息的 Map 中查找。如果存在该 Key，且其对应的 Value 集合中包含该包裹名，则将该包裹标记为“无法送达”。
4. **排序输出**：将所有被拦截的包裹存入 List，进行升序排序。如果 List 为空，输出 `none`。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        if (!sc.hasNextInt()) return;
        int M = sc.nextInt(); // 包裹数
        int N = sc.nextInt(); // 限制数
        sc.nextLine(); // 消耗换行符

        // 存储包裹的路径信息: packageName -> "start-end"
        Map<String, String> packagePathMap = new HashMap<>();
        // 存储所有的包裹名称，用于最后遍历（或者直接存路径信息）
        List<String> allPackages = new ArrayList<>();

        for (int i = 0; i < M; i++) {
            String line = sc.nextLine();
            if (line == null || line.isEmpty()) continue;
            String[] parts = line.split(" ");
            String pkgName = parts[0];
            String start = parts[1];
            String end = parts[2];
            packagePathMap.put(pkgName, start + " " + end);
            allPackages.add(pkgName);
        }

        // 存储限制信息: "start end" -> Set<packageName>
        Map<String, Set<String>> restrictions = new HashMap<>();
        for (int i = 0; i < N; i++) {
            String line = sc.nextLine();
            if (line == null || line.isEmpty()) continue;
            String[] parts = line.split(" ");
            if (parts.length < 3) continue;
            String s1 = parts[0];
            String s2 = parts[1];
            String routeKey = s1 + " " + s2;
            
            restrictions.putIfAbsent(routeKey, new HashSet<>());
            for (int j = 2; j < parts.length; j++) {
                restrictions.get(routeKey).add(parts[j]);
            }
        }

        // 查找被拦截的包裹
        List<String> intercepted = new ArrayList<>();
        for (String pkg : allPackages) {
            String path = packagePathMap.get(pkg);
            if (restrictions.containsKey(path)) {
                if (restrictions.get(path).contains(pkg)) {
                    intercepted.add(pkg);
                }
            }
        }

        // 输出
        if (intercepted.isEmpty()) {
            System.out.println("none");
        } else {
            Collections.sort(intercepted);
            for (int i = 0; i < intercepted.size(); i++) {
                System.out.print(intercepted.get(i) + (i == intercepted.size() - 1 ? "" : " "));
            }
            System.out.println();
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(M + N \cdot K + P \log P)$，其中 $M$ 是包裹数，$N$ 是限制规则数，$K$ 是每个规则中禁止包裹的平均数量，$P$ 是被拦截包裹的数量。主要开销在读取输入和最后的排序。
- **空间复杂度**：$O(M + N \cdot K)$，用于存储包裹路径和禁止通行的映射表。
