## 二、题目描述

小华按照地图去寻宝，地图上被划分成 $m$ 行和 $n$ 列的方格，横坐标范围是 $[0, n-1]$，纵坐标范围是 $[0, m-1]$。

在横坐标和纵坐标的各个数位之和不大于 $k$ 的方格中存在黄金（每个方格中仅存在一克黄金），但横坐标和纵坐标之和大于 $k$ 的方格存在危险不可进入。

小华从入口 (0,0) 进入，任何时候只能向左，右，上，下四个方向移动一格。

请问小华最多能获得多少克黄金？

## 二、输入描述

输入中包含3个数字，分别是 $m, n, k$。
坐标取值范围如下：
- $0 \le m \le 50$
- $0 \le n \le 50$
- $k$ 的取值范围如下：
- $0 \le k \le 100$

## 三、输出描述

输出小华最多能获得多少克黄金。

## 四、用例

### 用例1
**输入：**
```
40 40 18
```
**输出：**
```
1484
```

### 用例2
**输入：**
```
5 4 7
```
**输出：**
```
20
```

## 五、备注
1. 小华从 (0,0) 出发。
2. 只有满足 `数位之和(x) + 数位之和(y) <= k` 的格子才是安全的。
3. 小华只能在安全的格子之间移动。
4. 目标是统计从 (0,0) 出发所有能到达的安全格子的总数。

## 六、题目解析

### 1. 题目分析
本题是一个典型的**网格连通性问题**。
- **地图规模**：最大 $50 \times 50$，总格子数不超过 2500，规模较小。
- **准入条件**：
    1. 坐标在网格范围内：$0 \le r < m, 0 \le c < n$。
    2. 数位之和限制：`getDigitSum(r) + getDigitSum(c) <= k`。
- **移动规则**：上下左右移动。
- **核心任务**：从 (0,0) 开始，找出所有能够通过安全路径到达的格子数量。

### 2. 解题思路
由于我们需要从一个起点开始探索所有可达的节点，可以使用 **广度优先搜索 (BFS)** 或 **深度优先搜索 (DFS)**。

1. **预处理/辅助函数**：编写一个函数 `getDigitSum(int num)` 来计算一个数字的各个位之和。例如 `getDigitSum(12) = 1 + 2 = 3`。
2. **状态记录**：使用一个二维布尔数组 `visited[m][n]` 来记录哪些格子已经被访问过，防止重复计数和陷入死循环。
3. **搜索过程（以BFS为例）**：
   - 将起点 `(0,0)` 加入队列，并标记 `visited[0][0] = true`。
   - 只要队列不为空：
     - 取出队首坐标 `(currR, currC)`。
     - 黄金数 `ans++`。
     - 遍历四个方向（上、下、左、右）的新坐标 `(nextR, nextC)`。
     - 检查新坐标是否：
       - 在网格范围内。
       - 未被访问过。
       - 满足 `getDigitSum(nextR) + getDigitSum(nextC) <= k`。
     - 若满足，则标记已访问并加入队列。
4. **输出结果**：最终统计到的 `ans` 即为所求。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();
        int n = sc.nextInt();
        int k = sc.nextInt();

        System.out.println(countGold(m, n, k));
    }

    private static int countGold(int m, int n, int k) {
        // 标记是否访问过
        boolean[][] visited = new boolean[m][n];
        // 队列用于BFS
        Queue<int[]> queue = new LinkedList<>();

        // 检查起点(0,0)是否安全
        if (getDigitSum(0) + getDigitSum(0) <= k) {
            queue.offer(new int[]{0, 0});
            visited[0][0] = true;
        } else {
            return 0; // 起点都不安全，无法获得黄金
        }

        int count = 0;
        int[][] dirs = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

        while (!queue.isEmpty()) {
            int[] curr = queue.poll();
            count++;

            for (int[] dir : dirs) {
                int nextR = curr[0] + dir[0];
                int nextC = curr[1] + dir[1];

                // 范围检查
                if (nextR >= 0 && nextR < m && nextC >= 0 && nextC < n) {
                    // 访问检查和数位和检查
                    if (!visited[nextR][nextC] && (getDigitSum(nextR) + getDigitSum(nextC) <= k)) {
                        visited[nextR][nextC] = true;
                        queue.offer(new int[]{nextR, nextC});
                    }
                }
            }
        }

        return count;
    }

    // 计算数位之和
    private static int getDigitSum(int num) {
        int sum = 0;
        while (num > 0) {
            sum += num % 10;
            num /= 10;
        }
        return sum;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(m \times n)$。在最坏情况下（$k$ 很大），我们需要遍历网格中的每一个格子。每个格子的操作（计算数位和、入队出队）都是常数时间的。
- **空间复杂度**：$O(m \times n)$。主要开销是 `visited` 数组以及 BFS 过程中队列的存储空间。
