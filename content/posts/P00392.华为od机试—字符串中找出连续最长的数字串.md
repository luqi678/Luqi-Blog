## 一、 题目描述

请在一个字符串中找出连续最长的数字串，并返回这个数字串。
1. 如果存在长度相同的连续数字串，返回**最后一个**。
2. 如果没有符合条件的字符串，返回空字符串`""`。

**注意：**
* 数字串可以由数字 `0-9`、小数点 `.`、正负号 `±` 组成，长度包括组成数字串的所有符号。
* `.`、`±` 仅能出现一次。
* `.` 的两边必须是数字（即不能以点开头或结尾，且点前后必须紧邻数字）。
* `±` 仅能出现在开头且其后必须要有数字。
* 长度不定，可能含有空格。

## 二、输入描述

一个包含随机字符的字符串。

## 三、输出描述

连续最长的数字串。

## 四、用例

### 用例1
**输入**
```text
1234567890abcd9.+12345.678.9ed
```
**输出**
```text
+12345.678
```

### 用例2
**输入**
```text
123.456 789.012as-234.567
```
**输出**
```text
-234.567
```

## 五、备注
*   字符串中可能包含空格，处理时需注意。
*   由于要求“最后一个长度最长的”，在遍历过程中应使用 `>=` 判断长度。

## 六、题目解析

### 1. 题目分析
题目要求提取符合特定“数字规则”的最长子串。规则总结如下：
- **符号位**：`+` 或 `-` 可选，但只能在数字串最前面，且后面必须接数字。
- **整数部分**：必须有数字。
- **小数部分**：`.` 可选，如果有，则点前后必须都有数字。
- **唯一性**：正负号只能一个，小数点只能一个。

### 2. 解题思路
解决此题最优雅的方式是使用 **正则表达式**。

根据规则构造正则：
1.  符号位：`[+-]?` 表示可选的正负号。
2.  数字接符号：`\d+` 表示至少一个数字（确保符号后有数字）。
3.  小数部分：`(\.\d+)?` 表示可选的小数点及其后的数字。这样保证了小数点后面必须有数字，且由于前面已经有 `\d+`，也保证了小数点前面有数字。

组合正则：`[+-]?\d+(\.\d+)?`

**匹配流程：**
1.  使用正则引擎在输入字符串中查找所有匹配项。
2.  记录每个匹配项的长度。
3.  维护一个变量 `maxLen` 和 `result`。
4.  如果当前匹配项长度 `length >= maxLen`，更新 `result`（因为题目要求相同长度返回最后一个，所以用大于等于）。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.regex.Matcher;
import java.util.regex.Pattern;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        // 注意：题目提到长度不定，可能含有空格，建议读取整行
        if (!sc.hasNextLine()) {
            System.out.println("");
            return;
        }
        String input = sc.nextLine();

        // 正则解析：
        // [+-]?      : 可能存在的符号位
        // \d+        : 必须存在的整数部分（同时也满足了符号后必须有数字）
        // (\.\d+)?   : 可选的小数部分，且点后面必须有数字
        String regex = "[+-]?\\d+(\\.\\d+)?";
        Pattern pattern = Pattern.compile(regex);
        Matcher matcher = pattern.matcher(input);

        String maxStr = "";
        int maxLen = 0;

        while (matcher.find()) {
            String current = matcher.group();
            // 题目要求：长度相同时返回最后一个
            if (current.length() >= maxLen) {
                maxLen = current.length();
                maxStr = current;
            }
        }

        System.out.println(maxStr);
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：O(n)，其中 n 为字符串的长度。正则表达式扫描字符串一次。
*   **空间复杂度**：O(n)，用于存储匹配到的字符串片段以及输入字符串。

## 引用
- 华为OD机试真题策略：针对此类字符串提取题，正则匹配是最稳妥且高效的方法。
- 注意事项：如果题目不准使用正则，则需要使用**双指针（滑动窗口）**配合复杂的条件逻辑（状态机思路）进行模拟，难度会大幅上升。
