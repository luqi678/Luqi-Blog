## 二、 题目描述

给一个字符串和一个二维字符数组，如果该字符串存在于该数组中，则按字符串的字符顺序输出字符串每个字符所在单元格的位置下标字符串，如果找不到返回字符串“N”。

1. 需要按照字符串的字符组成顺序搜索，且搜索到的位置必须是相邻单元格，其中“相邻单元格”是指那些水平相邻或垂直相邻的单元格。
2. 同一个单元格内的字母不允许被重复使用。
3. 假定在数组中最多只存在一个可能的匹配。

## 二、输入描述

1. 第 1 行为一个数字 N 指示二维数组在后续输入所占的行数。
2. 第 2 行到第 N+1 行输入为一个二维大写字符数组，每行字符用半角 `,` 分割。
3. 第 N+2 行为待查找的字符串，由大写字符组成。
4. 二维数组的大小为 $N \times N$，$0 < N \le 100$。
5. 单词长度 $K$，$0 < K < 1000$。

## 三、输出描述

输出一个位置下标字符串，拼接格式为：`第1个字符行下标,第1个字符列下标,第2个字符行下标,第2个字符列下标… ,第N个字符行下标,第N个字符列下标`。

## 四、用例

### 用例1
**输入**
```text
4
A,C,C,F
C,D,E,D
B,E,S,S
F,E,C,A
ACCESS
```
**输出**
```text
0,0,0,1,0,2,1,2,2,2,2,3
```
**说明**
ACCESS 分别对应二维数组的 [0,0] [0,1] [0,2] [1,2] [2,2] [2,3] 下标位置。

### 用例2
**输入**
```text
2
A,B
C,D
HELLO
```
**输出**
```text
N
```

## 五、备注
1. 矩阵大小 $N \times N$ 最大为 $100 \times 100$。
2. 目标字符串长度最大为 1000。
3. 如果有解，题目保证解唯一。

## 六、题目解析

### 1. 题目分析
这是一个典型的二维矩阵搜索问题。题目要求在矩阵中寻找一条路径，使得路径上的字符连起来等于目标字符串。约束条件是：
- 只能向上下左右四个方向移动。
- 每个单元格只能使用一次（即不可重复经过同一坐标）。
- 必须按字符串字符顺序。

### 2. 解题思路
1. **遍历起点**：首先在矩阵中寻找目标字符串的第一个字符。每找到一个匹配的起始位置，就开始进行深度优先搜索（DFS）。
2. **深度优先搜索 (DFS)**：
   - 维护当前匹配到字符串的第几个索引 `index`。
   - 记录当前已访问的路径坐标。
   - 向四个方向尝试移动。
   - **回溯**：为了防止路径重叠，在进入下一步搜索前，标记当前单元格已访问，搜索结束后（如果该路径不通）需将其标记还原。
3. **输出处理**：一旦找到完整路径，记录所有坐标并格式化输出。由于题目声明最多只有一个匹配，找到后可立即终止搜索并输出结果。

### 3. 代码实现

#### Java 实现

```java
import java.util.*;

public class Main {
    static int n;
    static char[][] matrix;
    static String target;
    static boolean[][] visited;
    static int[][] dirs = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
    static List<String> resPath = new ArrayList<>();

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        n = sc.nextInt();
        matrix = new char[n][n];
        visited = new boolean[n][n];

        for (int i = 0; i < n; i++) {
            String[] line = sc.next().split(",");
            for (int j = 0; j < n; j++) {
                matrix[i][j] = line[j].charAt(0);
            }
        }
        target = sc.next();

        // 遍历矩阵，寻找起始点
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (matrix[i][j] == target.charAt(0)) {
                    if (dfs(i, j, 0)) {
                        System.out.println(String.join(",", resPath));
                        return;
                    }
                }
            }
        }

        System.out.println("N");
    }

    private static boolean dfs(int r, int c, int index) {
        // 如果当前字符不匹配，直接返回
        if (matrix[r][c] != target.charAt(index)) {
            return false;
        }

        // 记录坐标
        resPath.add(r + "," + c);
        visited[r][c] = true;

        // 如果已经匹配到字符串末尾
        if (index == target.length() - 1) {
            return true;
        }

        // 尝试四个方向
        for (int[] dir : dirs) {
            int nr = r + dir[0];
            int nc = c + dir[1];

            if (nr >= 0 && nr < n && nc >= 0 && nc < n && !visited[nr][nc]) {
                if (dfs(nr, nc, index + 1)) {
                    return true;
                }
            }
        }

        // 回溯：如果当前路径走不通，移除记录并重置访问标记
        resPath.remove(resPath.size() - 1);
        visited[r][c] = false;
        return false;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N^2 \cdot 3^K)$，其中 $N$ 是矩阵边长，$K$ 是字符串长度。每个起点最多探索 $3^K$ 种可能（因为不走回头路，四方向减一）。但在实际机考数据中，由于字符匹配的剪枝效应，通常远快于理论上限。
- **空间复杂度**：$O(N^2 + K)$。$N^2$ 用于存储矩阵和访问数组，$K$ 用于递归栈和存储路径坐标。
