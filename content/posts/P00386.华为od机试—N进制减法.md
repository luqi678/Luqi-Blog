## 一、 题目描述

主管期望你实现一个基于字符串的N进制的减法。
需要对输入的两个字符串按照给定的N进制进行减法操作，输出正负符号和表示结果的字符串。

**限制条件：**
1. N值范围为大于等于2、小于等于35。
2. 有效的字符包括 `0-9` 以及小写字母 `a-z`。
3. 字符串有效字符个数最大为100个字符。
4. 输入的被减数和减数，除了单独的“0”以外，不能是以“0”开头的字符串。
5. 如果输入有异常（如非法字符、前导零、超出进制范围等）或计算过程中有异常，应当输出 `-1` 表示错误。

## 二、输入描述

输入为三个参数：
1. 第一个参数是整数形式的进制 $N$ 值。
2. 第二个参数为被减数字符串。
3. 第三个参数为减数字符串。

## 三、输出描述

输出包含两个部分（以空格分隔）：
1. 第一个是减法计算的结果符号：`0` 表示结果为正数或零，`1` 表示结果为负数。
2. 第二个是表示结果的字符串。
如果输入异常，则仅输出 `-1`。

## 四、用例

### 用例1
**输入**：
`2 11 1`

**输出**：
`0 10`

**说明**：
按二进制计算 $11_2 - 1_2 = 10_2$，计算正常，结果为正数，输出 `0 10`。

### 用例2
**输入**：
`8 07 1`

**输出**：
`-1`

**说明**：
按8进制，检查到被减数 `07` 不符合非0前导的要求（除了单独的0以外不能0开头），返回 `-1`。

## 五、备注
1. 字母 `a-z` 对应数值 `10-35`。
2. 结果字符串也需要遵循N进制表示法，并去掉多余的前导零。

## 六、题目解析

### 1. 题目分析
本题核心考点在于：
- **进制校验**：需要检查字符是否在 $0 \sim N-1$ 的范围内。
- **前导零校验**：除了 "0" 本身，其他数字不能以 "0" 开头。
- **大数运算**：字符串长度可达100位，远超 `long` 的范围，必须使用大数处理类（如 Java 的 `BigInteger`）。
- **进制转换**：涉及 $N$ 进制转十进制，进行减法后再转回 $N$ 进制。

### 2. 解题思路
1. **读取输入**：获取 $N$、被减数 $S1$、减数 $S2$。
2. **合法性检查**：
   - 检查 $N$ 的范围（2-35）。
   - 检查 $S1$ 和 $S2$ 是否符合前导零规则。
   - 检查 $S1$ 和 $S2$ 中的每一个字符，转换成数值后是否满足 $< N$。
3. **转换与计算**：
   - 利用 `BigInteger` 的构造函数 `new BigInteger(String val, int radix)` 将 $N$ 进制字符串转为十进制。
   - 执行减法：`result = val1.subtract(val2)`。
4. **格式化输出**：
   - 判断 `result` 的正负。
   - 如果 `result < 0`，符号位为 `1`，取绝对值。
   - 如果 `result >= 0`，符号位为 `0`。
   - 将结果转换回 $N$ 进制字符串：`result.abs().toString(radix)`。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.math.BigInteger;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) {
            System.out.println("-1");
            return;
        }

        int n = sc.nextInt();
        String s1 = sc.next();
        String s2 = sc.next();

        // 1. 验证进制范围
        if (n < 2 || n > 35) {
            System.out.println("-1");
            return;
        }

        // 2. 验证前导零和字符合法性
        if (!isValid(s1, n) || !isValid(s2, n)) {
            System.out.println("-1");
            return;
        }

        try {
            // 3. 使用 BigInteger 进行大数运算
            // BigInteger 构造函数支持 radix，范围为 Character.MIN_RADIX(2) 到 Character.MAX_RADIX(36)
            BigInteger val1 = new BigInteger(s1, n);
            BigInteger val2 = new BigInteger(s2, n);

            BigInteger diff = val1.subtract(val2);

            // 4. 处理符号并输出
            int sign = diff.compareTo(BigInteger.ZERO) < 0 ? 1 : 0;
            // toString(radix) 会返回小写字母表示的进制字符串
            String resStr = diff.abs().toString(n);

            System.out.println(sign + " " + resStr);

        } catch (Exception e) {
            // 捕捉可能的转换异常
            System.out.println("-1");
        }
    }

    /**
     * 校验字符串在 N 进制下是否合法
     */
    private static boolean isValid(String s, int n) {
        // 非空检查
        if (s == null || s.isEmpty()) return false;

        // 前导零检查：长度大于1且以0开头则非法
        if (s.length() > 1 && s.startsWith("0")) return false;

        // 字符范围检查
        for (char c : s.toCharArray()) {
            // Character.digit 会处理 0-9 和 a-z(不区分大小写)
            // 题目要求小写字母，此处由于 digit 兼容，需额外关注。但主要判断其对应数值是否 >= n
            int digit = Character.digit(c, n);
            if (digit == -1 || digit >= n) {
                return false;
            }
            // 题目明确有效字符是 0-9 和 a-z，需排除 A-Z
            if (Character.isUpperCase(c)) {
                return false;
            }
        }
        return true;
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(L)$，其中 $L$ 是字符串的长度（最大100）。主要开销在于字符串校验、大数转换和减法运算，`BigInteger` 的操作在百位级别是非常迅速的。
- **空间复杂度**：$O(L)$，用于存储输入的字符串和转换后的结果。

- 进制转换规则：$0-9$ 对应 $0-9$，$a-z$ 对应 $10-35$。
