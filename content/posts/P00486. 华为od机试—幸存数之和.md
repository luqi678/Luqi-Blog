## 一、 题目描述

一个跳数 `jump`，及幸存数量 `left`。给一个正整数数列 `nums`，运算过程为：
从索引 0 的位置开始向后跳，中间跳过了 `jump` 个数字，命中索引为 `jump + 1` 的数字，该数被敲出，并从该点起跳，以此类推，直到幸存 `left` 个数为止，然后返回幸存数之和。

**约束：**
1.  0 是第一个起跳点。
2.  起跳点和命中点之间间隔 `jump` 个数字，已被敲出的数字不计入在内。
3.  跳到末尾时无缝从头开始（循环查找），并可以多次循环。
4.  若起始时 `left > len(nums)` 则无需跳数处理过程，直接返回和。

## 二、输入描述

1.  第一行输入正整数数列，用逗号分隔。
2.  第二行输入跳数 `jump`。
3.  第三行输入幸存数量 `left`。

## 三、输出描述

输出幸存数之和。

## 四、备注

- 这是一个变种的**约瑟夫环（Josephus Problem）**问题。
- 与传统约瑟夫环的主要区别在于：**第一轮**跳跃的步长和**后续**跳跃的步长计算逻辑略有不同（基于题目描述的“从索引0起跳”与“从该点起跳”的具体语境）。
- **关键逻辑推导**：
    - **第一轮**：从索引 0 开始，跳过 `jump` 个，命中第 `jump + 1` 个。此时移动步长为 `jump + 1`。
    - **后续轮次**：元素被删除后，原位置被后面的元素填充（或逻辑上指针指向了下一个元素）。题目说明“从该点起跳”且“中间跳过 jump 个”。由于当前指针指向的元素已经是下一个待处理序列的第一个元素（即它自己也会被跳过），所以此时命中目标的偏移量仅为 `jump`。

## 五、用例

| 输入 | 输出 | 说明 |
| :--- | :--- | :--- |
| `1,2,3,4,5,6,7,8,9`<br>`4`<br>`3` | `13` | **初始数列**：`1,2,3,4,5,6,7,8,9` (Idx: 0-8)<br>**第1轮**：从Idx 0(Val 1)开始，跳过4个(2,3,4,5)，命中Idx 5(Val 6)。**删除6**。<br>剩余：`1,2,3,4,5,7,8,9`。下一轮起点逻辑上是Idx 5(Val 7)。<br>**第2轮**：从Idx 5(Val 7)开始，跳过4个(7,8,9,1)，命中Idx 1(Val 2)。**删除2**。<br>剩余：`1,3,4,5,7,8,9`。<br>**第3轮**：从Idx 1(Val 3)开始，跳过4个(3,4,5,7)，命中Idx 5(Val 8)。**删除8**。<br>...以此类推，最终剩余 `1, 3, 9`，和为 `13`。 |
| `1,2,3,4,5,6`<br>`10`<br>`10` | `21` | `left` (10) > `len` (6)，不进行删除，直接求和 `1+2+3+4+5+6 = 21`。 |

## 六、题目解析

### 核心思路

使用 **动态数组 (`ArrayList`)** 来模拟环形删除的过程。

1.  **处理特殊情况**：
    如果 `left >= nums.length`，直接返回原数组之和。

2.  **模拟删除过程**：
    * 维护一个当前指针 `idx`，初始为 0（但这题不需要显式从0遍历，直接计算第一次删除的位置）。
    * **第一次删除**：目标索引 `idx = (jump + 1) % currentSize`。这里 `+1` 是因为题目说从位置0开始，跳过 `jump` 个，意味着目标距离起点是 `jump + 1`。
    * **后续删除**：
        * 当删除了一个元素后，`ArrayList` 会自动将后面的元素前移。
        * 原来的 `idx` 此时指向的就是被删元素的**下一个元素**。
        * 根据题目逻辑“从该点起跳...中间跳过 jump 个”，现在的 `idx` 指向的元素就是第一个被跳过的元素。
        * 因此，新的删除位置是 `idx = (idx + jump) % currentSize`。
    * 重复上述过程直到数组大小等于 `left`。

3.  **求和**：
    计算剩余元素的总和并输出。

### 算法实现 (Java)

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) return;
        
        // 1. 读取输入
        String line = sc.nextLine();
        // 处理空输入等边缘情况
        if (line.trim().isEmpty()) {
            System.out.println(0);
            return;
        }
        
        int[] nums = Arrays.stream(line.split(","))
                           .map(String::trim)
                           .mapToInt(Integer::parseInt)
                           .toArray();

        if (!sc.hasNextLine()) return;
        int jump = Integer.parseInt(sc.nextLine().trim());

        if (!sc.hasNextLine()) return;
        int left = Integer.parseInt(sc.nextLine().trim());
        
        System.out.println(sumOfLeft(nums, jump, left));
    }

    public static int sumOfLeft(int[] nums, int jump, int left) {
        List<Integer> list = new ArrayList<>();
        int sum = 0;
        for (int num : nums) {
            list.add(num);
            sum += num;
        }

        // 约束4: left >= len 则直接返回总和
        if (left >= list.size()) {
            return sum;
        }

        // 模拟删除
        // 第一次删除的逻辑：从索引0开始，跳过jump个，命中第 jump+1 个
        // 注意：ArrayList索引从0开始，跳过jump个意味着下标增加 jump+1
        int idx = (jump + 1) % list.size();
        list.remove(idx);

        // 循环直到剩余 left 个
        while (list.size() > left) {
            // 后续删除逻辑：
            // 删除后，idx 指向了被删元素的后继者。
            // 题目意图是从这个后继者开始算，作为“跳过的第1个元素”。
            // 所以只需要再往后移 jump 步即可命中目标。
            idx = (idx + jump) % list.size();
            list.remove(idx);
        }

        // 计算剩余和
        int resultSum = 0;
        for (int num : list) {
            resultSum += num;
        }
        return resultSum;
    }
}
```
