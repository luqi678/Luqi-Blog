## 一、 题目描述

有一个简易内存池，内存按照大小粒度分类，每个粒度有若干个可用内存资源。用户会进行一系列内存申请，需要按需分配内存池中的资源，返回申请结果（成功或失败）列表。

**分配规则如下：**

1. 分配的内存要**大于等于**内存的申请量。
2. 存在满足需求的内存就必须分配，且优先分配**粒度最小**的（即最接近申请量的可用内存），但内存不能拆分使用。
3. 需要按**申请顺序**分配，先申请的先分配。
4. 有可用内存分配则申请结果为 `true`，没有可用则返回 `false`。

**注意：** 不考虑内存释放。

## 二、输入描述

输入为两行字符串：

1. **第一行为内存池资源列表：**
   - 包含内存粒度数据信息，粒度数据间用逗号分割。
   - 一个粒度信息内用冒号分割，冒号前为内存粒度大小，冒号后为数量。
   - 资源列表不大于 1024。
   - 每个粒度的数量不大于 4096。

2. **第二行为申请列表：**
   - 申请的内存大小间用逗号分割，申请列表不大于 100,000。

**示例：**
```text
64:2,128:1,32:4,1:128
50,36,64,128,127
```

## 三、输出描述

输出为内存池分配结果，用逗号连接的布尔字符串。

**示例：**
```text
true,true,true,false,false
```

## 四、用例

### 用例1
**输入：**
```text
64:2,128:1,32:4,1:128
50,36,64,128,127
```
**输出：**
```text
true,true,true,false,false
```

### 用例2
**输入：**
```text
10:1
11
```
**输出：**
```text
false
```

## 五、备注
1. 内存分配是**不可逆**的（不考虑释放）。
2. 查找满足条件（>=申请量）的**最小**粒度。

## 六、题目解析

### 1. 题目分析
本题的核心在于**贪心策略**和**快速查找**。
*   我们需要在现有的内存粒度中，找到第一个满足 `粒度 >= 申请大小` 的项。
*   由于要求“粒度最小”，我们需要对内存粒度进行升序排列。
*   申请是按顺序处理的，一旦分配，该粒度的可用数量减一。

### 2. 解题思路
1.  **数据解析：**
    *   将第一行输入的资源列表解析并存储。由于需要按粒度大小排序并频繁查找“大于等于某个值的最小值”，在 Java 中使用 `TreeMap<Integer, Integer>` 是最佳选择，其中 `key` 是内存粒度，`value` 是该粒度的剩余数量。
2.  **分配过程：**
    *   遍历申请列表。
    *   对于每一个申请量 `req`：
        *   使用 `TreeMap.ceilingKey(req)` 方法寻找大于或等于 `req` 的最小键。
        *   如果找不到（返回 `null`），则该次申请失败，记录 `false`。
        *   如果找到了键 `key`：
            *   对应数量减 1。
            *   如果数量减为 0，从 `TreeMap` 中移除该键。
            *   记录 `true`。
3.  **结果输出：**
    *   将结果列表转换成以逗号分隔的字符串。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        if (!sc.hasNextLine()) return;
        String resourceStr = sc.nextLine();
        if (!sc.hasNextLine()) return;
        String requestStr = sc.nextLine();

        // 1. 解析内存池资源并存入 TreeMap (自动按粒度大小升序)
        TreeMap<Integer, Integer> pool = new TreeMap<>();
        String[] resources = resourceStr.split(",");
        for (String res : resources) {
            String[] parts = res.split(":");
            int size = Integer.parseInt(parts[0]);
            int count = Integer.parseInt(parts[1]);
            pool.put(size, pool.getOrDefault(size, 0) + count);
        }

        // 2. 解析申请列表
        String[] requests = requestStr.split(",");
        List<String> results = new ArrayList<>();

        // 3. 顺序处理申请
        for (String reqStr : requests) {
            int req = Integer.parseInt(reqStr);
            
            // 找到大于等于 req 的最小 key
            Integer bestFitKey = pool.ceilingKey(req);

            if (bestFitKey != null) {
                // 分配成功
                results.add("true");
                // 更新数量
                int remaining = pool.get(bestFitKey) - 1;
                if (remaining == 0) {
                    pool.remove(bestFitKey);
                } else {
                    pool.put(bestFitKey, remaining);
                }
            } else {
                // 找不到满足条件的内存
                results.add("false");
            }
        }

        // 4. 输出结果
        System.out.println(String.join(",", results));
    }
}
```

### 4. 复杂度分析
*   **时间复杂度：**
    *   解析资源列表：$O(N \log N)$ 或 $O(N)$，取决于资源种类数 $N$。
    *   处理申请列表：设有 $M$ 个申请，每个申请在 `TreeMap` 中查找和更新的复杂度为 $O(\log N)$。总复杂度为 $O(M \log N)$。
    *   考虑到题目范围 $M \le 100,000$，$N \le 1024$，此算法性能优秀。
*   **空间复杂度：**
    *   $O(N + M)$，用于存储内存池状态和申请结果列表。
