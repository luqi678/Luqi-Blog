## 二、 题目描述

某部门计划通过结队编程来进行项目开发，已知该部门有 $N$ 名员工，每个员工有独一无二的职级。每三个员工形成一个小组进行结队编程，结队分组规则如下：

从部门中选出序号分别为 $i$、$j$、$k$ 的 3 名员工，他们的职级分别为 $level[i]$，$level[j]$，$level[k]$。结队小组需满足：
- $level[i] < level[j] < level[k]$ **或者** $level[i] > level[j] > level[k]$
- 其中 $0 \le i < j < k < n$。

请你按上述条件计算可能组合的小组数量。同一员工可以参加多个小组。

## 二、输入描述

- 第一行输入：员工总数 $n$
- 第二行输入：按序号依次排列的员工的职级 $level$，中间用空格隔开。

**备注：**
- $1 \le n \le 6000$
- $1 \le level[i] \le 10^5$
- 职级是独一无二的。

## 三、输出描述

- 可能结队的小组数量（注意结果可能较大，需考虑长整型）。

## 四、用例

### 用例1
**输入**
```text
4
1 2 3 4
```
**输出**
```text
4
```
**说明**
可能结队成的组合：(1,2,3)、(1,2,4)、(1,3,4)、(2,3,4)。所有组合都满足 $level[i] < level[j] < level[k]$。

### 用例2
**输入**
```text
3
5 4 7
```
**输出**
```text
0
```
**说明**
根据结队条件，无法组建满足递增或递减的小组。

---

## 五、备注
1. $n$ 的规模达到 6000，简单的 $O(n^3)$ 暴力算法会超时（$6000^3 = 2.16 \times 10^{11}$）。
2. 结果数值可能超过 $2^{31}-1$，需使用 `long` 类型。

## 六、题目解析

### 1. 题目分析
题目要求统计满足特定顺序（全增或全减）的三元组 $(i, j, k)$ 的数量。
关键点在于：**固定中间元素 $j$**。

对于每一个员工 $j$（职级为 $level[j]$）：
- 如果我们知道在 $j$ 左侧有多少个职级比它小的员工（记作 $leftSmaller$）；
- 如果我们知道在 $j$ 右侧有多少个职级比它大的员工（记作 $rightGreater$）；
- 那么以 $j$ 为中间节点的递增三元组数量就是 $leftSmaller \times rightGreater$。

同理：
- 如果我们知道在 $j$ 左侧比它大的员工数（$leftGreater$）；
- 在 $j$ 右侧比它小的员工数（$rightSmaller$）；
- 那么以 $j$ 为中间节点的递减三元组数量就是 $leftGreater \times rightSmaller$。

总数 = $\sum_{j=1}^{n-2} (leftSmaller_j \times rightGreater_j + leftGreater_j \times rightSmaller_j)$。

### 2. 解题思路

由于 $n=6000$，我们可以使用两种方式：
1. **$O(n^2)$ 遍历**：对于每个 $j$，遍历 $0 \dots j-1$ 统计左侧，遍历 $j+1 \dots n-1$ 统计右侧。$6000^2 = 3.6 \times 10^7$，在 1s 限制内通常可以跑完。
2. **$O(n \log n)$ 树状数组 (Binary Indexed Tree)**：
   - 职级最高 $10^5$，可以先进行离散化，或者直接针对职级建立树状数组。
   - 第一次从左往右遍历，统计每个元素左侧比它小的个数。
   - 第二次从右往左遍历，统计每个元素右侧比它小的个数。
   - 这种方法在 $n$ 更大的情况下表现更优。

下面提供 $O(n^2)$ 的实现，代码简单且能通过机考。

### 3. 代码实现

#### Java8 实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        int[] levels = new int[n];
        for (int i = 0; i < n; i++) {
            levels[i] = sc.nextInt();
        }

        long totalCount = 0;

        // 枚举中间元素 j，范围是 [1, n-2]
        for (int j = 1; j < n - 1; j++) {
            long leftSmaller = 0;
            long leftGreater = 0;
            long rightSmaller = 0;
            long rightGreater = 0;

            // 统计左侧 i < j 的情况
            for (int i = 0; i < j; i++) {
                if (levels[i] < levels[j]) {
                    leftSmaller++;
                } else if (levels[i] > levels[j]) {
                    leftGreater++;
                }
            }

            // 统计右侧 k > j 的情况
            for (int k = j + 1; k < n; k++) {
                if (levels[k] < levels[j]) {
                    rightSmaller++;
                } else if (levels[k] > levels[j]) {
                    rightGreater++;
                }
            }

            // 组合数累加
            // 递增：leftSmaller * rightGreater
            // 递减：leftGreater * rightSmaller
            totalCount += (leftSmaller * rightGreater) + (leftGreater * rightSmaller);
        }

        System.out.println(totalCount);
    }
}
```

#### Python 实现 (高效版思路)
Python 由于解释器速度较慢，$O(n^2)$ 在 $6000$ 规模下可能略显吃力，建议使用简单的计数逻辑。

```python
import sys

def solve():
    try:
        line1 = sys.stdin.readline()
        if not line1:
            return
        n = int(line1.strip())
        line2 = sys.stdin.readline()
        if not line2:
            return
        levels = list(map(int, line2.split()))
    except EOFError:
        return

    ans = 0
    for j in range(1, n - 1):
        mid_val = levels[j]
        
        ls = 0
        lg = 0
        rs = 0
        rg = 0
        
        # 统计左边
        for i in range(j):
            if levels[i] < mid_val:
                ls += 1
            else:
                lg += 1
        
        # 统计右边
        for k in range(j + 1, n):
            if levels[k] < mid_val:
                rs += 1
            else:
                rg += 1
                
        ans += (ls * rg) + (lg * rs)
        
    print(ans)

if __name__ == "__main__":
    solve()
```

### 4. 复杂度分析
- **时间复杂度**：$O(n^2)$。外层循环遍历中间节点 $j$（$n$ 次），内层两个循环分别遍历 $j$ 左侧和右侧，合计也是 $n$ 次。
- **空间复杂度**：$O(n)$。用于存储职级数组。
