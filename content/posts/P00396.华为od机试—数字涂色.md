## 一、 题目描述

疫情过后，希望小学终于又重新开学了，三年二班开学第一天的任务是将后面的黑板报重新制作。

黑板上已经写上了 $N$ 个正整数，同学们需要给这每个数分别上一种颜色。

为了让黑板报既美观又有学习意义，老师要求**同种颜色的所有数都可以被这种颜色中最小的那个数整除**。

现在请你帮帮小朋友们，算算最少需要多少种颜色才能给这 $N$ 个数进行上色。

## 二、输入描述

第一行有一个正整数 $N$，表示数字的个数。
第二行有 $N$ 个 `int` 型数（保证输入数据在 $[1, 100]$ 范围中），表示黑板上各个正整数的值。

## 三、输出描述

输出只有一个整数，为最少需要的颜色种数。

## 四、用例

### 用例1
**输入**：
```
3
2 4 6
```
**输出**：
```
1
```
**说明**：所有数都能被 2 整除。

### 用例2
**输入**：
```
4
2 3 4 9
```
**输出**：
```
2
```
**说明**：2 与 4 涂一种颜色，4 能被 2 整除；3 与 9 涂另一种颜色，9 能被 3 整除。不能 4 个数涂同一个颜色，因为 3 与 9 不能被 2 整除。所以最少的颜色是两种。

## 五、备注
1. $N$ 的范围未明确，但数字本身在 $[1, 100]$。
2. 同一种颜色的数必须满足：该颜色中所有的数 $\pmod{\text{最小数}} == 0$。

## 六、题目解析

### 1. 题目分析
题目要求求出最少的颜色种类。规则是：同一组（颜色）中的所有数字必须能被该组中最小的那个数字整除。
这是一个典型的**贪心算法**问题。为了让一种颜色尽可能覆盖更多的数字，我们应该优先选择最小的数字作为“基准”（除数），因为更小的数更容易被其他数整除。

### 2. 解题思路
1.  **排序**：首先将输入的所有数字进行从小到大排序。
2.  **去重（可选）**：如果数组中有重复的数字，比如两个 `2`，其实第二个 `2` 肯定能被第一个 `2` 整除，所以重复数字不影响结果（或者说可以直接视为同色）。
3.  **贪心标记**：
    *   从数组中最小的、尚未涂色的数字开始，将其作为一种新颜色的“基底”。
    *   遍历其后面所有的数字，如果某个数字能被这个“基底”整除，则将其标记为“已涂色”。
    *   继续寻找下一个尚未涂色的最小数字，重复上述过程。
4.  **计数**：每开启一轮寻找，颜色计数器加 1。

### 3. 代码实现

#### Java8实现

```java
import java.util.Arrays;
import java.util.Scanner;
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取N
        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();
        
        // 读取数字并存入列表
        List<Integer> nums = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            nums.add(sc.nextInt());
        }
        
        // 1. 排序，从小到大
        Collections.sort(nums);
        
        // 2. 特殊情况处理：如果包含1，那么1可以整除任何数，只需要1种颜色
        if (nums.contains(1)) {
            System.out.println(1);
            return;
        }

        int colorCount = 0;
        // 用于标记该位置的数字是否已经涂色
        boolean[] colored = new boolean[n];

        // 3. 贪心涂色
        for (int i = 0; i < n; i++) {
            // 如果当前数字还没涂色
            if (!colored[i]) {
                colorCount++; // 开启一种新颜色
                int base = nums.get(i); // 以当前最小数为基准
                colored[i] = true;
                
                // 遍历后面的数，能整除基准的都涂上同一种颜色
                for (int j = i + 1; j < n; j++) {
                    if (!colored[j] && nums.get(j) % base == 0) {
                        colored[j] = true;
                    }
                }
            }
        }
        
        // 4. 输出结果
        System.out.println(colorCount);
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(N \log N + N^2)$。其中排序消耗 $O(N \log N)$，双重循环在最坏情况下（所有数互质）消耗 $O(N^2)$。考虑到数字范围在 100 以内，$N$ 不会极大，该复杂度完全可接受。
*   **空间复杂度**：$O(N)$。需要额外的空间存储数字列表和标记数组。
