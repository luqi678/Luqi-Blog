## 一、 题目描述

当小区通信设备上报告警时，系统会自动生成待处理的工单，华为工单调度系统需要根据不同的策略，调度外线工程师（FME）上站去修复工单对应的问题。

根据与运营商签订的合同，不同严重程度的工单被处理并修复的时长要求不同，这个要求被修复的时长我们称之为 SLA 时间。

假设华为和运营商 A 签订了运维合同，部署了一套调度系统，只有 1 个外线工程师（FME），每个工单根据问题严重程度会给一个评分，在 SLA 时间内完成修复的工单，华为获得工单评分对应的积分，超过 SLA 完成的工单不获得积分，但必须完成该工单。运营商最终会根据积分进行付款。

请你设计一种调度策略，根据现状得到调度结果完成所有工单，让这个外线工程师处理的工单获得的总积分最多。

假设从某个调度时刻开始，当前工单数量为 $N$，不会产生新的工单，每个工单处理修复耗时为 1 小时，请设计你的调度策略，完成业务目标。不考虑外线工程师在小区之间行驶的耗时。

## 二、输入描述

第一行为一个整数 $N$，表示工单的数量。 ($N \le 10^6$)

接下来 $N$ 行，每行包括两个整数。第一个整数表示工单的 SLA 时间（小时），第二个数表示该工单的积分。

## 三、输出描述

输出一个整数表示可以获得的最大积分。

## 四、用例

### 用例1

**输入：**
```text
7
1 6
1 7
3 2
3 1
2 4
2 5
6 1
```

**输出：**
```text
15
```

### 用例2

**输入：**
```text
3
1 10
2 20
2 30
```

**输出：**
```text
50
```

## 五、备注

- 工单数量 $N \le 10^6$
- SLA 时间 $\le 7 \times 10^5$
- 答案的最大积分不会超过 $2147483647$（即在 `int` 范围内，但求和过程建议使用 `long` 以防万一）。

## 六、题目解析

### 1. 题目分析

本题的核心是：**在有限的时间内，如何选择任务以获得最高积分**。
已知条件：
1. 只有一个工程师。
2. 每个工单耗时固定为 1 小时。
3. 每个工单有截止时间（SLA）和分值。
4. 目标是最大化在 SLA 内完成的工单分值总和。

### 2. 解题思路

这是一个典型的**贪心算法**问题，配合**最小堆（优先队列）**来实现。

1.  **排序**：首先将所有工单按照 SLA 时间从小到大排序。因为我们要按时间轴去考虑哪些工单可以被“挤进”计划。
2.  **贪心策略**：
    *   遍历排序后的工单。
    *   如果当前工单的 SLA 时间大于当前已选中的工单数量，说明当前时间点还能塞进这个工单，直接放入优先队列（最小堆）。
    *   如果当前工单的 SLA 时间等于或小于当前已选中的工单数量，说明时间已经满了。此时我们需要判断：如果当前工单的分数比已选工单中分数最低的那个（堆顶）还要高，那么我们“反悔”——抛弃掉那个分数最低的工单，把当前这个分数更高的工单加进去。
3.  **最终求和**：堆中剩下的所有工单分数之和即为最大积分。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
        // 使用快读，防止 N=10^6 时读取超时
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine());
        
        int[][] jobs = new int[n][2];
        for (int i = 0; i < n; i++) {
            String[] parts = br.readLine().split(" ");
            jobs[i][0] = Integer.parseInt(parts[0]); // SLA 时间
            jobs[i][1] = Integer.parseInt(parts[1]); // 积分
        }
        
        // 1. 按 SLA 时间从小到大排序
        Arrays.sort(jobs, (a, b) -> a[0] - b[0]);
        
        // 2. 最小堆，存储已选择工单的分数
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        
        for (int i = 0; i < n; i++) {
            int sla = jobs[i][0];
            int score = jobs[i][1];
            
            if (sla > pq.size()) {
                // 如果当前截止时间大于已选任务数，直接添加
                pq.offer(score);
            } else if (!pq.isEmpty() && score > pq.peek()) {
                // 如果时间满了，但当前任务积分更高，替换掉堆顶（最小积分任务）
                pq.poll();
                pq.offer(score);
            }
        }
        
        // 3. 计算堆中所有积分总和
        long totalScore = 0;
        while (!pq.isEmpty()) {
            totalScore += pq.poll();
        }
        
        System.out.println(totalScore);
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(N \log N)$。
    *   排序需要 $O(N \log N)$。
    *   遍历 $N$ 个任务，每次对优先队列的操作（插入/删除）是 $O(\log N)$。
*   **空间复杂度**：$O(N)$。用于存储任务数组和优先队列。
