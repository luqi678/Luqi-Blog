## 一、 题目描述

假设你是大型科技公司的数据分析师，负责分析公司内部员工的社交网络。你需要编写一个函数来计算每个员工的影响力分数。影响力分数定义为该员工直接和间接影响的员工数量。

## 二、输入描述

1. 第一行：$n$，表示员工总数。
2. 接下来 $n$ 行：表示员工的社交网络关系。
   - 第 $i$ 行（从 0 开始计数）包含员工 $i$ 直接影响的员工 ID 列表，ID 之间以空格分隔。
   - 如果某员工没有直接影响到任何人，则用 `*` 表示。

## 三、输出描述

输出一行，包含 $n$ 个整数，表示每个员工的影响力分数，以空格分隔。

## 四、用例

### 用例1

**输入：**
```text
4
1
2
3
*
```

**输出：**
```text
3 2 1 0
```

### 用例2

**输入：**
```text
5
1 2
3
4
*
*
```

**输出：**
```text
4 1 1 0 0
```

### 用例3

**输入：**
```text
6
1 2
3
4
5
0
*
```

**输出：**
```text
5 2 5 1 5 0
```

## 五、备注

- 员工总数 $n < 20$。
- 自身不算在影响力分数内。
- `*` 表示没有直接影响到的员工。

## 六、题目解析

### 1. 题目分析

本题考察的是**有向图的可达性**问题。
- 每个员工可以看作图中的一个节点。
- 如果员工 A 直接影响员工 B，则存在一条从 A 指向 B 的有向边。
- 影响力分数定义为该员工能到达的所有不同节点的总数（不包括自己）。

### 2. 解题思路

1. **建模**：使用邻接表（Adjacency List）来存储员工的影响关系。由于 $n < 20$，规模非常小，不需要复杂的图算法。
2. **搜索**：
   - 对于每一个员工 $i$（$0 \le i < n$），从该节点出发进行深度优先搜索（DFS）或广度优先搜索（BFS）。
   - 使用一个 `Set` 或者 `visited` 数组来记录搜索过程中访问过的所有节点。
   - 最终的影响力分数为：`搜索到的节点总数 - 1`（如果搜索包含了起始节点本身且自身不算分）或者直接计算除起始点外能到达的节点数。
3. **循环引用处理**：由于题目中可能存在 $4 \to 0 \to 1$ 这样的环状结构（如用例3），在 DFS 过程中必须记录已访问节点，防止陷入死循环。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        
        int n = Integer.parseInt(sc.nextLine().trim());
        List<List<Integer>> adj = new ArrayList<>();
        
        // 读取每个员工的影响关系
        for (int i = 0; i < n; i++) {
            String line = sc.nextLine().trim();
            List<Integer> neighbors = new ArrayList<>();
            if (!"*".equals(line)) {
                String[] parts = line.split("\\s+");
                for (String p : parts) {
                    neighbors.add(Integer.parseInt(p));
                }
            }
            adj.add(neighbors);
        }

        int[] result = new int[n];
        
        // 对每个员工计算影响力
        for (int i = 0; i < n; i++) {
            result[i] = getInfluenceScore(i, adj, n);
        }

        // 格式化输出
        for (int i = 0; i < n; i++) {
            System.out.print(result[i] + (i == n - 1 ? "" : " "));
        }
        System.out.println();
    }

    /**
     * 使用BFS或DFS计算从start节点出发能到达的节点数量（不含start）
     */
    private static int getInfluenceScore(int start, List<List<Integer>> adj, int n) {
        boolean[] visited = new boolean[n];
        Queue<Integer> queue = new LinkedList<>();
        
        queue.add(start);
        visited[start] = true;
        int count = 0;
        
        while (!queue.isEmpty()) {
            int curr = queue.poll();
            for (int neighbor : adj.get(curr)) {
                if (!visited[neighbor]) {
                    visited[neighbor] = true;
                    count++;
                    queue.add(neighbor);
                }
            }
        }
        return count;
    }
}
```

### 4. 复杂度分析

- **时间复杂度**：$O(n \times (n + e))$，其中 $n$ 是员工总数，$e$ 是边数（即社交关系数）。因为对每个员工都执行了一次遍历。由于 $n < 20$，总计算次数在几百次以内，效率极高。
- **空间复杂度**：$O(n + e)$，用于存储邻接表和访问数组。
