## 一、 题目描述

公司某部门软件教导团正在组织新员工每日打卡学习活动，他们开展这项学习活动已经一个月了，所以想统计下这个月优秀的打卡员工。

每个员工会对应一个id，每天的打卡记录记录当天打卡员工的id集合，一共30天。

请你实现代码帮助统计出打卡次数 **top 5** 的员工。

**排序规则：**
1. 打卡次数消耗最多的员工排在前面。
2. 假如打卡次数相同，将较早参与打卡的员工排在前面。
3. 如果开始参与打卡的时间还是一样，将id较小的员工排在前面。

**注：**
- 不考虑并列的情况，按规则返回前5名员工的id即可。
- 如果当月打卡的员工少于5人，按规则排序返回所有有打卡记录的员工id。
- 员工编号范围为 $0$ 到 $N-1$。

## 二、输入描述

- 第一行输入为新员工数量 $N$，表示新员工编号 id 为 $0$ 到 $N-1$，$N$ 的范围为 $[1, 100]$。
- 第二行输入为 30 个整数，表示每天打卡的员工数量，每天至少有 1 名员工打卡。
- 之后 30 行为每天打卡的员工 id 集合，id 不会重复，用空格隔开。

## 三、输出描述

按顺序输出打卡 top 5 员工的 id，用空格隔开。

## 五、备注
- $N$ 的范围为 $[1, 100]$。
- 打卡天数固定为 30 天。
- 每天打卡人数至少为 1。

## 六、题目解析

### 1. 题目分析
本题属于多条件排序问题。我们需要记录每个员工的三个关键维度：
1. **打卡总次数** (`count`)：次数越多，排名越靠前。
2. **首次打卡日期** (`firstDay`)：日期越早（索引越小），排名越靠前。
3. **员工ID** (`id`)：ID 越小，排名越靠前。

### 2. 解题思路
1. **初始化数据结构**：创建一个 `Employee` 类或者使用数组来记录每个 ID 的打卡信息。由于 $N$ 最大只有 100，我们可以直接开辟长度为 100 的数组。
2. **读取数据**：
    - 读取 $N$。
    - 读取 30 天中每天打卡的人数。
    - 循环 30 次（天数），在每一天中读取对应数量的 ID：
        - 更新该 ID 的打卡次数 `count++`。
        - 如果是该 ID 第一次出现，记录当前天数作为 `firstDay`。
3. **过滤与排序**：
    - 将所有有打卡记录的员工放入一个列表。
    - 使用自定义比较器（Comparator）按照上述三个规则进行排序。
4. **输出结果**：取排序后的前 5 名。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    static class Employee {
        int id;
        int count = 0;
        int firstDay = -1;

        public Employee(int id) {
            this.id = id;
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        if (!sc.hasNextInt()) return;
        int n = sc.nextInt();

        // 记录每一天应读入的员工数量
        int[] dailyCounts = new int[30];
        for (int i = 0; i < 30; i++) {
            dailyCounts[i] = sc.nextInt();
        }

        // 使用Map或数组存储员工信息，这里用数组更直观
        Employee[] employees = new Employee[n];
        for (int i = 0; i < n; i++) {
            employees[i] = new Employee(i);
        }

        // 处理30天的打卡数据
        for (int day = 0; day < 30; day++) {
            int numToday = dailyCounts[day];
            for (int j = 0; j < numToday; j++) {
                int id = sc.nextInt();
                if (id < n) {
                    employees[id].count++;
                    if (employees[id].firstDay == -1) {
                        employees[id].firstDay = day;
                    }
                }
            }
        }

        // 过滤出有打卡记录的员工
        List<Employee> list = new ArrayList<>();
        for (Employee e : employees) {
            if (e.count > 0) {
                list.add(e);
            }
        }

        // 多条件排序
        Collections.sort(list, (a, b) -> {
            if (a.count != b.count) {
                return b.count - a.count; // 次数降序
            }
            if (a.firstDay != b.firstDay) {
                return a.firstDay - b.firstDay; // 首次打卡时间升序
            }
            return a.id - b.id; // ID升序
        });

        // 输出前5名
        StringBuilder sb = new StringBuilder();
        int limit = Math.min(list.size(), 5);
        for (int i = 0; i < limit; i++) {
            sb.append(list.get(i).id);
            if (i < limit - 1) {
                sb.append(" ");
            }
        }
        System.out.println(sb.toString());
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(30 \times M + N \log N)$，其中 $M$ 是每天平均打卡人数，$N$ 是员工总数。由于 $N \le 100$，且天数固定为 30，整体运算量极小。
- **空间复杂度**：$O(N)$，用于存储员工的打卡统计信息。
