## 一、 题目描述

如果一个字符串正读和反读都一样（大小写敏感），则称它为一个「回文串」。例如：
- `leVel` 是一个「回文串」，因为它的正读和反读都是 `leVel`；
- `a` 也是「回文串」；
- `art` 不是一个「回文串」，因为它的反读 `tra` 与正读不同；
- `Level` 不是一个「回文串」，因为它的反读 `leveL` 与正读不同（因大小写敏感）。

给你一个仅包含大小写字母的字符串，请用这些字母构造出一个**最长的回文串**。若有多个最长的，返回其中**字典序最小**的回文串。

字符串中的每个位置的字母最多备用一次，也可以不用。

## 二、输入描述

一个仅包含大小写字母的字符串。

## 三、输出描述

输出构造出的最长回文串，若有多个最长的，输出字典序最小的那一个。

## 四、用例

### 用例1

**输入**
```text
abczcccddzz
```

**输出**
```text
ccdzazdcc
```

### 用例2

**输入**
```text
ABabBabA
```

**输出**
```text
ABabbaBA
```

## 五、备注

- 字符串长度 $1 \le length \le 1000$。
- 字符仅包含 'a'-'z' 和 'A'-'Z'。

## 六、题目解析

### 1. 题目分析
本题有两个核心要求：
1. **长度最长**：回文串由“成对的字符”和一个可选的“中心字符”组成。为了使长度最长，我们应该尽可能使用所有成对的字符。如果有任何字符在分配完成对后还有剩余，我们可以选一个作为中心。
2. **字典序最小**：在长度最长的前提下，我们要让字符串的字典序尽可能小。
   - 回文串的结构是：`左半部分 + (中心字符) + 右半部分`。
   - 要使整体字典序最小，`左半部分`的字符应该按 ASCII 码从小到大排列。
   - `中心字符` 应该在所有可作为中心的备选字符中选择 ASCII 码最小的那一个。

### 2. 解题思路
1. **统计频率**：使用哈希表或长度为 128 的数组统计每个字符出现的次数。
2. **提取成对字符**：遍历统计结果。对于每个字符，如果其出现次数为 $count$，则可以取 $count / 2$ 对字符放入回文串的左右两边。
3. **确定中心字符**：在统计过程中，如果某个字符出现次数是奇数，或者在取完对子后还有剩余，该字符就可以作为中心字符。为了保证字典序最小，我们在所有这些剩余字符中寻找 ASCII 码最小的一个。
4. **拼接结果**：
   - 将所有成对的字符按 ASCII 码升序排列，构造出回文串的左半部分。
   - 如果存在中心字符，将其接在左半部分后面。
   - 将左半部分翻转，接在最后面。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNext()) return;
        String s = sc.next();
        
        // 1. 统计字符频率
        int[] counts = new int[128];
        for (char c : s.toCharArray()) {
            counts[c]++;
        }
        
        StringBuilder leftPart = new StringBuilder();
        Character center = null;
        
        // 2. 构造左半部分 (字典序最小，所以从小到大遍历 ASCII)
        for (int i = 0; i < 128; i++) {
            int pairCount = counts[i] / 2;
            for (int j = 0; j < pairCount; j++) {
                leftPart.append((char) i);
            }
        }
        
        // 3. 寻找中心字符 (为了使长度最长且字典序最小)
        // 只要有任何字符剩下的数量是奇数，就说明可以放一个在中心
        // 找所有剩余字符中 ASCII 最小的一个
        for (int i = 0; i < 128; i++) {
            if (counts[i] % 2 == 1) {
                center = (char) i;
                break; // 找到第一个（最小的）就退出
            }
        }
        
        // 4. 拼接最终结果
        StringBuilder result = new StringBuilder();
        result.append(leftPart);
        if (center != null) {
            result.append(center);
        }
        // 翻转左半部分得到右半部分
        StringBuilder rightPart = new StringBuilder(leftPart).reverse();
        result.append(rightPart);
        
        System.out.println(result.toString());
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N + K)$，其中 $N$ 是字符串长度（用于统计频率），$K$ 是字符集大小（常数 128，用于遍历频率数组）。整体线性复杂度。
- **空间复杂度**：$O(N + K)$，用于存储字符频率和构造结果字符串。
