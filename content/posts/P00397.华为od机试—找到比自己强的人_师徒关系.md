## 一、 题目描述

给定一组师徒关系数组，例如 `[[2,1],[3,2]]`。
每组数据 `[a, b]` 表示排名为 `a` 的员工是排名为 `b` 的员工的导师。
数字代表排名，排名数字越小表示能力越强（例如排名 1 比排名 2 强）。
现在需要找出每个排名对应的导师，其拥有的**比自己强**的徒弟（包括直接徒弟和间接徒弟）的总数。

**注意：**
1. 徒弟的徒弟也是该导师的徒弟（间接徒弟）。
2. "比自己强" 意味着徒弟的排名数字小于导师的排名数字。

## 二、输入描述

输入为一个字符串，表示二维数组。
例如：`[[2,1],[3,2]]` 或 `[[2, 1], [3, 2]]`（可能包含空格）。

## 三、输出描述

输出为一个数组，表示从排名 1 到最大排名的每一位导师对应的“更强徒弟”数量。
例如：`[0,1,2]`

## 四、用例

### 用例1
**输入：**
`[[2,1],[3,2]]`

**输出：**
`[0,1,2]`

**说明：**
- 排名第 1 的员工：没有徒弟，比他强的徒弟数量为 0。
- 排名第 2 的员工：徒弟是排名第 1 的员工。1 < 2，所以比他强的徒弟数量为 1。
- 排名第 3 的员工：直接徒弟是 2，间接徒弟是 1。2 < 3 且 1 < 3，所以比他强的徒弟数量为 2。
- 最终输出为 `[0, 1, 2]`。

### 用例2
**输入：**
`[[5,2],[2,1]]`

**输出：**
`[0,1,0,0,2]`

**说明：**
- 最大排名是 5，因此输出数组长度为 5。
- 排名 1：无徒弟 -> 0
- 排名 2：徒弟 1，1 < 2 -> 1
- 排名 3：无徒弟 -> 0
- 排名 4：无徒弟 -> 0
- 排名 5：徒弟 2 和 1，均小于 5 -> 2

## 五、备注
1. 排名均为正整数。
2. 师徒关系可能构成多棵树（森林）。
3. 如果某个排名在输入中没有出现，且小于最大排名，则其对应的更强徒弟数量为 0。

## 六、题目解析

### 1. 题目分析
本题考察的是**图的遍历**（DFS/BFS）以及**字符串解析**。
核心逻辑在于：
- 构建一个有向图，由导师指向徒弟。
- 对于每一个排名 $R$，遍历其所有可达的节点（即所有子孙节点）。
- 在子孙节点中，统计排名值小于 $R$ 的节点个数。

### 2. 解题思路
1. **字符串处理**：输入的格式是 `[[a,b],[c,d]]`。我们需要将其解析为整数对。由于格式可能不标准（如空格或分隔符差异），可以使用正则表达式提取所有数字，然后两两配对。
2. **建图**：使用邻接表（`Map<Integer, List<Integer>>`）来存储师徒关系。同时记录出现过的最大排名 $maxRank$。
3. **遍历计数**：
   - 遍历从 1 到 $maxRank$ 的每一个排名 $i$。
   - 如果 $i$ 作为导师在图中存在，则从 $i$ 开始进行深度优先搜索（DFS）或广度优先搜索（BFS）。
   - 使用一个 `Set` 来记录已访问的徒弟，防止重复计数或陷入环路（虽然题目背景通常是 DAG）。
   - 在遍历过程中，如果发现徒弟的排名 $j < i$，则计数器加 1。
4. **输出格式**：将统计结果转换为 `[x,y,z]` 形式的字符串。

### 3. 代码实现

#### Java 实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) return;
        String input = sc.nextLine();

        // 1. 解析输入字符串
        // 提取所有数字
        List<Integer> allNums = new ArrayList<>();
        StringBuilder sb = new StringBuilder();
        for (char c : input.toCharArray()) {
            if (Character.isDigit(c)) {
                sb.append(c);
            } else {
                if (sb.length() > 0) {
                    allNums.add(Integer.parseInt(sb.toString()));
                    sb.setLength(0);
                }
            }
        }
        if (sb.length() > 0) allNums.add(Integer.parseInt(sb.toString()));

        // 2. 构建邻接表
        Map<Integer, List<Integer>> adj = new HashMap<>();
        int maxRank = 0;
        for (int i = 0; i < allNums.size(); i += 2) {
            int mentor = allNums.get(i);
            int student = allNums.get(i + 1);
            adj.computeIfAbsent(mentor, k -> new ArrayList<>()).add(student);
            maxRank = Math.max(maxRank, Math.max(mentor, student));
        }

        // 3. 计算每个排名的更强徒弟数
        int[] results = new int[maxRank];
        for (int i = 1; i <= maxRank; i++) {
            if (!adj.containsKey(i)) {
                results[i - 1] = 0;
                continue;
            }

            // DFS 寻找所有徒弟
            Set<Integer> descendants = new HashSet<>();
            dfs(i, adj, descendants);

            // 统计比自己强的
            int count = 0;
            for (int studentRank : descendants) {
                if (studentRank < i) {
                    count++;
                }
            }
            results[i - 1] = count;
        }

        // 4. 格式化输出
        StringBuilder output = new StringBuilder("[");
        for (int i = 0; i < results.length; i++) {
            output.append(results[i]);
            if (i < results.length - 1) {
                output.append(",");
            }
        }
        output.append("]");
        System.out.println(output.toString());
    }

    /**
     * 深度优先遍历获取所有后代节点
     */
    private static void dfs(int curr, Map<Integer, List<Integer>> adj, Set<Integer> visited) {
        if (!adj.containsKey(curr)) return;
        for (int neighbor : adj.get(curr)) {
            // 如果该徒弟还没被统计过
            if (visited.add(neighbor)) {
                dfs(neighbor, adj, visited);
            }
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N \cdot (V + E))$，其中 $N$ 是排名总数，$V$ 和 $E$ 是图的顶点和边数。在最坏情况下（一条链），复杂度可能接近 $O(N^2)$。对于华为机考的数据规模，通常可以接受。
- **空间复杂度**：$O(V + E)$，用于存储邻接表和访问集合。
