## 二、题目描述

给定字符串 A、B 和正整数 V，A 的长度与 B 的长度相等。请计算 A 中满足如下条件的最大连续子串的长度：

1. 该连续子串在 A 和 B 中的位置和长度均相同。
2. 该连续子串 $|A[i] – B[i]|$ 之和小于等于 V。其中 $|A[i] – B[i]|$ 表示两个字母 ASCII 码之差的绝对值。

## 二、输入描述

输入为三行：
1. 第一行为字符串 A，仅包含小写字符，1 <= A.length <= 1000。
2. 第二行为字符串 B，仅包含小写字符，1 <= B.length <= 1000。
3. 第三行为正整数 V，0 <= V <= 10000。

## 三、输出描述

字符串最大连续子串的长度，要求该子串 $|A[i] – B[i]|$ 之和小于等于 V。

## 四、用例

### 用例1

**输入**
```text
xxcdefg
cdefghi
5
```

**输出**
```text
2
```

**说明**
A 和 B 在相同位置的差值数组为：
|'x'-'c'| = 21, |'x'-'d'| = 20, |'c'-'e'| = 2, |'d'-'f'| = 2, |'e'-'g'| = 2, |'f'-'h'| = 2, |'g'-'i'| = 2
差值序列：[21, 20, 2, 2, 2, 2, 2]
V = 5。
满足条件的子串有 [2, 2]（和为4 <= 5），长度为 2。

### 用例2

**输入**
```text
abcd
abcd
10
```

**输出**
```text
4
```

**说明**
差值均为0，全长满足条件。

## 五、备注

1. 字符串长度相等，且范围在 1000 以内，意味着 $O(n^2)$ 或 $O(n)$ 的算法都可以通过。
2. V 的范围到 10000，需要注意累加和的计算。

## 六、题目解析

### 1. 题目分析
题目要求找一个最长的连续区间 $[i, j]$，使得在此区间内 $\sum_{k=i}^{j} |A[k] - B[k]| \le V$。
由于区间是连续的，且我们需要寻找的是“最大长度”，这具有明显的单调性（如果长度为 L 的窗口满足，不代表 L+1 满足，但如果一个固定起始点的窗口和已经超过 V，再往后加字符一定超过 V），这非常适合使用**滑动窗口（双指针）**来解决。

### 2. 解题思路
1. **预处理**：先计算出 A 和 B 对应位置字符 ASCII 码差值的绝对值，存入一个数组 `diffs`。
2. **初始化**：定义两个指针 `left` 和 `right` 都指向 0，定义当前窗口内的差值之和 `currentSum = 0`，以及最大长度 `maxLen = 0`。
3. **移动右边界**：
   - 将 `diffs[right]` 加入 `currentSum`。
4. **收缩左边界**：
   - 如果 `currentSum > V`，则不断将 `diffs[left]` 从 `currentSum` 中减去，并将 `left` 右移，直到 `currentSum <= V`。
5. **更新结果**：
   - 每次移动右边界并调整完左边界后，计算当前窗口长度 `right - left + 1`，并更新 `maxLen`。
6. **循环结束**：输出 `maxLen`。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 读取输入
        if (!sc.hasNextLine()) return;
        String a = sc.nextLine();
        if (!sc.hasNextLine()) return;
        String b = sc.nextLine();
        if (!sc.hasNextInt()) return;
        int v = sc.nextInt();
        
        int n = a.length();
        int[] diffs = new int[n];
        
        // 1. 预处理差值数组
        for (int i = 0; i < n; i++) {
            diffs[i] = Math.abs(a.charAt(i) - b.charAt(i));
        }
        
        // 2. 滑动窗口
        int left = 0;
        int currentSum = 0;
        int maxLen = 0;
        
        for (int right = 0; right < n; right++) {
            currentSum += diffs[right];
            
            // 3. 如果当前和超过阈值，收缩左窗口
            while (currentSum > v && left <= right) {
                currentSum -= diffs[left];
                left++;
            }
            
            // 4. 更新最大长度
            maxLen = Math.max(maxLen, right - left + 1);
        }
        
        // 输出结果
        System.out.println(maxLen);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(n)$。其中 $n$ 是字符串的长度。左右指针均只向右移动一次。
- **空间复杂度**：$O(n)$。用于存储差值数组 `diffs`，如果不存数组直接在滑动窗口中计算，可以优化到 $O(1)$（不计输入字符串空间）。
