## 一、 题目描述

九宫格又是一款广为流传的游戏，起源于河图洛书。
游戏规则是：1到9九个数字放在3x3的格子中，要求每行、每列以及两个对角线上的三数之和都等于15。

现在有一种新的玩法，给九个不同的数字，将这九个数字放在3x3的格子中，要求每行、每列以及两个对角线上的三数之积相等（三阶积幻方）。

**规则说明：**
- 每行、每列以及两个对角线上的三数之积相等。
- 给出九个不同的数字，重新排列后满足要求。
- 排列输出格式：第1-3个数字为第一行，4-6为第二行，7-9为第三行。
- 输出所有满足要求的排列，并按字典序升序排序。

## 二、输入描述

九个不同的数字，每个数字之间用空格分开。
- $0 < \text{数字} < 10^7$
- $0 < \text{满足要求的乘积} < 2^{31}-1$

## 三、输出描述

九个数字所有满足要求的排列，每个数字之间用空格分开。每行输出一个满足要求的排列。
要求输出的排列升序排序。

## 五、备注
1. 用例保证至少有一种排列组合满足条件。
2. 乘积可能较大，计算总积时建议使用 `long` 或 `BigInteger`。

## 六、题目解析

### 1. 题目分析
本题要求构造一个“三阶积幻方”。核心属性是：
设 9 个数为 $x_0, x_1, \dots, x_8$，目标乘积为 $P$。
- $x_0 \cdot x_1 \cdot x_2 = P$ (行1)
- $x_3 \cdot x_4 \cdot x_5 = P$ (行2)
- $x_6 \cdot x_7 \cdot x_8 = P$ (行3)
- $x_0 \cdot x_3 \cdot x_6 = P$ (列1)
- $x_1 \cdot x_4 \cdot x_7 = P$ (列2)
- $x_2 \cdot x_5 \cdot x_8 = P$ (列3)
- $x_0 \cdot x_4 \cdot x_8 = P$ (主对角线)
- $x_2 \cdot x_4 \cdot x_6 = P$ (副对角线)

通过数学推导：
所有 9 个数的乘积 $T = (x_0 x_1 x_2) \cdot (x_3 x_4 x_5) \cdot (x_6 x_7 x_8) = P^3$。
因此，**目标乘积 $P$ 是这 9 个数乘积的立方根**。

### 2. 解题思路
1.  **预处理**：读取 9 个数并进行升序排序。排序是为了保证我们在全排列搜索时能自然地按字典序生成结果。
2.  **计算目标值**：计算 9 个数总乘积的立方根得到 $P$。由于数字较大，建议用 `BigInteger` 计算乘积再求立方根。
3.  **搜索与剪枝（DFS）**：
    按照 0 到 8 的索引顺序填入数字，并在关键点进行剪枝：
    - 填完 index 2：检查 `path[0]*path[1]*path[2] == P`
    - 填完 index 5：检查 `path[3]*path[4]*path[5] == P`
    - 填完 index 6：检查列1 `path[0]*path[3]*path[6] == P` 和 副对角线 `path[2]*path[4]*path[6] == P`
    - 填完 index 7：检查列2 `path[1]*path[4]*path[7] == P`
    - 填完 index 8：检查最后剩下的行3、列3和主对角线。

### 3. 代码实现

#### Java实现

```java
import java.util.*;
import java.math.BigInteger;

public class Main {
    static long[] nums = new long[9];
    static boolean[] used = new boolean[9];
    static long[] path = new long[9];
    static long targetP;
    static List<String> res = new ArrayList<>();

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        for (int i = 0; i < 9; i++) {
            if (sc.hasNextLong()) {
                nums[i] = sc.nextLong();
            }
        }
        Arrays.sort(nums);

        // 计算目标乘积 P = (所有数乘积)的1/3次方
        BigInteger totalProduct = BigInteger.ONE;
        for (long num : nums) {
            totalProduct = totalProduct.multiply(BigInteger.valueOf(num));
        }
        targetP = (long) Math.pow(totalProduct.doubleValue(), 1.0 / 3.0 + 1e-9);

        dfs(0);

        // 结果已经是字典序，直接输出
        for (String s : res) {
            System.out.println(s);
        }
    }

    public static void dfs(int index) {
        if (index == 9) {
            // 最后一次性检查最后一行、最后一列、主对角线
            if (path[6] * path[7] * path[8] == targetP &&
                path[2] * path[5] * path[8] == targetP &&
                path[0] * path[4] * path[8] == targetP) {
                
                StringBuilder sb = new StringBuilder();
                for (int i = 0; i < 9; i++) {
                    sb.append(path[i]).append(i == 8 ? "" : " ");
                }
                res.add(sb.toString());
            }
            return;
        }

        for (int i = 0; i < 9; i++) {
            if (used[i]) continue;

            path[index] = nums[i];
            
            // 剪枝逻辑
            if (index == 2) { // 第一行填满
                if (path[0] * path[1] * path[2] != targetP) continue;
            }
            if (index == 5) { // 第二行填满
                if (path[3] * path[4] * path[5] != targetP) continue;
            }
            if (index == 6) { 
                // 检查第一列(0,3,6)和副对角线(2,4,6)
                if (path[0] * path[3] * path[6] != targetP) continue;
                if (path[2] * path[4] * path[6] != targetP) continue;
            }
            if (index == 7) {
                // 检查第二列(1,4,7)
                if (path[1] * path[4] * path[7] != targetP) continue;
            }

            used[i] = true;
            dfs(index + 1);
            used[i] = false;
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：理论上限为 $9! = 362,880$。由于我们在第 2, 5, 6, 7 层都进行了极其严格的剪枝，实际搜索空间非常小。在现代计算机上通常在几毫秒内即可完成。
- **空间复杂度**：$O(1)$。除了存储结果的列表外，递归深度仅为 9。
