## 一、 题目描述

张三购买了一辆续航里程数达 1000 公里的某自动驾驶新能源车。某一天车辆充满电后，需从甲城出发前往距离 $D$ 公里远的乙城，全程走高速，车载导航提示沿途有 $N$ 个休息站均可提供充电服务，各休息站均可实时提供当前充电排队时间（小时）。

请协助规划时间最优的休息站充电方案，返回最短的旅行用时。

**规则说明：**
1. 高速上的行驶速度固定为 100 公里/小时。
2. 汽车可以将电完全用光，1000 公里续航的汽车按 100 公里/小时可以开 10 个小时。
3. 每次充电时间固定为 1 小时，完成后电量充满。
4. 各站点充电排队时间不会变化，充电排队过程不耗电。
5. 规划时不必考虑保留安全续航里程数。

## 二、输入描述

- 第一行表示甲乙两城的距离 $D$（单位：公里）。
- 第二行表示沿途休息站的数量 $N$。
- 第三行起，每行 2 个数据，分别表示休息站距离甲城的距离（公里），以及充电排队所需时间（小时）。（休息站按离出发点从近到远排序）。

**数据范围：**
- $0 \le D \le 1,000,000$，$D$ 是 100 的整数倍。
- $0 \le N \le 10,000$。

## 三、输出描述

- 旅程总计花费的最短时间（小时）。
- 若无法到达终点，则返回 -1。

## 四、用例

### 用例1
**输入：**
```text
1500
4
300 2
600 1
1000 0
1200 0
```
**输出：**
```text
16
```
**说明：**
- 最佳方案：只在第 3 个休息站（位置 1000）充电。
- 行程耗时：$1500 / 100 = 15$ 小时。
- 充电排队：0 小时，充电时长：1 小时。
- 总计：16 小时。

### 用例2
**输入：**
```text
800
2
300 0
600 0
```
**输出：**
```text
8
```
**说明：**
- 续航 1000 公里，直接到达，耗时 $800 / 100 = 8$ 小时。

## 五、备注
1. 行驶时间 = 总距离 / 100。由于距离 $D$ 是 100 的倍数，行驶时间恒为整数。
2. 充电总耗时 = 充电次数 × 1 + 排队总时间。
3. 这是一个在给定约束（续航里程）下的路径优化问题。

## 六、题目解析

### 1. 题目分析
汽车每跑 1000 公里必须充电。充电的行为会带来两个代价：**固定的 1 小时充电时间** + **该站点的排队时间**。
我们需要求的是：`总行驶时间 + 最小化的(充电固定耗时 + 排队耗时)`。
由于行驶时间 $D/100$ 是固定的，我们的目标转化为：在保证能走到终点的前提下，选择若干个充电站，使 `(每个站点的排队时间 + 1)` 之和最小。

### 2. 解题思路
这是一个典型的动态规划问题，且具有线性结构，可以通过单调队列进行优化。

**定义状态：**
- 设 $dp[i]$ 为到达第 $i$ 个休息站并**完成充电**后的最小额外耗时（不包括行驶时间）。
- 设 $dist[i]$ 为第 $i$ 个站点的距离，$wait[i]$ 为第 $i$ 个站点的排队时间。
- 增加一个虚拟起点：$dist[0] = 0, wait[0] = 0, dp[0] = 0$。

**转移方程：**
- 对于第 $i$ 个站点，它可以从前面的第 $j$ 个站点转移过来，前提是 $dist[i] - dist[j] \le 1000$。
- $dp[i] = (wait[i] + 1) + \min \{ dp[j] \}$，其中 $j < i$ 且 $dist[i] - dist[j] \le 1000$。

**终点计算：**
- 最终结果 = $D/100 + \min \{ dp[j] \}$，其中 $D - dist[j] \le 1000$。

**优化：**
- $N$ 的范围是 10,000，$O(N^2)$ 的暴力转移在极端情况下可能偏慢。
- 因为 $dist[i]$ 是递增的，对于每个 $i$，满足条件的 $j$ 是一个连续的滑动窗口。我们可以使用**单调队列**来维护窗口内的 $dp[j]$ 最小值，将复杂度降为 $O(N)$。

### 3. 代码实现

#### Java实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int D = sc.nextInt();
        if (!sc.hasNextInt()) {
            System.out.println(D / 100);
            return;
        }
        int N = sc.nextInt();

        // 存储站点信息：[距离, 排队时间]
        int[][] stations = new int[N + 1][2];
        stations[0][0] = 0; // 起点
        stations[0][1] = 0;
        for (int i = 1; i <= N; i++) {
            stations[i][0] = sc.nextInt();
            stations[i][1] = sc.nextInt();
        }

        // dp[i] 表示在第 i 个休息站充电完后的最小总额外时间
        long[] dp = new long[N + 1];
        Arrays.fill(dp, -1);
        dp[0] = 0;

        // 单调队列，存储索引，对应 dp 值单调递增
        Deque<Integer> deque = new ArrayDeque<>();
        deque.offer(0);

        for (int i = 1; i <= N; i++) {
            // 1. 移除超出续航范围（1000km）的站点
            while (!deque.isEmpty() && stations[i][0] - stations[deque.peekFirst()][0] > 1000) {
                deque.pollFirst();
            }

            // 2. 如果队列为空，说明无法到达当前站点
            if (deque.isEmpty()) {
                dp[i] = -1;
            } else {
                // 3. 转移：当前站点的额外时间 = 最小前置dp + 排队时间 + 1
                int bestJ = deque.peekFirst();
                dp[i] = dp[bestJ] + stations[i][1] + 1;

                // 4. 将当前 dp[i] 加入单调队列
                while (!deque.isEmpty() && dp[deque.peekLast()] >= dp[i]) {
                    deque.pollLast();
                }
                deque.offerLast(i);
            }
        }

        // 5. 计算到终点的最小耗时
        long minExtra = Long.MAX_VALUE;
        boolean reachable = false;
        
        // 检查直接从哪些站点（包括起点）可以到达终点
        for (int i = 0; i <= N; i++) {
            if (dp[i] != -1 && D - stations[i][0] <= 1000) {
                minExtra = Math.min(minExtra, dp[i]);
                reachable = true;
            }
        }

        if (!reachable) {
            System.out.println("-1");
        } else {
            System.out.println(D / 100 + minExtra);
        }
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(N)$。每个休息站最多入队一次、出队一次，计算 $dp$ 值为常数级。
- **空间复杂度**：$O(N)$。用于存储站点信息和 $dp$ 数组。
