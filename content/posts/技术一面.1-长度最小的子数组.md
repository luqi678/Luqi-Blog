## 一、 题目描述
给定一个含有 $n$ 个正整数的数组，和一个正整数 `target`。
找出该数组中，满足其和 $\ge target$ 的长度最小的**连续子数组**，并返回其长度。如果不存在符合条件的子数组，返回 0。

## 二、输入描述
*   第一行输入一个正整数 `target`。
*   第二行输入一个包含 $n$ 个正整数的数组 `nums`（通常以逗号或空格分隔）。

## 三、输出描述
*   输出一个整数，表示满足条件的最小子数组长度。

## 四、用例
### 用例1
**输入：** 
target = 7, nums = [2,3,1,2,4,3]
**输出：** 
2
**解释：** 子数组 [4,3] 的和大于等于 7，且长度为 2，是满足条件的最小子数组。

### 用例2
**输入：** 
target = 4, nums = [1,4,4]
**输出：** 
1
**解释：** 子数组 [4] 的和大于等于 4，且长度为 1。

### 用例3
**输入：** 
target = 11, nums = [1,1,1,1,1,1,1,1]
**输出：** 
0
**解释：** 数组中不存在和大于等于 11 的子数组。

## 五、备注
*   $1 \le target \le 10^9$
*   $1 \le nums.length \le 10^5$
*   $1 \le nums[i] \le 10^5$

## 六、题目解析
### 1. 题目分析
本题要求在给定数组中找到一个“最短”的“连续”子数组，且其内部元素的“和”达到一个阈值。
*   **关键词**：连续子数组、和、最小长度。
*   **直观解法**：暴力法。枚举所有可能的子数组起点和终点，计算和并对比长度。时间复杂度为 $O(n^2)$，在 $n=10^5$ 的数据规模下会超时。

### 2. 解题思路
由于数组中的元素均为**正整数**，子数组的和具有单调性（即窗口扩大，和一定增加；窗口缩小，和一定减少）。这种场景非常适合使用**滑动窗口（Sliding Window）**算法：
1.  定义两个指针 `left` 和 `right` 表示窗口的边界，初始均指向索引 0。
2.  移动右指针 `right`，将 `nums[right]` 累加到当前窗口和 `sum` 中。
3.  一旦 `sum >= target`：
    *   更新最小长度 `ans = min(ans, right - left + 1)`。
    *   尝试缩小窗口：将 `nums[left]` 从 `sum` 中减去，并将 `left` 右移，直到 `sum < target`。在这个过程中不断更新最小长度。
4.  重复上述步骤直到 `right` 遍历完整个数组。

### 3. 代码实现
#### Java8实现
```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        // 模拟输入处理
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) return;
        int target = sc.nextInt();
        
        // 处理数组输入，假设输入格式为 2 3 1 2 4 3
        // 实际面试中可能根据题目具体格式（如带中括号）调整解析逻辑
        int[] nums = null;
        if (sc.hasNext()) {
            sc.nextLine(); // 消耗掉target后的换行
            String line = sc.nextLine().replaceAll("\\[|\\]|,", " ").trim();
            nums = Arrays.stream(line.split("\\s+"))
                         .mapToInt(Integer::parseInt)
                         .toArray();
        }

        System.out.println(minSubArrayLen(target, nums));
    }

    public static int minSubArrayLen(int target, int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int n = nums.length;
        int minLen = Integer.MAX_VALUE;
        int left = 0;
        int sum = 0;

        for (int right = 0; right < n; right++) {
            sum += nums[right];
            
            // 当窗口和大于等于目标值时，尝试收缩左边界
            while (sum >= target) {
                minLen = Math.min(minLen, right - left + 1);
                sum -= nums[left];
                left++;
            }
        }

        return minLen == Integer.MAX_VALUE ? 0 : minLen;
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：$O(n)$。虽然代码中有 `while` 嵌套在 `for` 循环中，但每个元素最多被 `right` 指针访问一次，被 `left` 指针访问一次，总操作次数为 $2n$。
*   **空间复杂度**：$O(1)$。只需要常数级别的辅助变量（`left`, `sum`, `minLen` 等）。
