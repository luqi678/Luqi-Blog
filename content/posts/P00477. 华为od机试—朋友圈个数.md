## 一、 题目描述

给定一组朋友关系，统计一下该朋友关系网中的朋友圈个数。

**朋友圈的定义**：
1.  一个朋友圈至少由 **3个** 朋友组成。
2.  要求同一个朋友圈中的 **任意两个人** 都具有直接的朋友关系（即完全子图）。
3.  **不考虑子朋友圈**：即当发现 `[A,B,C,D]` 组成一个朋友圈时，`[A,B,C]`、`[A,B,D]` 等子朋友圈不单独计数（即求解极大完全子图）。

## 二、输入描述

1.  先输入一个数字 $N$ 代表关系的总数。
2.  后面每条关系一行，两个成员以逗号分隔，如 `A,B`。

## 三、输出描述

输出一个整数，表示整个关系网中朋友圈的个数。

**补充说明**：
1.  $Friends.length \ge 1$, $Friends.length \le 10^3$ (边数)。
2.  输入的总朋友个数 $\le 100$ (点数)。
3.  输入保证 `Friend[i][0]` 和 `Friend[i][1]` 一定不一样，且同一关系不会反向输入，即不会同时输入 `[A,B]` 和 `[B,A]`。

## 四、用例

**示例 1**

```text
输入：
3
A,B
A,C
B,D

输出：
0

说明：
没有构成3人及以上的两两互连的圈子。
```

**示例 2**

```text
输入：
7
A,B
A,C
B,C
A,D
D,E
B,D
A,E

输出：
3

说明：
存在三个朋友圈：{A,B,C}, {A,B,D}, {A,D,E}。
注意 {A,B,C,D} 不是朋友圈，因为 C 和 D 没有边。
```

**示例 3**

```text
输入：
6
A,B
A,C
B,C
A,D
B,D
D,C

输出：
1

说明：
{A,B,C,D} 构成了两两互连的极大集合。
虽然 {A,B,C} 也是两两互连，但它是 {A,B,C,D} 的子集，不单独计数。
```

## 五、备注

* 本题实际上是图论中寻找 **极大团 (Maximal Clique)** 的问题。
* 需要过滤掉节点数小于 3 的团。

## 六、题目解析

### 1. 核心概念

题目要求“任意两人都有直接关系”，这在图论中被称为 **团 (Clique)** 或者 **完全子图**。
题目要求“不考虑子朋友圈”，这意味着我们需要找的是 **极大团 (Maximal Clique)**，即不能再向其中添加任何一个节点使其仍保持为团的集合。

简单来说：
1.  **团**：图中一个顶点集合，集合内任意两个顶点之间都有边。
2.  **极大团**：一个团，如果再加入任何一个其他顶点，它就不再是团了。

### 2. 算法选择：Bron-Kerbosch 算法

由于题目限制点的数量 $N \le 100$，我们可以使用经典的 **Bron-Kerbosch 算法** 来列出图中所有的极大团。

该算法是一个递归回溯算法，维护三个集合：
* $R$ (Current Clique)：当前正在构建的团中的顶点集合。
* $P$ (Potential Candidates)：可能加入到 $R$ 中的候选顶点集合。
* $X$ (Excluded)：已经处理过、不应该再加入 $R$ 的顶点集合（用于去重和保证极大性）。

**算法步骤**：
1.  如果 $P$ 和 $X$ 都为空，说明 $R$ 是一个极大团。判断 $R$ 的大小是否 $\ge 3$，如果是，计数加 1。
2.  遍历 $P$ 中的每一个顶点 $v$：
    * 递归调用：`BronKerbosch(R + {v}, P ∩ N(v), X ∩ N(v))`。
        * $R + \{v\}$：将 $v$ 加入当前团。
        * $P \cap N(v)$：候选集变为“原候选集”与“$v$ 的邻居”的交集（保证加入的新点必须和 $R$ 中所有点相连）。
        * $X \cap N(v)$：排除集同理。
    * 回溯：从 $P$ 中移除 $v$，并将 $v$ 加入 $X$（表示包含 $v$ 的情况已经处理完了）。

### 3. 数据预处理

输入给出的是字符串（如 "A", "B"），为了方便矩阵或邻接表操作，我们需要：
1.  使用 `HashSet` 或 `TreeSet` 收集所有出现的名字。
2.  使用 `HashMap` 将名字映射为 `0` 到 `n-1` 的整数 ID。
3.  构建邻接矩阵 `adj[n][n]`，如果 `i` 和 `j` 是朋友，则 `adj[i][j] = true`。

### 4. 参考代码

#### Java 实现

```java
import java.util.*;

public class Main {
    static boolean[][] graph;
    static int nodeCount;
    static int resultCount = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNext()) return;
        
        int edges = Integer.parseInt(sc.nextLine().trim());
        
        // 用于存储所有边的临时列表
        List<String[]> edgeList = new ArrayList<>();
        // 用于给名字分配 ID
        Set<String> names = new HashSet<>();
        
        for (int i = 0; i < edges; i++) {
            String line = sc.nextLine();
            String[] parts = line.split(",");
            edgeList.add(parts);
            names.add(parts[0]);
            names.add(parts[1]);
        }
        
        // 名字转 ID 映射
        List<String> nameList = new ArrayList<>(names);
        Map<String, Integer> nameToId = new HashMap<>();
        for (int i = 0; i < nameList.size(); i++) {
            nameToId.put(nameList.get(i), i);
        }
        
        nodeCount = names.size();
        graph = new boolean[nodeCount][nodeCount];
        
        // 构建邻接矩阵
        for (String[] edge : edgeList) {
            int u = nameToId.get(edge[0]);
            int v = nameToId.get(edge[1]);
            graph[u][v] = true;
            graph[v][u] = true;
        }
        
        // Bron-Kerbosch 算法所需的集合
        // R: 当前团中的节点
        List<Integer> R = new ArrayList<>();
        // P: 候选节点 (初始为所有节点)
        List<Integer> P = new ArrayList<>();
        for (int i = 0; i < nodeCount; i++) {
            P.add(i);
        }
        // X: 已排除节点
        List<Integer> X = new ArrayList<>();
        
        bronKerbosch(R, P, X);
        
        System.out.println(resultCount);
    }

    /**
     * Bron-Kerbosch 算法寻找所有极大团
     * @param R 当前团集合
     * @param P 候选集合
     * @param X 排除集合
     */
    static void bronKerbosch(List<Integer> R, List<Integer> P, List<Integer> X) {
        // 如果 P 和 X 都为空，说明 R 是极大团
        if (P.isEmpty() && X.isEmpty()) {
            if (R.size() >= 3) {
                resultCount++;
            }
            return;
        }

        // 遍历 P 的副本，因为在循环中会修改 P
        List<Integer> PCopy = new ArrayList<>(P);
        
        for (Integer v : PCopy) {
            List<Integer> newR = new ArrayList<>(R);
            newR.add(v);
            
            // newP = P ∩ Neighbors(v)
            List<Integer> newP = new ArrayList<>();
            for (Integer pNode : P) {
                if (graph[v][pNode]) {
                    newP.add(pNode);
                }
            }
            
            // newX = X ∩ Neighbors(v)
            List<Integer> newX = new ArrayList<>();
            for (Integer xNode : X) {
                if (graph[v][xNode]) {
                    newX.add(xNode);
                }
            }
            
            bronKerbosch(newR, newP, newX);
            
            // 回溯：从 P 中移除 v，加入 X
            P.remove(v);
            X.add(v);
        }
    }
}
```

#### Python 实现

```python
import sys

def solve():
    try:
        input_data = sys.stdin.read().splitlines()
    except Exception:
        return

    if not input_data:
        return

    # 读取边数
    try:
        n_edges = int(input_data[0].strip())
    except ValueError:
        return

    edges = []
    names = set()
    
    # 解析所有边
    for i in range(1, n_edges + 1):
        line = input_data[i].strip()
        if not line:
            continue
        u_name, v_name = line.split(',')
        edges.append((u_name, v_name))
        names.add(u_name)
        names.add(v_name)

    # 名字映射到 ID
    sorted_names = sorted(list(names))
    name_to_id = {name: i for i, name in enumerate(sorted_names)}
    node_count = len(names)
    
    # 构建邻接矩阵 (实际上用邻接集合更方便求交集)
    adj = {i: set() for i in range(node_count)}
    
    for u_name, v_name in edges:
        u = name_to_id[u_name]
        v = name_to_id[v_name]
        adj[u].add(v)
        adj[v].add(u)

    result_count = 0

    # Bron-Kerbosch 算法
    def bron_kerbosch(R, P, X):
        nonlocal result_count
        if not P and not X:
            if len(R) >= 3:
                result_count += 1
            return

        # 这里的 P 是 set，无法直接遍历同时修改，需要转 tuple 或 list
        for v in list(P):
            # 递归：R U {v}, P ∩ N(v), X ∩ N(v)
            bron_kerbosch(
                R | {v},
                P & adj[v],
                X & adj[v]
            )
            # 回溯
            P.remove(v)
            X.add(v)

    # 初始调用
    # R = 空集
    # P = 所有节点
    # X = 空集
    initial_P = set(range(node_count))
    bron_kerbosch(set(), initial_P, set())
    
    print(result_count)

if __name__ == "__main__":
    solve()
```
