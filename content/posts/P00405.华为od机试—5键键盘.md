## 一、 题目描述

有一个特殊的5键键盘，上面有 `a`，`ctrl-c`，`ctrl-x`，`ctrl-v`，`ctrl-a` 五个键。

1. **a 键**：在屏幕上输出一个字母 `a`。
2. **ctrl-c**：将当前选择的字母复制到剪贴板。
3. **ctrl-x**：将当前选择的字母复制到剪贴板，并清空选择的字母。
4. **ctrl-v**：将剪贴板里的字母输出到屏幕。
5. **ctrl-a**：选择当前屏幕上的所有字母。

**注意：**
- 剪贴板初始为空，新的内容被复制到剪贴板时会覆盖原来的内容。
- 当屏幕上没有字母时，`ctrl-a` 无效。
- 当没有选择字母时，`ctrl-c` 和 `ctrl-x` 无效。
- 当有字母被选择时，`a` 和 `ctrl-v` 这两个有输出功能的键会先清空选择的字母，再进行输出。

给定一系列键盘输入，输出最终屏幕上字母的数量。

## 二、输入描述

输入为一行，用数字 **1 2 3 4 5** 代表 `a`，`ctrl-c`，`ctrl-x`，`ctrl-v`，`ctrl-a` 五个键的输入，数字用空格分隔。

## 三、输出描述

输出一个数字，为最终屏幕上字母的数量。

## 四、用例

### 用例1
**输入**
```text
1 1 1
```
**输出**
```text
3
```
**说明**
连续键入3个a，故屏幕上字母的长度为3。

### 用例2
**输入**
```text
1 1 5 1 5 2 4 4
```
**输出**
```text
2
```
**说明**
1. 输入两个 `a`：屏幕 `aa`，选择 0，剪贴板 0。
2. `5` (ctrl-a)：屏幕 `aa`，选择 2 (aa)，剪贴板 0。
3. `1` (a)：清空选择的 2 个 `a`，输出 1 个 `a`：屏幕 `a`，选择 0，剪贴板 0。
4. `5` (ctrl-a)：屏幕 `a`，选择 1 (a)，剪贴板 0。
5. `2` (ctrl-c)：屏幕 `a`，选择 1，剪贴板 1 (a)。
6. `4` (ctrl-v)：清空选择的 1 个 `a`，输出剪贴板的 1 个 `a`：屏幕 `a`，选择 0，剪贴板 1。
7. `4` (ctrl-v)：直接输出剪贴板的 1 个 `a`：屏幕 `aa`，选择 0，剪贴板 1。
最终长度为 2。

## 五、备注
- 输入指令的个数在 $[1, 10000]$ 范围内。
- 最终结果可能较大，但在模拟过程中只需记录数量。

## 六、题目解析

### 1. 题目分析
本题是一道典型的**模拟题**。我们需要维护三个核心状态：
1. `screenCount`: 当前屏幕上字母的数量。
2. `selectionCount`: 当前被选中的字母数量。
3. `clipboardCount`: 当前剪贴板中字母的数量。

### 2. 解题思路
根据题目描述，逐一处理每个按键：
- **数字 1 (a)**: 
  - 如果有选中（`selectionCount > 0`），先减去选中的数量，然后加 1。
  - 重置选中数量为 0。
- **数字 2 (ctrl-c)**:
  - 如果有选中，更新 `clipboardCount = selectionCount`。
- **数字 3 (ctrl-x)**:
  - 如果有选中，更新 `clipboardCount = selectionCount`，屏幕数量 `screenCount -= selectionCount`，重置 `selectionCount = 0`。
- **数字 4 (ctrl-v)**:
  - 如果有选中，先 `screenCount -= selectionCount`，重置 `selectionCount = 0`。
  - 然后 `screenCount += clipboardCount`。
- **数字 5 (ctrl-a)**:
  - `selectionCount = screenCount`。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextLine()) return;
        
        String line = sc.nextLine();
        String[] ops = line.split(" ");
        
        long screenCount = 0;    // 屏幕上的字母数
        long selectionCount = 0; // 选中的字母数
        long clipboardCount = 0; // 剪贴板中的字母数
        
        for (String op : ops) {
            switch (op) {
                case "1": // a
                    if (selectionCount > 0) {
                        screenCount -= selectionCount;
                        selectionCount = 0;
                    }
                    screenCount++;
                    break;
                case "2": // ctrl-c
                    if (selectionCount > 0) {
                        clipboardCount = selectionCount;
                    }
                    break;
                case "3": // ctrl-x
                    if (selectionCount > 0) {
                        clipboardCount = selectionCount;
                        screenCount -= selectionCount;
                        selectionCount = 0;
                    }
                    break;
                case "4": // ctrl-v
                    if (selectionCount > 0) {
                        screenCount -= selectionCount;
                        selectionCount = 0;
                    }
                    screenCount += clipboardCount;
                    break;
                case "5": // ctrl-a
                    selectionCount = screenCount;
                    break;
                default:
                    break;
            }
        }
        
        System.out.println(screenCount);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(n)$，其中 $n$ 是输入指令的数量。我们只需要遍历一次操作序列。
- **空间复杂度**：$O(n)$，用于存储输入的指令字符串数组。如果使用 `Scanner.next()` 逐个读取，可以将空间复杂度降低到 $O(1)$。
