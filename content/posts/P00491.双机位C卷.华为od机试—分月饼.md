## 二、 题目描述

中秋节，公司分月饼，m个员工，买了n个月饼，m <= n。
每个员工至少分1个月饼，但可以分多个。

分配规则如下：
1. 单人分到最多月饼的个数为 Max1，单人分到第二多月饼的个数是 Max2，则 $Max1 - Max2 \le 3$。
2. 同理，单人分到第 $i$ 多月饼的个数是 $Max(i)$，单人分到第 $i+1$ 多月饼的个数是 $Max(i+1)$，则 $Max(i) - Max(i+1) \le 3$。
3. 请问有多少种分月饼的方法？

**注：** $1+3$ 和 $3+1$ 算同一种分法。

## 二、输入描述

第一行输入两个整数 `m` 和 `n`，表示 `m` 个员工，`n` 个月饼。
限制：$1 \le m \le n \le 30$（通常 OD 考题此范围适用）。

## 三、输出描述

输出有多少种月饼分法。

## 四、用例

### 用例1

**输入：**
```
2 4
```

**输出：**
```
2
```

**说明：**
4 = 1 + 3（3 - 1 = 2 $\le$ 3，符合条件）
4 = 2 + 2（2 - 2 = 0 $\le$ 3，符合条件）
共2种。

### 用例2

**输入：**
```
3 12
```

**输出：**
```
6
```

**说明：**
可能的分配方案（非降序排列）：
1. 2, 3, 7 (不符合, 7-3=4 > 3)
2. 2, 4, 6 (符合)
3. 2, 5, 5 (符合)
4. 3, 3, 6 (符合)
5. 3, 4, 5 (符合)
6. 4, 4, 4 (符合)
...以此类推计算出总数。

## 五、备注

1. 题目要求方案不重叠，即 $1+3$ 和 $3+1$ 是一样的，我们可以通过规定分配序列为**非降序**来去重。
2. 核心约束：$a_{i} - a_{i-1} \le 3$ 且 $a_{i} \ge a_{i-1}$。

## 六、题目解析

### 1. 题目分析
*   员工数量 $m$，月饼数量 $n$。
*   每人至少 1 个月饼。
*   有序序列 $a_1, a_2, ..., a_m$ 需满足：
    *   $\sum_{i=1}^{m} a_i = n$
    *   $1 \le a_1 \le a_2 \le ... \le a_m$
    *   $a_{i} - a_{i-1} \le 3$ （对于 $i \in [2, m]$）

### 2. 解题思路
本题可以使用 **DFS (深度优先搜索)** 来解决，为了提高效率，需要加入合理的**剪枝**。

**递归函数设计：**
`dfs(index, min_val, remaining_n)`
*   `index`: 当前正在分给第几个员工（从 0 开始）。
*   `min_val`: 当前员工最少能拿多少个月饼。由于是非降序，他至少要拿和前一个人一样多的月饼。
*   `remaining_n`: 剩下还没分配的月饼总数。

**剪枝与约束：**
1.  **最大值限制：** 当前员工能拿的最大月饼数，除了受限于 `min_val + 3`，还受限于“剩下的月饼必须够分给后面的人”。
2.  **平均值剪枝：** 如果剩下的月饼平摊到后面的人身上，平均值超过了 `min_val + 3` 或者还没达到 `min_val`，说明当前路径无效。
3.  **最后一名员工：** 当分配到最后一名员工时，他必须拿走所有剩余的 `remaining_n`。只需判断此时的 `remaining_n` 是否满足 $min\_val \le remaining\_n \le min\_val + 3$。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    private static int m;
    private static int n;
    private static int count = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (sc.hasNextInt()) {
            m = sc.nextInt();
            n = sc.nextInt();
            
            // 特殊情况处理
            if (m == 1) {
                System.out.println(1);
                return;
            }

            // 从第1个员工开始分配，最少拿1个，总共n个
            // dfs(员工索引, 该员工最少拿多少, 剩余月饼数)
            dfs(0, 1, n);
            System.out.println(count);
        }
    }

    /**
     * @param personIdx 当前分配的员工索引
     * @param minVal    当前员工至少要拿的月饼数 (a_i >= a_{i-1})
     * @param rest      剩余月饼总数
     */
    private static void dfs(int personIdx, int minVal, int rest) {
        // 分配到最后一个人
        if (personIdx == m - 1) {
            // 最后一个人必须拿走全部 rest
            // 且满足与前一个人的差值在 [0, 3] 之间
            if (rest >= minVal && rest <= minVal + 3) {
                count++;
            }
            return;
        }

        // 当前员工可以拿的数量区间 [minVal, minVal + 3]
        // 同时还要受限于剩余月饼数必须能让后面的人每人至少拿 i 个
        int maxPossible = minVal + 3;
        
        for (int i = minVal; i <= maxPossible; i++) {
            // 剩余人数 (不包括当前这一个)
            int remainingPeople = m - 1 - personIdx;
            
            // 剪枝：如果当前人拿了i个，剩下 rest - i
            // 后面的人每个人至少要拿 i 个，所以 rest - i 必须 >= remainingPeople * i
            if (rest - i < remainingPeople * i) {
                break;
            }
            
            // 剪枝：后面的人每个人最多拿 i + 3 个（这里其实是一个弱约束，可以进一步优化）
            // 实际上由于是连跳，i+3, i+6... 比较复杂，只需保证 rest-i 不会太大到后面无法消耗即可
            if (rest - i > remainingPeople * (i + 3 * remainingPeople)) {
                // 这是一个较松的边界，最极端的非降序情况是 i, i+3, i+6...
                // 如果 rest-i 大于这个等差数列的和，则无法满足条件
                continue; 
            }

            dfs(personIdx + 1, i, rest - i);
        }
    }
}
```

### 4. 复杂度分析
*   **时间复杂度**：由于 $N, M$ 的范围较小（最大30），且通过非降序和差值 $\le 3$ 的强剪枝，搜索空间被极大地压缩。实际运行效率非常高，接近 $O(\text{可行方案数})$。
*   **空间复杂度**：主要为递归栈深度，最大为 $O(M)$。
