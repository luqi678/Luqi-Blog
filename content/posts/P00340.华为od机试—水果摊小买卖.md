## 二、 题目描述

小王手里有点闲钱，想着做点卖水果的小买卖。给出两个数组 m、n，用 m[i] 代表第 i 个水果的成本价，n[i] 代表第 i 个水果能卖出的价钱。假如现在有本钱 k，试问最后最多能赚多少钱？

**说明：**
1. 每种水果只需买一次，只能卖一次。
2. 数组 m、n 大小不超过 50。
3. 数组元素为正整数，不超过 1000。

## 二、 输入描述

1. 第一行：逗号分隔的数组 m 的元素值（成本价）。
2. 第二行：逗号分隔的数组 n 的元素值（卖出价）。
3. 第三行：本钱 k。

## 三、 输出描述

最多能赚取多少钱（即最终手里的总钱数）。

## 四、 用例

### 用例1

**输入**
```text
4,2,6,4
5,3,8,7
15
```

**输出**
```text
22
```

**说明**
本钱 15。
1. 水果 1：成本 2，售价 3，利润 1。买入后剩余 13，卖出后总钱数 16。
2. 水果 0：成本 4，售价 5，利润 1。买入后剩余 12，卖出后总钱数 17。
3. 水果 3：成本 4，售价 7，利润 3。买入后剩余 13，卖出后总钱数 20。
4. 水果 2：成本 6，售价 8，利润 2。买入后剩余 14，卖出后总钱数 22。
总共赚了 7 元，最终金额 22。

### 用例2

**输入**
```text
10,20,30
15,25,35
5
```

**输出**
```text
5
```

**说明**
本钱只有 5，而所有水果成本都大于 5，无法购买任何水果，最后还是 5。

## 五、 备注

1. 水果只有在成本小于等于当前本钱时才能购买。
2. 卖出后，本钱会增加（本钱 = 原本钱 - 成本 + 售价）。
3. 目标是最大化最终的总钱数。

## 六、 题目解析

### 1. 题目分析
这是一个经典的贪心算法问题，类似于 LeetCode 上的 "IPO" 问题。
我们需要在现有的本钱下，不断选择能买得起且能盈利的水果。由于每种水果只能买卖一次，为了能买到更多的水果，我们应该优先考虑那些**目前能买得起**且**利润为正**的水果。

### 2. 解题思路
1.  **数据结构**：将每个水果的成本 `m[i]` 和售价 `n[i]` 组合在一起。
2.  **预处理**：过滤掉那些不赚钱（售价 <= 成本）的水果，因为卖它们只会亏本或不赚，对增加总钱数没有帮助。
3.  **排序**：将剩余的水果按照**成本价从小到大**进行排序。
    *   为什么要按成本排序？因为本钱是有限的，只有本钱达到了成本门槛才能启动交易。从小成本开始交易，可以快速累积本钱，从而有机会购买后续成本更高的水果。
4.  **遍历与模拟**：
    *   遍历排序后的水果列表。
    *   如果当前本钱 `k` 大于等于当前水果的成本，则买入并立即卖出，更新本钱 `k = k + (售价 - 成本)`。
    *   如果当前本钱连成本最低的还没买过的水果都买不起，则无法继续，结束循环。
5.  **输出结果**：最终的 `k` 即为所求。

### 3. 代码实现

#### Java8实现

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 处理输入
        if (!sc.hasNextLine()) return;
        String[] mStr = sc.nextLine().split(",");
        if (!sc.hasNextLine()) return;
        String[] nStr = sc.nextLine().split(",");
        if (!sc.hasNextLine()) return;
        int k = Integer.parseInt(sc.nextLine().trim());

        int len = mStr.length;
        int[] m = new int[len];
        int[] n = new int[len];
        for (int i = 0; i < len; i++) {
            m[i] = Integer.parseInt(mStr[i].trim());
            n[i] = Integer.parseInt(nStr[i].trim());
        }

        // 将水果包装成对象或数组
        List<Fruit> fruits = new ArrayList<>();
        for (int i = 0; i < len; i++) {
            // 只考虑有利润的水果
            if (n[i] > m[i]) {
                fruits.add(new Fruit(m[i], n[i] - m[i]));
            }
        }

        // 按照成本从小到大排序
        Collections.sort(fruits, (a, b) -> a.cost - b.cost);

        // 贪心策略：只要买得起就买
        // 注意：因为成本是从小到大排的，本钱是在不断增加的，
        // 只要前面的能买起，后面的也有机会买起。
        for (Fruit f : fruits) {
            if (k >= f.cost) {
                k += f.profit;
            } else {
                // 因为是按成本排序的，如果当前的买不起，后面的成本更高也一定买不起
                break;
            }
        }

        System.out.println(k);
    }

    static class Fruit {
        int cost;
        int profit;

        Fruit(int cost, int profit) {
            this.cost = cost;
            this.profit = profit;
        }
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(N \log N)$，其中 $N$ 是水果的数量。主要消耗在对水果进行排序上。遍历水果列表的时间复杂度为 $O(N)$。
*   **空间复杂度**：$O(N)$，用于存储水果的成本和利润信息。
