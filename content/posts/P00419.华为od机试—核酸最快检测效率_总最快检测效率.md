## 一、 题目描述

在系统、网络均正常的情况下组织核酸采样员和志愿者对人群进行核酸检测筛查。每名采样员的效率不同，采样效率为 $N$ 人/小时。

由于外界变化，采样员的效率会以 $M$ 人/小时为粒度发生变化，$M$ 为采样效率浮动粒度，$M = N \times 10\%$。输入保证 $N \times 10\%$ 的结果为整数。

**采样员效率浮动规则：**
1. 采样员需要一名志愿者协助组织才能发挥正常效率（即 $N$ 人/小时）。
2. 在有一名志愿者的基础上，每增加一名志愿者，效率提升 $1M$，最多提升 $3M$（即最多增加到 3 名额外志愿者，共 4 名志愿者）。
3. 如果没有志愿者协助组织，效率下降 $2M$。

**问题：**
给定采样员人数、志愿者人数以及各采样员的基准效率值，求如何安排志愿者使得总检测效率最快？（总检测效率为各采样员效率值相加）。

## 二、输入描述

第一行：
- 第一个值：采样员人数 $S$，取值范围 $[1, 100]$。
- 第二个值：志愿者人数 $V$，取值范围 $[1, 500]$。

第二行：
- 各采样员基准效率值（单位：人/小时），取值范围 $[60, 600]$。
- 保证序列中每项值计算 $10\%$ 为整数。

## 三、输出描述

- 一个整数，代表总最快检测效率（单位：人/小时）。

## 四、用例

### 用例1
**输入：**
```
2 2
200 200
```
**输出：**
```
400
```
**说明：**
有两个采样员，两个志愿者。每人分配一个志愿者，发挥正常效率 $200 + 200 = 400$。如果一个分配 2 个，另一个分配 0 个，效率为 $(200+20) + (200-40) = 380$。

### 用例2
**输入：**
```
1 3
100
```
**输出：**
```
120
```
**说明：**
1个采样员，3个志愿者。分配1个志愿者效率100，分配2个110，分配3个120。最高120。

## 五、备注
- 采样员最多可配备 4 名志愿者（1名基础 + 3名提升）。超过 4 名志愿者对单名采样员无额外效率提升。

## 六、题目解析

### 1. 题目分析
本题属于典型的**资源分配问题**。我们需要将有限的“志愿者”分配给不同的“采样员”，以获得全局最优的“检测效率”。

每个采样员分配不同数量的志愿者 $k$ 得到的效率如下（设基准效率为 $E$，浮动粒度 $M = E \times 0.1$）：
- $k=0$：效率 = $E - 2M$
- $k=1$：效率 = $E$
- $k=2$：效率 = $E + M$
- $k=3$：效率 = $E + 2M$
- $k \ge 4$：效率 = $E + 3M$

### 2. 解题思路
由于采样员人数 $S \le 100$，志愿者人数 $V \le 500$，且每个采样员的选择支较少（0 到 4 个志愿者），可以使用**动态规划 (Dynamic Programming)** 求解。

**DP 状态定义：**
- $dp[i][j]$：表示前 $i$ 个采样员分配 $j$ 个志愿者所能达到的最大总效率。

**转移方程：**
对于第 $i$ 个采样员，我们可以尝试分配 $k$ 个志愿者 ($k \in \{0, 1, 2, 3, 4\}$)：
$dp[i][j] = \max(dp[i-1][j-k] + \text{getEfficiency}(E_i, k))$
其中 $j \ge k$。

**初始状态：**
- $dp[0][0] = 0$。
- 其余初始化为负无穷或 0（本题效率均为正，初始化为 0 即可）。

**优化：**
- 空间复杂度可以优化为一维数组（滚动数组），因为 $dp[i]$ 只依赖于 $dp[i-1]$。
- 如果志愿者人数 $V > 4 \times S$，则多出的志愿者无效，可以将 $V$ 限制在 $4 \times S$。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 读取采样员人数和志愿者人数
        if (!sc.hasNextInt()) return;
        int S = sc.nextInt();
        int V = sc.nextInt();

        int[] baseEfficiencies = new int[S];
        for (int i = 0; i < S; i++) {
            baseEfficiencies[i] = sc.nextInt();
        }

        // 每个采样员最多只能有效利用4个志愿者
        int maxEffectiveVolunteers = Math.min(V, S * 4);

        // dp[j] 表示分配 j 个志愿者的最大效率
        long[] dp = new long[maxEffectiveVolunteers + 1];

        // 初始化第一个采样员的情况
        int e0 = baseEfficiencies[0];
        int m0 = e0 / 10;
        for (int j = 0; j <= maxEffectiveVolunteers; j++) {
            dp[j] = calculateEff(e0, m0, j);
        }

        // 遍历剩下的采样员
        for (int i = 1; i < S; i++) {
            int e = baseEfficiencies[i];
            int m = e / 10;
            // 从后往前更新，防止重复计算（0/1背包思想）
            for (int j = maxEffectiveVolunteers; j >= 0; j--) {
                long maxVal = 0;
                // 尝试给当前采样员分配 k 个志愿者
                for (int k = 0; k <= 4 && k <= j; k++) {
                    maxVal = Math.max(maxVal, dp[j - k] + calculateEff(e, m, k));
                }
                dp[j] = maxVal;
            }
        }

        System.out.println(dp[maxEffectiveVolunteers]);
    }

    /**
     * 计算单个采样员在分配 k 个志愿者时的效率
     */
    private static int calculateEff(int e, int m, int k) {
        if (k == 0) return e - 2 * m;
        if (k == 1) return e;
        if (k == 2) return e + m;
        if (k == 3) return e + 2 * m;
        return e + 3 * m; // k >= 4
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(S \times V \times K)$，其中 $S$ 是采样员人数，$V$ 是志愿者人数，$K$ 是每个采样员分配志愿者的上限（常数 5）。代入最大值 $100 \times 500 \times 5 = 250,000$，远小于机考的性能阈值。
- **空间复杂度**：$O(V)$。通过滚动数组优化后，只需要存储志愿者维度的状态。
