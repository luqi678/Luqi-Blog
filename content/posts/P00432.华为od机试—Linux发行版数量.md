## 二、 题目描述

Linux 操作系统有多个发行版，distrowatch.com 提供了各个发行版的资料。这些发行版互相存在关联，例如 Ubuntu 基于 Debian 开发，而 Mint 又基于 Ubuntu 开发，那么我们认为 Mint 同 Debian 也存在关联。

发行版集是一个或多个相关存在关联的操作系统发行版，集合内不包含没有关联的发行版。给你一个 $n \times n$ 的矩阵 `isConnected`，其中 `isConnected[i][j] = 1` 表示第 $i$ 个发行版和第 $j$ 个发行版直接关联，而 `isConnected[i][j] = 0` 表示二者不直接相连。

返回最大的发行版集中发行版的数量。

## 三、 输入描述

第一行输入发行版的总数量 $N$。
之后每行表示各发行版间是否直接相关（即一个 $N \times N$ 的邻接矩阵）。

$1 \le N \le 200$

## 四、 输出描述

输出最大的发行版集中发行版的数量。

## 五、 用例

### 用例1

**输入：**
```text
4
1 1 0 0
1 1 1 0
0 1 1 0
0 0 0 1
```

**输出：**
```text
3
```

**说明：**
发行版0和1直接关联，1和2直接关联，因此{0, 1, 2}属于同一个发行版集。发行版3自成一个集合。最大的集合大小为3。

### 用例2

**输入：**
```text
3
1 1 0
1 1 0
0 0 1
```

**输出：**
```text
2
```

## 六、 备注

1. 矩阵是对称的，即 `isConnected[i][j] == isConnected[j][i]`。
2. `isConnected[i][i] == 1` 始终成立。

## 七、 题目解析

### 1. 题目分析
本题的本质是求图中**最大连通分量的大小**。
题目给出了一个邻接矩阵，如果 `matrix[i][j] == 1`，则节点 $i$ 和节点 $j$ 连通。连通性具有传递性（$A-B, B-C \Rightarrow A-C$）。我们需要找到包含节点数最多的那个连通分量，并输出其节点个数。

### 2. 解题思路
解决此类问题通常有两种主流算法：

1.  **并查集 (Disjoint Set Union, DSU)**：
    *   初始化 $N$ 个集合，每个集合初始大小为1。
    *   遍历矩阵的所有对 $(i, j)$，如果 `matrix[i][j] == 1`，则合并 $i$ 和 $j$ 的集合。
    *   在合并过程中维护每个集合的 `size`。
    *   最后找出最大的 `size`。

2.  **深度优先搜索 (DFS) / 广度优先搜索 (BFS)**：
    *   使用一个 `visited` 数组记录节点是否被访问过。
    *   遍历每个节点 $i$，如果 $i$ 未被访问，则从 $i$ 开始进行一次 DFS 或 BFS。
    *   在一次搜索过程中，每访问到一个新节点，计数器加1。
    *   一次完整的搜索结束后，得到一个连通分量的大小，更新全局最大值。

鉴于 $N$ 的范围较小 ($N \le 200$)，两种方法的时间复杂度均为 $O(N^2)$，均可轻松通过。

### 3. 代码实现

#### Java8实现 (采用 BFS)

```java
import java.util.LinkedList;
import java.util.Queue;
import java.util.Scanner;

public class MaxDistributionSet {

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        // 1. 读取输入
        if (sc.hasNext()) {
            int n = sc.nextInt();
            
            // 读取邻接矩阵
            int[][] isConnected = new int[n][n];
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    isConnected[i][j] = sc.nextInt();
                }
            }
            
            // 2. 计算最大连通分量
            System.out.println(findMaxComponentSize(n, isConnected));
        }
        
        sc.close();
    }

    /**
     * 使用 BFS 寻找最大的发行版集数量
     */
    private static int findMaxComponentSize(int n, int[][] isConnected) {
        boolean[] visited = new boolean[n]; // 记录是否被访问过
        int maxCount = 0; // 记录最大的集合数量

        // 遍历每一个节点，如果是新的未访问节点，则是一个新的连通分量
        for (int i = 0; i < n; i++) {
            if (!visited[i]) {
                // 发现新大陆，开始 BFS 统计这个圈子有多少人
                int currentCount = bfs(i, n, isConnected, visited);
                maxCount = Math.max(maxCount, currentCount);
            }
        }
        
        return maxCount;
    }

    /**
     * 标准 BFS 模板
     * @param startNode 起始节点
     * @param n 总节点数
     * @param grid 邻接矩阵
     * @param visited 访问记录数组
     * @return 当前连通分量的节点个数
     */
    private static int bfs(int startNode, int n, int[][] grid, boolean[] visited) {
        Queue<Integer> queue = new LinkedList<>();
        
        // 起始点入队
        queue.offer(startNode);
        visited[startNode] = true;
        
        int count = 0;

        while (!queue.isEmpty()) {
            int curr = queue.poll();
            count++; // 统计当前集合的数量

            // 遍历当前节点 curr 的所有邻居
            for (int j = 0; j < n; j++) {
                // 如果 j 是 curr 的邻居 (grid[curr][j] == 1)，且 j 还没被访问过
                if (grid[curr][j] == 1 && !visited[j]) {
                    queue.offer(j);
                    visited[j] = true; // 入队时立即标记，防止重复入队
                }
            }
        }
        
        return count;
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(N^2)$。我们需要读取整个 $N \times N$ 的矩阵，并且在 DFS 遍历过程中，每个节点被访问一次，每个边（矩阵中的 1）被检查两次。
*   **空间复杂度**：$O(N^2)$ 用于存储邻接矩阵（如果是流式处理可以优化到 $O(N)$），以及 $O(N)$ 用于 `visited` 数组和递归调用栈。由于 $N=200$，空间压力极小。
