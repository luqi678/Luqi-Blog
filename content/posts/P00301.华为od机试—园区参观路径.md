## 二、 题目描述

园区某部门举办了 Family Day，邀请员工及其家属参加；将公司园区视为一个矩形，起始园区设置在左上角，终点园区设置在右下角；家属参观园区时，只能向右和向下园区前进，求从起始园区到终点园区会有多少条不同的参观路径。

- 0 表示可以参观。
- 1 表示不能参观（障碍物）。

## 二、 输入描述

第一行为园区的行数 $M$ 和列数 $N$（中间用空格隔开）。
后面 $M$ 行，每行表示该园区是否可以参观，0 表示可以参观，1 表示不能参观。

## 三、 输出描述

输出为一个整数，表示不同的路径数量。

## 四、 用例

### 用例1

**输入：**
```text
3 3
0 0 0
0 1 0
0 0 0
```

**输出：**
```text
2
```

**说明：**
从 (0,0) 到 (2,2)，中间 (1,1) 是障碍。
路径1：(0,0) -> (0,1) -> (0,2) -> (1,2) -> (2,2)
路径2：(0,0) -> (1,0) -> (2,0) -> (2,1) -> (2,2)
共 2 条。

### 用例2

**输入：**
```text
2 2
0 1
0 0
```

**输出：**
```text
1
```

## 五、 备注
- $1 \le M, N \le 100$
- 结果可能较大，建议使用 `long` 类型存储路径数量。
- 如果起点或终点本身就是 1（不可参观），则路径数为 0。

## 六、 题目解析

### 1. 题目分析
本题是经典的**矩阵路径计数**问题。由于移动规则限制为“只能向右”和“向下”，这意味着到达任何一个点 $(i, j)$ 的路径，只能来自于它的上方点 $(i-1, j)$ 或左方点 $(i, j-1)$。

### 2. 解题思路
我们采用 **动态规划 (Dynamic Programming)** 来解决：

1.  **定义状态**：设 `dp[i][j]` 为从起点 $(0,0)$ 到达位置 $(i, j)$ 的不同路径总数。
2.  **转移方程**：
    - 如果当前位置 `grid[i][j] == 1`，则该点不可达，`dp[i][j] = 0`。
    - 如果当前位置 `grid[i][j] == 0`，则 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`。
3.  **边界条件**：
    - 起点 `dp[0][0]`：如果 `grid[0][0] == 0`，则 `dp[0][0] = 1`；否则为 0。
    - 第一行：只能从左边走过来，如果遇到障碍，其后所有格子都不可达。
    - 第一列：只能从上边走过来，如果遇到障碍，其下所有格子都不可达。
4.  **最终结果**：`dp[M-1][N-1]`。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();
        int n = sc.nextInt();
        
        int[][] grid = new int[m][n];
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                grid[i][j] = sc.nextInt();
            }
        }
        
        // 如果起点或终点是障碍，直接输出 0
        if (grid[0][0] == 1 || grid[m - 1][n - 1] == 1) {
            System.out.println(0);
            return;
        }
        
        long[][] dp = new long[m][n];
        
        // 初始化起点
        dp[0][0] = 1;
        
        // 初始化第一行
        for (int j = 1; j < n; j++) {
            if (grid[0][j] == 0) {
                dp[0][j] = dp[0][j - 1];
            } else {
                dp[0][j] = 0; // 遇到障碍，后面都无法到达
            }
        }
        
        // 初始化第一列
        for (int i = 1; i < m; i++) {
            if (grid[i][0] == 0) {
                dp[i][0] = dp[i - 1][0];
            } else {
                dp[i][0] = 0; // 遇到障碍，下面都无法到达
            }
        }
        
        // 填充 DP 表
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                if (grid[i][j] == 0) {
                    dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                } else {
                    dp[i][j] = 0;
                }
            }
        }
        
        System.out.println(dp[m - 1][n - 1]);
    }
}
```

### 4. 复杂度分析
- **时间复杂度**：$O(M \times N)$，需要遍历整个矩阵一次。
- **空间复杂度**：$O(M \times N)$，使用了一个额外的二维数组 `dp`。
    - *优化建议*：可以使用滚动数组将空间复杂度降低到 $O(N)$。
