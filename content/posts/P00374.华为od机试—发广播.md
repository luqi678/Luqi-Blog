## 一、 题目描述

某地有 $N$ 个广播站，站点之间有些有连接，有些没有。有连接的站点在接受到广播后会互相发送。
给定一个 $N \times N$ 的二维数组 `matrix`，数组的元素都是字符 `'0'` 或者 `'1'`。
- `matrix[i][j] = '1'`，代表 $i$ 和 $j$ 站点之间有连接；
- `matrix[i][j] = '0'`，代表没连接。

现在要发一条广播，问初始最少给几个广播站发送，才能保证所有的广播站都收到消息。

## 二、输入描述

输入为 $N$ 行，每行有 $N$ 个数字，为 $0$ 或 $1$，由空格分隔，构成 $N \times N$ 的数组。
$N$ 的范围为 $1 \le N \le 50$。

## 三、输出描述

输出一个数字，为需要广播的服务器数量。

## 四、用例

### 用例1
**输入**
```text
1 0 0
0 1 0
0 0 1
```
**输出**
```text
3
```
**说明**
3 台服务器互不连接，所以需要分别广播这 3 台服务器。

### 用例2
**输入**
```text
1 1
1 1
```
**输出**
```text
1
```
**说明**
2 台服务器相互连接，所以只需要广播其中一台服务器。

## 五、备注
- 站点自身与自身是连接的（即对角线 `matrix[i][i]` 通常为 1）。
- 连接具有传递性：如果 A 与 B 连接，B 与 C 连接，那么给 A 发广播，B 和 C 都能收到。

## 六、题目解析

### 1. 题目分析
本题可以抽象为一个**无向图的连通分量**问题。
- 每个广播站是图中的一个节点。
- `matrix[i][j] = '1'` 表示节点 $i$ 和节点 $j$ 之间有一条边。
- 题目要求“最少给几个站发广播能覆盖全网”，等价于求这个图中有多少个**最大连通分量**。每一个连通分量只需要一台服务器接收广播，整个分量内的所有节点就都能收到。

### 2. 解题思路
解决连通分量问题主要有两种常用算法：
1.  **并查集 (Disjoint Set Union, DSU)**：遍历所有的连接关系，将有连接的两个站点进行 `union` 操作，最后统计集合（根节点）的数量。
2.  **搜索算法 (DFS/BFS)**：维护一个 `visited` 数组，从每个未访问的节点出发进行一次完整的遍历，每进行一次完整的搜索（直到没有新节点可达），就代表发现了一个新的连通分量。

由于 $N$ 的范围很小（最大 50），两种方法的时间复杂度均能轻松通过。

### 3. 代码实现

#### Java8实现（并查集方案）

```java
import java.util.Scanner;
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        List<String[]> lines = new ArrayList<>();
        
        // 题目未给出N的具体数值，先读取第一行来确定N
        if (!sc.hasNextLine()) return;
        String firstLine = sc.nextLine().trim();
        if (firstLine.isEmpty()) return;
        
        String[] firstRow = firstLine.split("\\s+");
        int n = firstRow.length;
        lines.add(firstRow);
        
        // 读取后续的 n-1 行
        for (int i = 1; i < n; i++) {
            if (sc.hasNextLine()) {
                lines.add(sc.nextLine().trim().split("\\s+"));
            }
        }

        // 初始化并查集
        int[] parent = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
        }

        // 遍历矩阵，进行合并
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (lines.get(i)[j].equals("1")) {
                    union(parent, i, j);
                }
            }
        }

        // 统计连通分量个数
        int count = 0;
        for (int i = 0; i < n; i++) {
            if (parent[i] == i) {
                count++;
            }
        }
        System.out.println(count);
    }

    // 并查集查找根节点（路径压缩）
    private static int find(int[] parent, int i) {
        if (parent[i] == i) {
            return i;
        }
        return parent[i] = find(parent, parent[i]);
    }

    // 并查集合并
    private static void union(int[] parent, int i, int j) {
        int rootI = find(parent, i);
        int rootJ = find(parent, j);
        if (rootI != rootJ) {
            parent[rootI] = rootJ;
        }
    }
}
```

#### Python实现（DFS方案）

```python
import sys

def solve():
    # 读取所有输入
    input_data = sys.stdin.read().splitlines()
    if not input_data:
        return
    
    matrix = []
    for line in input_data:
        row = line.strip().split()
        if row:
            matrix.append(row)
    
    n = len(matrix)
    visited = [False] * n
    count = 0

    def dfs(curr):
        for neighbor in range(n):
            if matrix[curr][neighbor] == '1' and not visited[neighbor]:
                visited[neighbor] = True
                dfs(neighbor)

    for i in range(n):
        if not visited[i]:
            count += 1
            visited[i] = True
            dfs(i)
            
    print(count)

if __name__ == "__main__":
    solve()
```

### 4. 复杂度分析
- **时间复杂度**：$O(N^2)$。我们需要遍历 $N \times N$ 的矩阵来获取所有的边关系。并查集的合并和查找操作在路径压缩下几乎是常数级别的，而 DFS 也会访问每个节点和每条边一次。
- **空间复杂度**：$O(N^2)$ 用于存储输入的矩阵（如果直接处理流可以降到 $O(N)$），并查集或 `visited` 数组需要 $O(N)$ 的空间。由于 $N \le 50$，该复杂度非常理想。
