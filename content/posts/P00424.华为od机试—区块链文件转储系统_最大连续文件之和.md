## 二、题目描述

区块链底层存储是一个链式文件系统，由顺序的 $N$ 个文件组成，每个文件的大小不一，依次为 $F_1, F_2, ..., F_n$。随着时间的推移，所占存储会越来越大。

云平台考虑将区块链按文件转储到廉价的 SATA 盘，只有**连续**的区块链文件才能转储到 SATA 盘上，且转的文件之和不能超过 SATA 盘的容量。

假设每块 SATA 盘容量为 $M$，求能转储的最大连续文件之和。

## 二、输入描述

第一行为 SATA 盘容量 $M$，$1000 \le M \le 1,000,000$。
第二行为区块链文件大小序列 $F_1, F_2, ..., F_n$。其中 $1 \le n \le 100,000$，$1 \le F_i \le 500$。

## 三、输出描述

求能转储的最大连续文件大小之和。

## 四、用例

### 用例1

**输入**
```
1000
100 300 500 400 400 150 100
```
**输出**
```
950
```
**说明**
最大序列和为 950，序列为 `[400, 400, 150]`。

### 用例2

**输入**
```
1000
100 500 400 150 500 100
```
**输出**
```
1000
```
**说明**
最大序列和为 1000，序列为 `[100, 500, 400]`。

## 五、备注
- 文件必须是连续的。
- 文件大小均为正整数。
- 需要在 $O(n)$ 时间复杂度内解决，以应对 $n=100,000$ 的数据规模。

## 六、题目解析

### 1. 题目分析
本题要求在给定的正整数数组中，找到一个**连续子数组**，使得其元素之和最大，且该和不能超过给定的容量 $M$。

### 2. 解题思路
由于数组中的所有元素都是正整数（$F_i \ge 1$），随着子数组长度的增加，其和具有单调递增性。这非常适合使用**滑动窗口（Sliding Window）**或**双指针**算法来解决：

1.  初始化两个指针 `left` 和 `right` 都指向序列的起始位置，初始化 `currentSum` 为 0，`maxSum` 为 0。
2.  移动 `right` 指针，将当前指向的文件大小累加到 `currentSum` 中。
3.  如果 `currentSum` 超过了 SATA 盘的容量 $M$：
    -   移动 `left` 指针，从 `currentSum` 中减去 `left` 指向的文件大小，直到 `currentSum` 小于等于 $M$。
4.  在每一步（确保 `currentSum <= M` 后），更新 `maxSum = max(maxSum, currentSum)`。
5.  如果 `maxSum` 已经达到了 $M$，可以直接提前结束循环（因为不可能有更大的值了）。
6.  遍历结束后，`maxSum` 即为所求。

### 3. 代码实现

#### Java8实现

```java
import java.util.Scanner;
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 处理输入：SATA盘容量 M
        if (!sc.hasNextInt()) return;
        int m = sc.nextInt();
        sc.nextLine(); // 消耗换行符

        // 处理输入：区块链文件序列
        if (!sc.hasNextLine()) return;
        String line = sc.nextLine();
        int[] files = Arrays.stream(line.split(" "))
                            .mapToInt(Integer::parseInt)
                            .toArray();

        int n = files.length;
        int left = 0;
        int currentSum = 0;
        int maxSum = 0;

        // 滑动窗口
        for (int right = 0; right < n; right++) {
            currentSum += files[right];

            // 如果当前窗口和大于M，则缩小左边界
            while (currentSum > m && left <= right) {
                currentSum -= files[left];
                left++;
            }

            // 更新最大值
            if (currentSum <= m) {
                maxSum = Math.max(maxSum, currentSum);
            }

            // 如果已经达到最大可能值M，可直接退出
            if (maxSum == m) {
                break;
            }
        }

        System.out.println(maxSum);
    }
}
```

### 4. 复杂度分析

*   **时间复杂度**：$O(n)$。其中 $n$ 是文件的数量。每个元素最多被 `right` 指针访问一次，被 `left` 指针访问一次，总操作次数为 $2n$。
*   **空间复杂度**：$O(n)$。需要存储输入的文件序列。如果直接流式读取处理，可以将空间优化到 $O(1)$，但在机考环境下，$O(n)$ 的存储完全可以接受。
